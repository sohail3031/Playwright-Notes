In this lesson, the focus is on one of the most important concepts in Playwright: **Auto-Waiting**.

Auto-waiting helps make tests:

- More reliable    
- Less flaky
- Easier to maintain

---

# What is Auto-Waiting?

Many Playwright actions automatically wait for conditions before performing the action.

Examples:

- `click()`    
- `fill()`
- `textContent()`
- locator assertions like `toHaveText()`

These methods wait until the element is:

- Attached to the DOM
- Visible
- Stable
- Enabled
- Ready for interaction

Default timeout:

- **30 seconds** for actions
- **5 seconds** for locator assertions

---

# Base Code

```typescript
import { test, expect } from "@playwright/test";

test.beforeEach(async ({ page }) => {
  await page.goto("http://uitestingplayground.com/ajax");
  await page.getByText("Button Triggering AJAX Request").click();
});
```

The page delays showing the success message for about **15 seconds**.

---

# Example 1 — Auto-Wait with `click()`

```typescript
test("auto waiting", async ({ page }) => {
  const successButton = page.locator(".bg-success");

  await successButton.click();
});
```

## What Happens?

- Playwright waits automatically for `.bg-success`    
- Once the button appears, it performs the click
- Since the message appears after 15 seconds and default timeout is 30 seconds → test passes

---

# Example 2 — Reducing Global Timeout

```typescript
// playwright.config.ts
export default defineConfig({
  timeout: 10000,
})
```

```typescript
test("auto waiting", async ({ page }) => {
  const successButton = page.locator(".bg-success");

  await successButton.click();
});
```

## Why This Test Fails

The AJAX response takes:

- 15 seconds    

But timeout is:

- 10 seconds

So Playwright stops waiting and throws a timeout error.

---

# Example 3 — `textContent()` Supports Auto-Waiting

```typescript
test("auto waiting", async ({ page }) => {
  const successButton = page.locator(".bg-success");

  const text = await successButton.textContent();

  expect(text).toEqual("Data loaded with AJAX get request.");
});
```

## Why It Works

`textContent()`:

- waits for the element to be attached    
- then extracts the text

So Playwright waits until the success element appears.

---

# Example 4 — `allTextContents()` Does NOT Auto-Wait

```typescript
test("auto waiting", async ({ page }) => {
  const successButton = page.locator(".bg-success");

  const text = await successButton.allTextContents();

  expect(text).toEqual("Data loaded with AJAX get request.");
});
```

## Why This Test Fails

`allTextContents()`:

- does NOT support auto-waiting    

So Playwright executes immediately:

- element not yet visible
- returns empty array/value
- assertion fails

---

# Example 5 — Fixing `allTextContents()` with `waitFor()`

```typescript
test("auto waiting", async ({ page }) => {
  const successButton = page.locator(".bg-success");

  await successButton.waitFor({ state: "attached" });

  const text = await successButton.allTextContents();

  expect(text).toContain("Data loaded with AJAX get request.");
});
```

## Why It Works

`waitFor()` explicitly waits until:

- element is attached to DOM

Then:

- `allTextContents()` runs safely

---

# Example 6 — Locator Assertions Have Built-In Waiting

```typescript
test("auto waiting", async ({ page }) => {
  const successButton = page.locator(".bg-success");

  await expect(successButton).toHaveText("Data loaded with AJAX get request.");
});
```

## Why This Test Fails

Locator assertions:

- wait only 5 seconds by default    

But AJAX response appears after:

- 15 seconds

So assertion times out.

---

# Example 7 — Increasing Assertion Timeout

```typescript
test("auto waiting", async ({ page }) => {
  const successButton = page.locator(".bg-success");

  await expect(successButton).toHaveText(
    "Data loaded with AJAX get request.",
    {
      timeout: 20000,
    }
  );
});
```

## Why It Works

Custom timeout:

- 20 seconds    

Response arrives in:

- 15 seconds

So assertion succeeds.

# Alternative Waiting Methods

---

# 1. `waitForSelector()`

```typescript
test("auto waiting", async ({ page }) => {
  const successButton = page.locator(".bg-success");

  await page.waitForSelector(".bg-success");

  const text = await successButton.allTextContents();

  expect(text).toContain("Data loaded with AJAX get request.");
});
```

## Why It Works

Playwright waits until:

- selector exists in DOM    

Then continues execution.

---

# 2. `waitForResponse()`

```typescript
test("auto waiting", async ({ page }) => {
  const successButton = page.locator(".bg-success");

  await page.waitForResponse(
    "http://uitestingplayground.com/ajaxdata"
  );

  const text = await successButton.allTextContents();

  expect(text).toContain("Data loaded with AJAX get request.");
});
```

## Why It Works

Playwright waits until:

- API/network response finishes    

Useful when UI depends on backend responses.

---

# 3. `waitForLoadState("networkidle")`

```typescript
test("auto waiting", async ({ page }) => {
  const successButton = page.locator(".bg-success");

  await page.waitForLoadState("networkidle");

  const text = await successButton.allTextContents();

  expect(text).toContain("Data loaded with AJAX get request.");
});
```

## Why It Works

Playwright waits until:

- all network activity stops    

## Why This is NOT Recommended

Some applications:

- constantly make background requests

This can:

- slow tests
- create flaky tests
- cause unnecessary waits

Use only when truly needed.

# Common Auto-Wait Conditions

Playwright may wait for:

| Condition       | Meaning                        |
| --------------- | ------------------------------ |
| Attached        | Element exists in DOM          |
| Visible         | User can see it                |
| Stable          | Not moving/animating           |
| Receives Events | Not blocked by another element |
| Enabled         | Not disabled                   |
| Editable        | Input can be typed into        |

---

# Important Difference: General vs Locator Assertions

| Type              | Auto-Wait? | Example                              |
| ----------------- | ---------- | ------------------------------------ |
| General Assertion | ❌ No       | `expect(value).toEqual(5)`           |
| Locator Assertion | ✅ Yes      | `await expect(locator).toHaveText()` |

---

# Best Practices

## Prefer Built-In Auto-Waiting

Good:

```typescript
await button.click();
await expect(locator).toHaveText("Success");
```

Avoid unnecessary manual waits.

---

## Avoid Hard Waits

Bad:

```typescript
await page.waitForTimeout(5000);
```

Why?

- slows tests    
- unreliable
- fragile

Use condition-based waits instead.

---

# Key Takeaways

## Methods WITH auto-waiting

Examples:

- `click()`    
- `fill()`
- `textContent()`
- `toHaveText()`

## Methods WITHOUT auto-waiting

Examples:

- `allTextContents()`
- some utility methods

These may require:

```typescript
await locator.waitFor();
```

---

# Recommended Waiting Strategy

Priority order:

1. Built-in Playwright auto-waiting    
2. Locator assertions
3. `waitFor()` on locator
4. `waitForResponse()` if API-driven
5. `waitForSelector()`
6. `waitForLoadState()` only if necessary
7. `waitForTimeout()` as last resort
