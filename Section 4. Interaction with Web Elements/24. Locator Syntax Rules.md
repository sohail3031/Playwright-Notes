In this lesson, we learn how to locate elements in Playwright using different **locator strategies**. Locators are essential because they allow Playwright to identify and interact with elements on a web page.

---

# Base Setup

Before testing locators, we navigate to the required page using `beforeEach()`:

```typescript
import { test } from "@playwright/test";

test.beforeEach(async ({ page }) => {
  await page.goto("http://localhost:4200/");
  await page.getByText("Forms").click();
  await page.getByText("Form Layouts").click();
});
```

---

# 1. Locate by Tag Name

To locate elements using the HTML tag name:

```typescript
test("Locator Syntax Rules", async ({ page }) => {
  page.locator("input");
});
```

### Explanation

- `input` selects all input elements on the page    
- This is NOT unique if multiple elements exist

---

# 2. Locate by ID

Use `#` followed by the ID value:

```typescript
page.locator("#inputEmail");
```

### Why this works

- `#` represents ID selector in CSS    
- IDs should be unique in the DOM

---

# 3. Locate by Class Name

Use `.` followed by class name:

```typescript
page.locator(".shape-rectangle");
```

### Explanation

- `.` represents class selector    
- Selects all elements with this class
- May return multiple elements

---

# 4. Locate by Attribute

Use square brackets `[]`:

```typescript
page.locator('[placeholder="Email"]');
```

### Explanation

- Matches HTML attributes    
- Very powerful and commonly used

---

# 5. Locate by Full Class Value

```typescript
page.locator(
  '[class="input-full-width size-medium status-basic shape-rectangle nb-transition"]'
);
```

### Explanation

- Matches the entire class string    
- Very strict and fragile
- Not recommended in real projects

---

# 6. Combine Selectors

You can combine multiple selectors for more precision.

### Tag + Attribute + Class

```typescript
page.locator('input[placeholder="Email"].shape-rectangle');
```

### Tag + Multiple Attributes

```typescript
page.locator('input[placeholder="Email"][nbinput]');
```

### Why combine selectors?

- Improves uniqueness    
- Reduces risk of selecting wrong element

---

# 7. Locate by XPath (Not Recommended)

```typescript
page.locator('//*[@id="inputEmail"]');
```

### Why XPath is discouraged

- Tied to DOM structure (fragile)    
- Hard to maintain
- Not recommended by Playwright best practices

---

# 8. Locate by Text (Partial Match)

```typescript
page.locator(':text("Using")');
```

### Explanation

- Matches elements containing text "Using"    
- Useful for dynamic UI

---

# 9. Locate by Exact Text

```typescript
page.locator(':text-is("Using the Grid")');
```

### Explanation

- Matches exact text only    
- More strict than partial match

---

# 10. Important Concept: Locator vs Action

### Locator alone does NOTHING

```typescript
page.locator("#inputEmail");
```

- This only finds the element    
- No interaction happens

---

### Action triggers execution

```typescript
await page.locator("#inputEmail").click();
```

### Why `await` is required?

- `click()` is asynchronous (returns a Promise)    
- Playwright must wait for action completion

---

# 11. Common Mistake: Missing `await`

```typescript
test("Locator Syntax Rules", async ({ page }) => {
  page.locator("#inputEmail").click(); // ❌ wrong method
});
```

Correct version:

```typescript
await page.locator("#inputEmail").click();
```

---

# 12. Handling Multiple Elements

### Problem: Locator returns multiple elements

```typescript
await page.locator("input").click();
```

Error:

- Playwright finds multiple inputs
- Cannot decide which one to click

---

### Solution 1: Use `.first()`

```typescript
await page.locator("input").first().click();
```

### Explanation

- Selects the first matching element

---

### Solution 2: Use `.nth(index)`

```typescript
await page.locator("input").nth(0).click();
```

# 13. Key Locator Rules Summary

## Tag Name

```text
input
```

## ID

```text
#idValue
```

## Class

```text
.className
```

## Attribute

```text
[attribute="value"]
```

## Combined Selectors

```text
input[placeholder="Email"].className
```

## Text

```text
:text("text")
:text-is("exact text")
```

---

# 14. Best Practices

✔ Prefer:

- `getByText`
- `getByRole`
- Stable attributes (`data-testid`)

❌ Avoid:

- XPath
- Full class matching
- Overly complex selectors

---

# Summary

In this lesson, we learned:

- Different locator strategies in Playwright    
- Tag, ID, class, attribute selectors
- Combining multiple selectors
- Text-based locators
- XPath (and why to avoid it)
- Why locators alone don’t execute actions
- Importance of `await`
- Handling multiple elements using `.first()`

These locator strategies form the foundation of Playwright automation.
