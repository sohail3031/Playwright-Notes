Before diving deeper into Playwright automation, locators, and interacting with web elements, it is important to understand some basic **HTML** and **DOM terminology**.

Understanding these concepts is crucial because Playwright interacts directly with DOM elements. Knowing the correct terminology will help you:

- Understand locators better
- Debug automation issues
- Work effectively with Playwright commands

---

# What is the DOM?

DOM stands for:

```text
Document Object Model
```

The DOM represents the structure of a webpage as a tree of HTML elements.

Every webpage consists of:

- HTML tags    
- Attributes
- Text content
- Nested elements

Playwright interacts with these DOM elements during automation.

---

# Basic HTML Structure

Example:

```html
<input
    id="email"
    class="input-full-width size-medium status-basic shape-rectangle"
    placeholder="Email"
>
```

---

# HTML Tags

HTML elements are created using **tags**.

Example:

```html
<input>
```

The tag name here is:

```text
input
```

Tags usually:

- Start with `<`    
- End with `>`

---

# Opening and Closing Tags

Most HTML tags come in pairs:

```html
<div>
    Content
</div>
```

- `<div>` → Opening tag    
- `</div>` → Closing tag

Closing tags contain:

- `/`    
- Same tag name

---

# Self-Closing Tags

Some tags do not require closing tags.

Example:

```html
<input>
```

`input` is a self-closing element.

---

# HTML Attributes

Attributes provide additional information about an element.

Example:

```html
<input id="email" placeholder="Email">
```

Attributes:

- `id`    
- `placeholder`

---

# Attribute Values

Attributes can contain values.

Example:

```html
placeholder="Email"
```

Here:

- `placeholder` → Attribute name    
- `"Email"` → Attribute value

---

# `id` and `class` Attributes

Two of the most important HTML attributes are:

- `id`
- `class`

Example:

```html
<input
    id="email"
    class="input-full-width size-medium status-basic"
>
```

---

# Understanding the `class` Attribute

The `class` attribute can contain multiple values separated by spaces.

Example:

```html
class="input-full-width size-medium status-basic"
```

This actually contains three separate classes:

```text
input-full-width
size-medium
status-basic
```

Each class can be used independently as a CSS locator.

This is very important when working with Playwright locators.

---

# HTML Text

Text between opening and closing tags is called HTML text.

Example:

```html
<button>Sign In</button>
```

The text is:

```text
Sign In
```

---

# Text vs Properties

Not everything visible on a webpage is actual HTML text.

Sometimes visible values are stored inside element properties or attributes instead of text nodes.

Example:

```html
<input placeholder="Enter email">
```

Here:

- `"Enter email"` is a property value    
- It is NOT HTML text    

---

# How to Identify Real HTML Text

Check the DOM structure.

If the value appears between tags:

```html
<button>Login</button>
```

Then it is HTML text.

If the value appears inside an attribute:

```html
<input placeholder="Login">
```

Then it is a property/attribute value.

This distinction is important for:

- Assertions    
- Text validation
- Locator strategies

---

# Nested Elements

HTML elements can contain other elements.

Example:

```html
<div>
    <button>Submit</button>
</div>
```

Here:

- `button` is nested inside `div`    

In browser DevTools:

- Collapsed nested content is often shown with arrows    
- Clicking the arrow expands child elements

---

# Table Structure Example

Tables use specific HTML tags.

Example:

```html
<table>
    <tbody>
        <tr>
            <td>John</td>
            <td>Admin</td>
        </tr>
    </tbody>
</table>
```

---

# Common Table Tags

| Tag       | Meaning           |
| --------- | ----------------- |
| `<table>` | Table container   |
| `<tbody>` | Table body        |
| `<tr>`    | Table row         |
| `<td>`    | Table column/cell |

---

# DOM Relationships

DOM elements have relationships with each other.

These relationships are very important when creating advanced locators.

---

# Parent Elements

Elements above another element are called parent elements.

Example:

```html
<div>
    <button>Click</button>
</div>
```

`div` is the parent of `button`.

---

# Child Elements

Elements inside another element are called child elements.

Example:

```html
<div>
    <button>Click</button>
</div>
```

`button` is the child of `div`.

---

# Sibling Elements

Elements at the same level are called siblings.

Example:

```html
<td>Name</td>
<td>Email</td>
<td>Role</td>
```

All `<td>` elements are sibling elements.

Another example:

```html
<tr></tr>
<tr></tr>
<tr></tr>
```

All `<tr>` elements are siblings.

---

# Anchor or Key Element

Sometimes we use one element as a reference point for locating other elements.

This reference element is often called:

- Anchor element    
- Key element

Other elements are then located relative to it:

- Parent
- Child
- Sibling
