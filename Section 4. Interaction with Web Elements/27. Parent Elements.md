In this lesson, we learn how to locate **parent elements** in Playwright and use them to build more precise and reliable locators.

Parent locators are extremely useful when:

- Multiple similar elements exist on the page    
- Child elements are not unique
- You want to narrow down your search to a specific section or form

---

# Base Setup

```typescript
import { test } from "@playwright/test";

test.beforeEach(async ({ page }) => {
  await page.goto("http://localhost:4200/");
  await page.getByText("Forms").click();
  await page.getByText("Form Layouts").click();
});
```

---

# Why Parent Elements Matter

Suppose the page contains multiple forms:

- Using the Grid    
- Basic Form
- Inline Form

Each form contains:

- Email field
- Password field
- Buttons

If we try this:

```typescript
await page.getByRole("textbox", { name: "Email" }).click();
```

Playwright may find multiple matching email fields.

We need a way to identify the **correct parent form first**, then locate elements inside it.

---

# Understanding the DOM Structure

Example structure:

```html
<nb-card>
  <nb-card-header>Using the Grid</nb-card-header>

  <nb-card-body>
    <input type="email">
  </nb-card-body>
</nb-card>
```

Important relationships:

| Element          | Relationship |
| ---------------- | ------------ |
| `nb-card`        | Parent       |
| `nb-card-header` | Child        |
| `nb-card-body`   | Child        |
| Header & Body    | Siblings     |

---

# 1. Using `hasText` to Locate Parent Elements

We can locate the parent card by its visible text.

---

## Example

```typescript
test("locating parent elements", async ({ page }) => {

  await page
    .locator("nb-card", { hasText: "Using the Grid" })
    .getByRole("textbox", { name: "Email" })
    .click();

});
```

---

# How This Works

Steps:

1. Find all `nb-card` elements    
2. Filter cards containing text `"Using the Grid"`
3. Inside that card, find Email textbox
4. Click the textbox

---

# Why This Is Powerful

Instead of:

- relying on indexes    
- using `.first()`
- using `.nth()`

we narrow the search using meaningful parent context.

---

# 2. Using `has` with Another Locator

Instead of text, we can filter using another locator.

---

## Example

```typescript
test("locating parent elements", async ({ page }) => {

  await page
    .locator("nb-card", {
      has: page.locator("#inputEmail")
    })
    .getByRole("textbox", { name: "Email" })
    .click();

});
```

---

# Explanation

Here:

- `has` checks whether the parent contains a specific locator    
- Playwright returns only matching `nb-card`

---

# `hasText` vs `has`

| Method    | Purpose                   |
| --------- | ------------------------- |
| `hasText` | Filter by visible text    |
| `has`     | Filter by another locator |

---

# 3. Using `.filter()`

Playwright also provides a dedicated `.filter()` method.

---

## Example Using `hasText`

```typescript
test("locating parent elements", async ({ page }) => {

  await page
    .locator("nb-card")
    .filter({ hasText: "Basic form" })
    .getByRole("textbox", { name: "Email" })
    .click();

});
```

---

# Why Use `.filter()`?

Advantages:

- Cleaner syntax    
- Easier to chain multiple filters
- More readable

---

# 4. Filtering by Child Locator

---

## Example

```typescript
test("locating parent elements", async ({ page }) => {

  await page
    .locator("nb-card")
    .filter({
      has: page.locator(".status-danger")
    })
    .getByRole("textbox", { name: "Email" })
    .click();

});
```

---

# Explanation

This:

1. Finds all `nb-card` elements    
2. Keeps only cards containing `.status-danger`
3. Finds Email textbox inside matching card

---

# 5. Chaining Multiple Filters

One of the biggest advantages of `.filter()` is chaining multiple filters.

---

## Example

```typescript
test("locating parent elements", async ({ page }) => {

  await page
    .locator("nb-card")
    .filter({
      has: page.locator("nb-checkbox")
    })
    .filter({
      hasText: "Sign in"
    })
    .getByRole("textbox", { name: "Email" })
    .click();

});
```

---

# How This Works

Steps:

1. Find all `nb-card`    
2. Keep only cards containing checkbox
3. From those cards, keep only cards containing `"Sign in"`
4. Find Email textbox
5. Click it

This creates a unique and stable locator.

---

# Why Chaining Filters Is Useful

It allows us to progressively narrow down results until only one unique element remains.

This is much more reliable than:

- `.nth()`    
- `.first()`
- XPath chains

---

# 6. Going One Level Up to Parent (`..`)

Sometimes we only need to move one level up in the DOM hierarchy.

Playwright allows this using:

```typescript
locator("..")
```

---

## Example

```typescript
test("locating parent elements", async ({ page }) => {

  await page
    .locator(':text-is("Using the Grid")')
    .locator("..")
    .getByRole("textbox", { name: "Email" })
    .click();

});
```

---

# How This Works

Steps:

1. Find text `"Using the Grid"`    
2. Move one level up to parent element
3. Find Email textbox inside parent
4. Click it

---

# Important Note About `..`

This internally behaves like XPath parent traversal.

Although XPath is generally discouraged in Playwright, this is one of the few acceptable use cases.

---

# Recommended Best Practice

Prefer:

- `hasText`    
- `has`
- `.filter()`
- User-facing locators

Avoid relying heavily on:

- XPath
- Indexes
- Element order

---

# Locator Strategy Comparison

| Approach      | Recommended           |
| ------------- | --------------------- |
| `hasText`     | ✅ Yes                 |
| `filter()`    | ✅ Yes                 |
| `has` locator | ✅ Yes                 |
| `.nth()`      | ⚠ Use carefully       |
| XPath         | ⚠ Avoid when possible |
