## Introduction

In this lesson, we learn about **timeouts in Playwright**.

In the previous lesson, we learned about Playwright’s **auto-waiting** feature, where Playwright automatically waits for elements to become visible, clickable, enabled, and ready before performing actions.

Now we will learn:

- What timeouts are    
- Different types of timeouts in Playwright
- Default timeout values
- How to configure timeouts globally
- How to override timeouts for specific tests and actions

---

# Timeout Hierarchy in Playwright

Playwright timeouts are organized in layers.

```text
Global Timeout
    └── Test Timeout
            ├── Action Timeout
            ├── Navigation Timeout
            └── Expect Timeout
```

---

# 1. Global Timeout

## Definition

Global timeout is the **maximum execution time for the entire test run**.

Example:

- Suppose you have:    
    - 10 test files
    - 100 tests

If the total execution time exceeds the global timeout, Playwright terminates the entire run.

---

## Default Value

By default:

```text
Global timeout is NOT configured.
```

---

## Configure Global Timeout

```typescript
// playwright.config.ts
export default defineConfig({
  globalTimeout: 60000,
});
```

```text
60000 ms = 60 seconds
```

---

# 2. Test Timeout

## Definition

Test timeout is the maximum time allowed for a **single test**.

---

## Default Value

```text
30 seconds (30000 ms)
```

---

## Configure Test Timeout Globally

```typescript
// playwright.config.ts
export default defineConfig({
  timeout: 10000,
});
```

This sets:

```text
10 seconds timeout for each test
```

---

# Example — Test Fails Because of Timeout

## Base Code

```typescript
import { test, expect } from "@playwright/test";

test.beforeEach(async ({ page }) => {
  await page.goto("http://uitestingplayground.com/ajax");
  await page.getByText("Button Triggering AJAX Request").click();
});
```

---

## Test

```typescript
test("timeouts", async ({ page }) => {
  const successButton = page.locator(".bg-success");

  await successButton.click();
});
```

The button appears after:

```text
15 seconds
```

But if test timeout is:

```text
10 seconds
```

The test fails.

---

## Error

```bash
Test timeout of 10000ms exceeded.
```

---

# 3. Action Timeout

## Definition

Action timeout controls how long Playwright waits for actions like:

- click()    
- fill()
- check()
- hover()
- type()

---

## Default Value

```text
Not configured by default
```

If not configured, actions use the test timeout.

---

# Configure Action Timeout

```typescript
// playwright.config.ts
export default defineConfig({
  timeout: 40000,

  use: {
    actionTimeout: 5000,
  },
});
```

Now every action waits:

```text
Maximum 5 seconds
```

---

# Example

```typescript
test("timeouts", async ({ page }) => {
  const successButton = page.locator(".bg-success");

  await successButton.click();
});
```

The button appears after:

```text
15 seconds
```

But action timeout is:

```text
5 seconds
```

So the test fails.

---

## Error

```bash
Timeout 5000ms exceeded.
```

---

# Override Action Timeout for a Specific Command

```typescript
test("timeouts", async ({ page }) => {
  const successButton = page.locator(".bg-success");

  await successButton.click({ timeout: 16000 });
});
```

Now:

```text
Click waits up to 16 seconds
```

Since the button appears after 15 seconds, the test passes.

---

# Important Rule

```text
Action timeout cannot exceed test timeout.
```

Example:

```text
Action timeout = 40s
Test timeout = 30s
```

Actual timeout becomes:

```text
30 seconds
```

because the higher-level timeout limits the lower-level timeout.

---

# 4. Navigation Timeout

## Definition

Navigation timeout controls how long Playwright waits for:

- page.goto()    
- page.reload()
- navigation after clicking links

---

## Default Value

```text
Not configured by default
```

---

# Configure Navigation Timeout

```typescript
// playwright.config.ts
export default defineConfig({
  use: {
    navigationTimeout: 5000,
  },
});
```

---

# 5. Expect Timeout

## Definition

Expect timeout controls how long Playwright waits for:

```typescript
await expect(locator).toHaveText();
await expect(locator).toBeVisible();
```

---

## Default Value

```text
5 seconds
```

---

# Important

Only **locator assertions** have timeout.

General assertions do NOT wait.

---

## Example

### Locator Assertion

```typescript
await expect(button).toHaveText("Submit");
```

Playwright waits up to:

```text
5 seconds
```

for the text to appear.

---

## General Assertion

```typescript
expect(value).toEqual("Submit");
```

This executes immediately.

No waiting happens.

---

# Configure Expect Timeout

```typescript
// playwright.config.ts
export default defineConfig({
  expect: {
    timeout: 2000,
  },
});
```

Now locator assertions wait:

```text
2 seconds
```

---

# Full Playwright Config Example

```typescript
// playwright.config.ts
export default defineConfig({
  timeout: 10000,

  globalTimeout: 60000,

  expect: {
    timeout: 2000,
  },

  use: {
    actionTimeout: 5000,
    navigationTimeout: 5000,
  },
});
```

---

# Override Timeout for a Specific Test

```typescript
test("timeouts", async ({ page }) => {
  test.setTimeout(10000);

  const successButton = page.locator(".bg-success");

  await successButton.click({ timeout: 16000 });
});
```

This test still fails because:

```text
Test timeout = 10 seconds
```

Even though click timeout is 16 seconds.

---

# test.slow()

Sometimes tests are naturally slow.

Playwright provides:

```typescript
test.slow();
```

This multiplies the test timeout by:

```text
3x
```

---

# Example

```typescript
test("timeouts", async ({ page }) => {
  test.slow();

  const successButton = page.locator(".bg-success");

  await successButton.click();
});
```

If default timeout is:

```text
10 seconds
```

Then with:

```typescript
test.slow();
```

Actual timeout becomes:

```text
30 seconds
```

---

# Modify Timeout Using testInfo

You can dynamically modify timeout inside hooks.

---

## Example

```typescript
import { test, expect } from "@playwright/test";

test.beforeEach(async ({ page }, testInfo) => {
  await page.goto("http://uitestingplayground.com/ajax");

  await page.getByText("Button Triggering AJAX Request").click();

  testInfo.setTimeout(testInfo.timeout + 2000);
});
```

This increases timeout by:

```text
2 seconds
```

for every test in the suite.

---

# Complete Example

```typescript
// playwright.config.ts
export default defineConfig({
  timeout: 10000,

  globalTimeout: 60000,

  expect: {
    timeout: 2000,
  },

  use: {
    navigationTimeout: 5000,
  },
});
```

---

```typescript
import { test, expect } from "@playwright/test";

test.beforeEach(async ({ page }, testInfo) => {
  await page.goto("http://uitestingplayground.com/ajax");

  await page.getByText("Button Triggering AJAX Request").click();

  testInfo.setTimeout(testInfo.timeout + 2000);
});

test("timeouts", async ({ page }) => {
  test.slow();

  const successButton = page.locator(".bg-success");

  await successButton.click();
});
```

# Summary

## Timeout Types

| Timeout Type       | Purpose               | Default        |
| ------------------ | --------------------- | -------------- |
| Global Timeout     | Entire test run       | Not configured |
| Test Timeout       | Single test execution | 30 seconds     |
| Action Timeout     | click(), fill(), etc. | Not configured |
| Navigation Timeout | page navigation       | Not configured |
| Expect Timeout     | Locator assertions    | 5 seconds      |

---

# Key Concepts

## Global Timeout

- Applies to entire test suite    

---

## Test Timeout

- Applies to a single test    
- Default is 30 seconds

---

## Action Timeout

- Controls actions like click/fill    
- Can be overridden per command

---

## Navigation Timeout

- Controls page navigation waiting    

---

## Expect Timeout

- Applies only to locator assertions    
- Default is 5 seconds

---

# Important Rules

## Rule 1

```text
Child timeouts cannot exceed parent timeouts.
```

Example:

```text
Action Timeout ≤ Test Timeout ≤ Global Timeout
```

---

## Rule 2

Locator assertions wait automatically.

General assertions do not wait.

---

## Rule 3

Timeouts can be configured:

- Globally    
- Per test
- Per action
- Per assertion

---

# Best Practices

## Recommended

- Keep default test timeout reasonable    
- Override only when needed
- Use test.slow() for slow tests
- Prefer locator assertions

---

## Avoid

- Extremely high global timeouts    
- Excessive hard waits
- Large action timeouts everywhere
