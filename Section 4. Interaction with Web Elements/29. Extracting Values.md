In this lesson, we learn how to extract different types of values from the DOM using Playwright.

We will cover:

- Extracting single text values    
- Extracting multiple text values
- Extracting input field values
- Extracting attribute values

These techniques are extremely important for validations and assertions in automation testing.

---

# Base Setup

```javascript
import { test, expect } from "@playwright/test";

test.beforeEach(async ({ page }) => {
  await page.goto("http://localhost:4200/");
  await page.getByText("Forms").click();
  await page.getByText("Form Layouts").click();
});
```

---

# 1. Extracting a Single Text Value

Suppose we want to validate that the **Submit** button inside the **Basic Form** contains the correct text.

---

## Example

```javascript
test("extracting values", async ({ page }) => {

  // get text value -> passing test

  const basicForm = page
    .locator("nb-card")
    .filter({ hasText: "Basic form" });

  const buttonText = await basicForm
    .locator("button")
    .textContent();

  expect(buttonText).toEqual("Submit");

});
```

---

# Understanding `textContent()`

```javascript
const buttonText = await locator.textContent();
```

### What It Does

- Extracts the visible text inside an HTML element    
- Returns the text as a string

---

# Passing Assertion

```javascript
expect(buttonText).toEqual("Submit");
```

The test passes because the button text is exactly `"Submit"`.

---

# Example of a Failing Test

```javascript
test("extracting values", async ({ page }) => {

  // get text value -> failing test

  const basicForm = page
    .locator("nb-card")
    .filter({ hasText: "Basic form" });

  const buttonText = await basicForm
    .locator("button")
    .textContent();

  expect(buttonText).toEqual("Submit1");

});
```

---

# Why This Test Fails

Expected:

```text
Submit1
```

Actual value:

```text
Submit
```

Since the values are different, the assertion fails.

---

# 2. Extracting Multiple Text Values

Suppose we want to collect all radio button labels from the page.

Example:

- Option 1    
- Option 2
- Disabled Option

---

## Example

```javascript
test("extracting values", async ({ page }) => {

  // get all text values -> passing test

  const allRadioButtonsLabels = await page
    .locator("nb-radio")
    .allTextContents();

  expect(allRadioButtonsLabels).toContain("Option 1");

});
```

---

# Understanding `allTextContents()`

```javascript
await locator.allTextContents();
```

### What It Does

- Extracts text from all matching elements    
- Returns the values as an array

Example output:

```javascript
[
  "Option 1",
  "Option 2",
  "Disabled Option"
]
```

---

# Assertion

```javascript
expect(allRadioButtonsLabels).toContain("Option 1");
```

This verifies that `"Option 1"` exists inside the array.

---

# Example of a Failing Test

```javascript
test("extracting values", async ({ page }) => {

  // get all text values -> failing test

  const allRadioButtonsLabels = await page
    .locator("nb-radio")
    .allTextContents();

  expect(allRadioButtonsLabels).toContain("Option 12");

});
```

---

# Why This Test Fails

The array contains:

```javascript
[
  "Option 1",
  "Option 2",
  "Disabled Option"
]
```

But it does NOT contain:

```javascript
"Option 12"
```

So the assertion fails.

# 3. Extracting Input Field Values

Important concept:

Not every visible value on the page is actual text inside the DOM.

Input fields store values as properties.

---

# Example

```javascript
test("extracting values", async ({ page }) => {

  // get input value

  const basicForm = page
    .locator("nb-card")
    .filter({ hasText: "Basic form" });

  const emailField = basicForm.getByRole("textbox", {
    name: "Email"
  });

  await emailField.fill("test@test.com");

  const emailValue = await emailField.inputValue();

  expect(emailValue).toEqual("test@test.com");

});
```

---

# Understanding `inputValue()`

```javascript
await locator.inputValue();
```

### What It Does

- Extracts the current value inside an input field    
- Returns the typed value as a string

---

# Why `textContent()` Does NOT Work Here

When text is typed into an input field:

```html
<input value="test@test.com">
```

The value is stored as a property, not as visible DOM text.

So:

- `textContent()` → does NOT work    
- `inputValue()` → works correctly

---

# 4. Extracting Attribute Values

Sometimes we need to validate HTML attributes.

Example:

```html
<input placeholder="Email">
```

We may want to validate the placeholder value.

---

## Example

```javascript
test("extracting values", async ({ page }) => {

  // get attribute value

  const basicForm = page
    .locator("nb-card")
    .filter({ hasText: "Basic form" });

  const emailField = basicForm.getByRole("textbox", {
    name: "Email"
  });

  const placeholderValue =
    await emailField.getAttribute("placeholder");

  expect(placeholderValue).toEqual("Email");

});
```

---

# Understanding `getAttribute()`

```javascript
await locator.getAttribute("placeholder");
```

### What It Does

- Retrieves the value of a specific HTML attribute    
- Returns the attribute value as a string

---

# Commonly Used Attributes

| Attribute     | Example      |
| ------------- | ------------ |
| `placeholder` | `"Email"`    |
| `type`        | `"password"` |
| `href`        | Link URL     |
| `src`         | Image source |
| `title`       | Tooltip text |
| `value`       | Input value  |

---

# Summary of Extraction Methods

| Method              | Purpose                   |
| ------------------- | ------------------------- |
| `textContent()`     | Extract single text       |
| `allTextContents()` | Extract multiple texts    |
| `inputValue()`      | Extract input field value |
| `getAttribute()`    | Extract attribute value   |

---

# Important Notes

## Use `textContent()` When:

- Working with normal HTML text    
- Buttons
- Labels
- Paragraphs
- Headers

---

## Use `inputValue()` When:

- Working with `<input>` fields    
- Working with textboxes
- Working with typed user data

---

## Use `getAttribute()` When:

- Validating HTML attributes    
- Checking placeholders
- Checking URLs
- Validating element properties

---

# Best Practices

## Store Values in Variables

Good practice:

```javascript
const buttonText = await locator.textContent();
```

instead of using everything inline.

---

## Use Assertions After Extraction

Always validate extracted values using `expect()`.

---

## Choose the Correct Extraction Method

Do not use:

- `textContent()` for input values    

Use:

- `inputValue()` instead
