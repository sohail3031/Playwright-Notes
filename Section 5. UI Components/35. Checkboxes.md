In this lesson, we learn how to work with **checkboxes** in Playwright.

On the **Modal & Overlays → Toastr** page, there are three checkboxes. Two are selected by default, and one is not. This lesson covers how to interact with them using Playwright and the difference between using `click()`, `check()`, and `uncheck()`.

---

```typescript
import { test, expect } from "@playwright/test";

test.beforeEach(async ({ page }) => {
  await page.goto("http://localhost:4200/");
});
```

## Navigating to the Checkbox Page

To work with the checkboxes, we first navigate to:

- **Modal & Overlays**    
- **Toastr**

This page contains the checkboxes used in the examples below.

---

## Selecting a Checkbox Using `click()`

To interact with a checkbox, we first locate it. In this case, the checkbox can be selected by role because it has a label and is exposed as a checkbox element.

Example: selecting the **Hide on click** checkbox using `click()`.

```typescript
// This will not work because the checkbox is visually hidden
test("checkbox", async ({ page }) => {
  await page.getByText("Modal & Overlays").click();
  await page.getByText("Toastr").click();
  await page.getByRole("checkbox", { name: "Hide on click" }).click();
});
```

The above test fails because the checkbox is **hidden**. Since it is visually hidden, we need to use `{ force: true }`.

```typescript
test("checkbox", async ({ page }) => {
  await page.getByText("Modal & Overlays").click();
  await page.getByText("Toastr").click();
  await page.getByRole("checkbox", { name: "Hide on click" }).click({ force: true });
});
```

---

## Using `check()` Instead of `click()`

Playwright provides a dedicated method for checkboxes: `check()`.

```typescript
// The check() method only checks the checkbox if it is currently unchecked
test("checkbox", async ({ page }) => {
  await page.getByText("Modal & Overlays").click();
  await page.getByText("Toastr").click();
  await page.getByRole("checkbox", { name: "Hide on click" }).check({ force: true });
});
```

### Difference between `click()` and `check()`

- `click()` simply performs a mouse click on the checkbox.    
- `check()` first verifies the state of the checkbox:
    - If the checkbox is already checked, it does nothing.
    - If the checkbox is unchecked, it checks it.

So, `check()` is safer and more appropriate when your intention is specifically to ensure that the checkbox is checked.

---

## Using `uncheck()`

If you want to clear a checkbox, use `uncheck()`.

```typescript
// uncheck() will uncheck the checkbox
test("checkbox", async ({ page }) => {
  await page.getByText("Modal & Overlays").click();
  await page.getByText("Toastr").click();
  await page.getByRole("checkbox", { name: "Hide on click" }).uncheck({ force: true });
});
```

### Behavior of `uncheck()`

- If the checkbox is already unchecked, Playwright does nothing.    
- If it is checked, Playwright unchecks it.

---

## Working with Multiple Checkboxes

We can also combine `check()` and `uncheck()` for different checkboxes on the page.

```typescript
test("checkbox", async ({ page }) => {
  await page.getByText("Modal & Overlays").click();
  await page.getByText("Toastr").click();

  await page.getByRole("checkbox", { name: "Hide on click" }).uncheck({ force: true });
  await page
    .getByRole("checkbox", { name: "Prevent arising of duplicate toast" })
    .check({ force: true });
});
```

In this example:

- **Hide on click** is unchecked    
- **Prevent arising of duplicate toast** is checked

## Selecting All Checkboxes on the Page

If you want to check all checkboxes on the page:

1. Get a locator for all checkboxes.    
2. Convert the locator results into an array using `.all()`.
3. Loop through each checkbox.
4. Call `check()` on each checkbox.
5. Validate that each checkbox is checked.

```typescript
test("checkbox", async ({ page }) => {
  await page.getByText("Modal & Overlays").click();
  await page.getByText("Toastr").click();

  await page.getByRole("checkbox", { name: "Hide on click" }).uncheck({ force: true });
  await page
    .getByRole("checkbox", { name: "Prevent arising of duplicate toast" })
    .check({ force: true });

  const allBoxes = page.getByRole("checkbox");

  for (const box of await allBoxes.all()) {
    await box.check({ force: true });
    expect(await box.isChecked()).toBeTruthy();
  }
});
```

### Important Note

`page.getByRole("checkbox")` returns a **locator for all matching checkboxes**, not a normal JavaScript array.

To iterate over them, we use:

```typescript
await allBoxes.all()
```

This gives us an array of checkbox locators that we can loop through.

## Unchecking All Checkboxes on the Page

Similarly, we can uncheck all checkboxes and verify that each one is unchecked.

```typescript
test("checkbox", async ({ page }) => {
  await page.getByText("Modal & Overlays").click();
  await page.getByText("Toastr").click();

  await page.getByRole("checkbox", { name: "Hide on click" }).uncheck({ force: true });
  await page
    .getByRole("checkbox", { name: "Prevent arising of duplicate toast" })
    .check({ force: true });

  const allBoxes = page.getByRole("checkbox");

  for (const box of await allBoxes.all()) {
    await box.uncheck({ force: true });
    expect(await box.isChecked()).toBeFalsy();
  }
});
```

## Summary

### 1. Prefer `check()` and `uncheck()` for checkboxes

These methods are better than `click()` when your goal is to explicitly set the checkbox state.

- `check()` ensures the checkbox is checked.    
- `uncheck()` ensures the checkbox is unchecked.

---

### 2. `click()` vs `check()` / `uncheck()`

#### `click()`

- Performs a normal click
- Does **not** care about the current checkbox state
- Can toggle the checkbox on or off depending on its current state

#### `check()`

- Checks the checkbox only if it is currently unchecked
- Does nothing if it is already checked

#### `uncheck()`

- Unchecks the checkbox only if it is currently checked
- Does nothing if it is already unchecked

---

### 3. Hidden checkboxes may require `{ force: true }`

If a checkbox is visually hidden, normal interaction may fail. In that case, use:

```typescript
{ force: true }
```

---

### 4. To work with all checkboxes on a page

- Use a locator for all checkboxes:

```typescript
   const allBoxes = page.getByRole("checkbox");
```

- Convert them to an array using `.all()`
- Loop through them with JavaScript
- Perform `check()` or `uncheck()`
- Validate using `isChecked()`

---

### 5. Validation

To validate the checkbox state:

```typescript
expect(await box.isChecked()).toBeTruthy();
expect(await box.isChecked()).toBeFalsy();
```

- `toBeTruthy()` → checkbox is checked    
- `toBeFalsy()` → checkbox is unchecked

---

## Key Takeaway

For checkboxes in Playwright:

- Use `check()` to check them    
- Use `uncheck()` to uncheck them
- Use `click()` only when you intentionally want a raw click behavior
- Use `isChecked()` to validate the state
- Use `.all()` + a loop when working with multiple checkboxes
