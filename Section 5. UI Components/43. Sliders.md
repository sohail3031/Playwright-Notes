In this lesson, we learn how to automate **sliders** in Playwright.

The slider in this example is a circular temperature gauge. We can drag it around to change the displayed temperature. Automating this type of UI can be done in **two ways**:

1. **Update the slider’s DOM attributes directly**
2. **Simulate real mouse movement**

---

## What makes sliders different?

Unlike regular input fields, sliders often do not use a simple `<input>` where we can type a value directly. Instead, they rely on:

- **mouse dragging**
- **SVG elements or custom UI controls**
- **position-based updates**
- **JavaScript events** that react to movement

So to automate them, we either:

- directly change the underlying coordinates of the slider element, or
- simulate how a user would actually drag the slider with the mouse

---

# Approach 1: Update the Slider Coordinates Directly

The first approach is to **update the HTML/SVG attributes** responsible for the slider position.

When inspecting the temperature slider, the draggable circle contains attributes like:

- `cx`
- `cy`

These represent the **X and Y coordinates** of the slider handle.

So technically, we can move the slider by changing these coordinates.

---

## Starter Code

```typescript
import { test, expect } from "@playwright/test";

test.beforeEach(async ({ page }) => {
  await page.goto("http://localhost:4200/");
});
```

---

## Step 1: Update the slider position using `evaluate()`

```typescript
// the below code will work but only partially:
// the slider handle will move, but the displayed temperature value will not update
test("Sliders", async ({ page }) => {
  // update attribute
  const tempGauge = page.locator(
    '[tabtitle="Temperature"] ngx-temperature-dragger circle'
  );

  await tempGauge.evaluate((node) => {
    node.setAttribute("cx", "232.680");
    node.setAttribute("cy", "232.680");
  });
});
```

---

## Why this is only a partial solution

This code **moves the slider handle visually**, because we changed its coordinates in the DOM.

However, the **temperature value itself does not update**.

That happens because changing the DOM attributes alone does **not trigger the UI event** that normally fires when a real user interacts with the slider.

So the UI looks changed, but the application logic has not reacted yet.

---

## Step 2: Trigger the slider event by clicking it

To make the UI react properly, we can trigger an action on the slider after changing its coordinates.

```typescript
test("Sliders", async ({ page }) => {
  // update attribute
  const tempGauge = page.locator(
    '[tabtitle="Temperature"] ngx-temperature-dragger circle'
  );

  await tempGauge.evaluate((node) => {
    node.setAttribute("cx", "232.680");
    node.setAttribute("cy", "232.680");
  });

  await tempGauge.click();
});
```

### Why this works

- `evaluate()` updates the slider position in the DOM    
- `click()` triggers the UI event
- the application reacts and updates the displayed value accordingly

So this approach is a **shortcut** when you already know the coordinates you want to set.

---

# Approach 2: Simulate Real Mouse Movement

The second approach is more realistic and closer to how a real user interacts with the slider.

Here, we will:

1. identify the slider area    
2. get its **bounding box**
3. move the mouse to a starting position
4. hold the mouse button down
5. drag the mouse to a new position
6. release the mouse button
7. validate the new slider value

---

# Understanding `boundingBox()`

To drag the slider, we first need the coordinates of the slider area.

We use:

```typescript
await tempBox.boundingBox();
```

This returns the position and size of the slider element:

- `x` → left position    
- `y` → top position
- `width`
- `height`

You can think of it like a rectangular box around the slider.

---

## Why scroll the slider into view first?

Before moving the mouse, the slider should be visible in the browser viewport.

So we use:

```typescript
await tempBox.scrollIntoViewIfNeeded();
```

This ensures Playwright can interact with the slider reliably.

---

# Example 1: Drag the slider to the lower-right area

```typescript
test("Sliders", async ({ page }) => {
  // mouse movement
  const tempBox = page.locator('[tabtitle="Temperature"] ngx-temperature-dragger');

  await tempBox.scrollIntoViewIfNeeded();

  const box = await tempBox.boundingBox();

  if (!box) {
    return;
  }

  const x = box.x + box.width / 2;
  const y = box.y + box.height / 2;

  await page.mouse.move(x, y);
  await page.mouse.down();
  await page.mouse.move(x + 100, y);
  await page.mouse.move(x + 100, y + 100);
  await page.mouse.up();
});
```

---

## Breakdown of the mouse logic

### 1. Get the slider box

```typescript
const box = await tempBox.boundingBox();
```

This gives us the slider’s rectangular area.

---

### 2. Start from the center of the slider

```typescript
const x = box.x + box.width / 2;
const y = box.y + box.height / 2;
```

Instead of starting from the top-left corner, we calculate the **center** of the slider box.

That gives us a better starting point for dragging.

---

### 3. Move the mouse to the start point

```typescript
await page.mouse.move(x, y);
```

This places the mouse cursor at the center of the slider.

---

### 4. Hold the mouse button down

```typescript
await page.mouse.down();
```

This simulates pressing the left mouse button, which starts the drag.

---

### 5. Drag to the right

```typescript
await page.mouse.move(x + 100, y);
```

This moves the mouse **100 pixels to the right**.

---

### 6. Drag downward

```typescript
await page.mouse.move(x + 100, y + 100);
```

This continues dragging **100 pixels down**.

---

### 7. Release the mouse button

```typescript
await page.mouse.up();
```

This ends the drag action.

---

# Example 2: Drag the slider toward the lower-left area

If we want to move the slider in the opposite direction, we can move left instead of right.

```typescript
test("Sliders", async ({ page }) => {
  // mouse movement
  const tempBox = page.locator('[tabtitle="Temperature"] ngx-temperature-dragger');

  await tempBox.scrollIntoViewIfNeeded();

  const box = await tempBox.boundingBox();

  if (!box) {
    return;
  }

  const x = box.x + box.width / 2;
  const y = box.y + box.height / 2;

  await page.mouse.move(x, y);
  await page.mouse.down();
  await page.mouse.move(x - 100, y);
  await page.mouse.move(x - 100, y + 100);
  await page.mouse.up();
});
```

Here the key difference is:

```typescript
await page.mouse.move(x - 100, y);
await page.mouse.move(x - 100, y + 100);
```

This moves the slider toward the **left side** instead of the right side.

---

# Final Example: Drag the slider and validate the value

```typescript
test("Sliders", async ({ page }) => {
  // mouse movement
  const tempBox = page.locator('[tabtitle="Temperature"] ngx-temperature-dragger');

  await tempBox.scrollIntoViewIfNeeded();

  const box = await tempBox.boundingBox();

  if (!box) {
    return;
  }

  const x = box.x + box.width / 2;
  const y = box.y + box.height / 2;

  await page.mouse.move(x, y);
  await page.mouse.down();
  await page.mouse.move(x - 100, y);
  await page.mouse.move(x - 100, y + 100);
  await page.mouse.up();

  await expect(tempBox).toContainText("30");
});
```

---

# Why the `if (!box)` check is needed

`boundingBox()` can return `null` if Playwright cannot determine the element’s visible box.

So this guard prevents runtime errors:

```typescript
if (!box) {
  return;
}
```

# Summary

## 1. Sliders can be automated in two ways

### Option A: Update slider coordinates directly

Use `evaluate()` and change attributes like `cx` and `cy`:

```typescript
await tempGauge.evaluate((node) => {
  node.setAttribute("cx", "232.680");
  node.setAttribute("cy", "232.680");
});
```

This moves the handle, but you may still need to trigger an event:

```typescript
await tempGauge.click();
```

---

## 2. Simulating mouse movement is more realistic

You can automate the slider just like a real user by:

- locating the slider container    
- getting its bounding box
- moving the mouse
- holding the mouse button down
- dragging to the desired position
- releasing the mouse button

---

## 3. `boundingBox()` gives you the slider’s position and size

It provides:

- `x`    
- `y`
- `width`
- `height`

This allows you to calculate exactly where to move the mouse.

---

## 4. Starting from the center is usually easier

Instead of using the top-left corner of the slider box, we calculate the center:

```typescript
const x = box.x + box.width / 2;
const y = box.y + box.height / 2;
```

This gives a better starting point for dragging.

---

## 5. Always validate the final slider value

After moving the slider, verify the result with an assertion:

```typescript
await expect(tempBox).toContainText("30");
```

---

# Key Takeaway

When automating sliders in Playwright:

- **direct DOM updates** are useful as a shortcut    
- but **mouse movement** is closer to real user behavior
- use `boundingBox()` to calculate drag positions
- call `scrollIntoViewIfNeeded()` before dragging
- always assert the final displayed value after the slider changes
