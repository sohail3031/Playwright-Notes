In this lesson, we learn how to work with **dialog boxes in Playwright**.

There are **two types of dialog boxes** you may encounter in web applications:

1. **Web dialog boxes** – dialogs that are part of the DOM    
2. **Browser dialog boxes** – native browser popups such as `alert`, `confirm`, or `prompt`

---

## 1. Web Dialog Boxes vs Browser Dialog Boxes

### Web Dialog Boxes

These are regular UI components rendered inside the webpage.  
Examples:

- modal windows    
- popup forms
- custom confirmation dialogs

Since they are part of the DOM, they can be automated like any other web element:

- inspect the element
- find a locator
- click buttons or read text

---

### Browser Dialog Boxes

These are browser-level dialogs such as:

- `alert()`    
- `confirm()`
- `prompt()`

These **do not belong to the DOM**, so you cannot inspect them with DevTools like normal page elements.

To automate them, Playwright provides **dialog event handling**.

---

## Scenario in This Lesson

We go to:

- **Tables & Data**    
- **Smart Table**

Then we delete a row from the table by clicking the **trash icon**.  
When the delete icon is clicked, a **browser confirmation dialog** appears asking:

> **"Are you sure you want to delete?"**

By default, Playwright automatically dismisses browser dialogs unless we explicitly handle them.

---

## Test Setup

```typescript
import { test, expect } from "@playwright/test";

test.beforeEach(async ({ page }) => {
  await page.goto("http://localhost:4200/");
});
```

---

## Step 1: Click the Delete Icon

We first locate the row containing the email **`mdo@gmail.com`**, then locate the trash icon inside that row and click it.

```typescript
test("Dialog Boxes", async ({ page }) => {
  await page.getByText("Tables & Data").click();
  await page.getByText("Smart Table").click();

  await page
    .getByRole("table")
    .locator("tr", { hasText: "mdo@gmail.com" })
    .locator(".nb-trash")
    .click();
});
```

### What happens here?

- The browser confirmation dialog appears    
- But Playwright automatically dismisses it
- As a result, the row is **not deleted**

---

## Step 2: Handle the Browser Dialog with `page.on("dialog")`

To control browser dialogs, we register a dialog listener using `page.on("dialog", ...)`.

Inside the listener:

1. we can validate the dialog message
2. then accept the dialog using `dialog.accept()`

```typescript
test("Dialog Boxes", async ({ page }) => {
  await page.getByText("Tables & Data").click();
  await page.getByText("Smart Table").click();

  page.on("dialog", (dialog) => {
    expect(dialog.message()).toEqual("Are you sure you want to delete?");
    dialog.accept();
  });

  await page
    .getByRole("table")
    .locator("tr", { hasText: "mdo@gmail.com" })
    .locator(".nb-trash")
    .click();
});
```

---

## Step 3: Validate That the Row Was Deleted

After accepting the dialog, we can assert that the deleted row no longer exists.

```typescript
test("Dialog Boxes", async ({ page }) => {
  await page.getByText("Tables & Data").click();
  await page.getByText("Smart Table").click();

  page.on("dialog", (dialog) => {
    expect(dialog.message()).toEqual("Are you sure you want to delete?");
    dialog.accept();
  });

  await page
    .getByRole("table")
    .locator("tr", { hasText: "mdo@gmail.com" })
    .locator(".nb-trash")
    .click();

  await expect(page.locator("table tr").first()).not.toHaveText("mdo@gmail.com");
});
```

### 1. Navigate to the Smart Table page

```typescript
await page.getByText("Tables & Data").click();

```

### 2. Listen for the browser dialog

```typescript

```

### 3. Click the delete icon inside the matching row

```typescript
await page

  .locator(".nb-trash")
```


---

## Important Concept: Why the Row Locator Is Built This Way

The trash icon exists in **every row**, so locating `.nb-trash` alone is not enough.

Instead, we:

1. locate the **table**    
2. locate the specific **row** using unique row text such as the email
3. find the **trash icon inside that row**

This makes the selector precise and avoids clicking the wrong delete icon.

## Browser Dialog Methods in Playwright

Inside the dialog listener, you can use:

### Accept the dialog

```typescript
dialog.accept();
```

### Dismiss the dialog

```typescript
dialog.dismiss();
```

### Read the dialog message

```typescript
dialog.message();
```

---

## Summary

## 1. Two types of dialogs

### Web dialog boxes

- part of the DOM    
- inspect and automate like normal elements

### Browser dialog boxes

- not part of the DOM
- must be handled using Playwright dialog events

---

## 2. Playwright automatically dismisses browser dialogs

If you do nothing, Playwright usually dismisses native dialogs automatically.

That is why the delete operation does not happen unless you explicitly handle the dialog.

---

## 3. Use `page.on("dialog")` to handle browser popups

```typescript
page.on("dialog", (dialog) => {
  dialog.accept();
});
```

---

## 4. Validate the dialog message if needed

```typescript
expect(dialog.message()).toEqual("Are you sure you want to delete?");
```

---

## 5. Use a precise row locator before clicking delete

Instead of clicking the first trash icon you find, first locate the row with unique text, then find the icon inside it.

---

## Key Takeaway

When automating **browser dialog boxes** in Playwright:

- listen for the dialog using `page.on("dialog")`    
- validate the message if needed
- call `dialog.accept()` to confirm
- call `dialog.dismiss()` to cancel

For **web dialogs**, treat them like normal page elements because they are part of the DOM.
