In this lesson, we’ll learn how to automate **drag-and-drop interactions** and work with **iframes** in Playwright.

We’ll use a different test application for this lesson: the **GlobalSQA Drag & Drop demo site**. This application lets us drag photo items from one area to another. It also places the draggable elements inside an **iframe**, which means we must switch into that iframe before interacting with the elements.

In this lesson, we will cover:

- how to locate elements inside an iframe    
- how to perform drag and drop using Playwright’s built-in `dragTo()` method
- how to perform drag and drop manually using mouse actions for finer control
- how to validate that the dropped elements appear in the target area

---

## Base Setup

```typescript
import { test, expect } from "@playwright/test";

test.beforeEach(async ({ page }) => {
  await page.goto("https://www.globalsqa.com/demo-site/draganddrop/");
});
```

---

## Problem: Element Is Inside an iFrame

Let’s say we want to click on the image labeled **High Tatras 2**.

A first attempt might look like this:

```typescript
test("Drag & Drop with iFrames", async ({ page }) => {
  await page.locator("li", { hasText: "High Tatras 2" }).click();
});
```

### Why this fails

This code fails because the image elements are **not directly on the main page**. They are located inside an **iframe**.

An **iframe** is essentially an HTML page embedded inside another HTML page. Playwright cannot directly access elements inside an iframe unless you first switch to that iframe using `frameLocator()`.

---

## Step 1: Switch to the iFrame

To work with elements inside the iframe, first create a frame locator:

```typescript
test("Drag & Drop with iFrames", async ({ page }) => {
  const frame = page.frameLocator('[rel-title="Photo Manager"] iframe');

  await frame.locator("li", { hasText: "High Tatras 2" }).click();
});
```

### What’s happening here?

- `page.frameLocator(...)` targets the iframe    
- `frame.locator(...)` finds elements **inside** that iframe
- Once inside the frame, you can interact with elements normally

---

## Step 2: Perform Drag and Drop with `dragTo()`

Now that we can access the iframe content, we can drag **High Tatras 2** into the trash area.

```typescript
test("Drag & Drop with iFrames", async ({ page }) => {
  const frame = page.frameLocator('[rel-title="Photo Manager"] iframe');

  await frame.locator("li", { hasText: "High Tatras 2" }).dragTo(frame.locator("#trash"));
});
```

### How this works

- `frame.locator("li", { hasText: "High Tatras 2" })` locates the draggable image item    
- `frame.locator("#trash")` locates the drop target
- `dragTo()` performs the drag-and-drop action from source to destination

This is the simplest and cleanest way to automate drag-and-drop in Playwright.

---

## Step 3: Perform Drag and Drop with Manual Mouse Control

Sometimes you may want more precise control over the drag-and-drop behavior. In that case, you can simulate it manually using mouse actions.

Here’s an example where we drag **High Tatras 4** manually:

```typescript
test("Drag & Drop with iFrames", async ({ page }) => {
  const frame = page.frameLocator('[rel-title="Photo Manager"] iframe');

  await frame.locator("li", { hasText: "High Tatras 2" }).dragTo(frame.locator("#trash"));

  // More precise control
  await frame.locator("li", { hasText: "High Tatras 4" }).hover();
  await page.mouse.down();
  await frame.locator("#trash").hover();
  await page.mouse.up();
});
```

### What happens here?

1. Hover over **High Tatras 4**    
2. Press and hold the mouse button with `page.mouse.down()`
3. Move the mouse to the drop area using `hover()`
4. Release the mouse button with `page.mouse.up()`

This simulates a real drag-and-drop interaction more closely.

---

## Step 4: Validate the Dropped Items

After dragging the items, we want to verify that they actually appear inside the drop area.

```typescript
test("Drag & Drop with iFrames", async ({ page }) => {
  const frame = page.frameLocator('[rel-title="Photo Manager"] iframe');

  await frame.locator("li", { hasText: "High Tatras 2" }).dragTo(frame.locator("#trash"));

  // More precise control
  await frame.locator("li", { hasText: "High Tatras 4" }).hover();
  await page.mouse.down();
  await frame.locator("#trash").hover();
  await page.mouse.up();

  await expect(frame.locator("#trash li h5")).toHaveText([
    "High Tatras 2",
    "High Tatras 4",
  ]);
});
```

### Assertion explanation

The locator:

```typescript
frame.locator("#trash li h5")
```

selects the titles of the images that were dropped into the trash area.

Then we assert that the list contains:

- `High Tatras 2`    
- `High Tatras 4`

in that exact order.

# Key Takeaways

## Working with iFrames

- If a locator does not work even though it looks correct, check whether the element is inside an iframe.    
- Use `page.frameLocator()` to switch into the iframe.
- Once inside, use `frame.locator()` to interact with elements normally.

## Drag and Drop

Playwright offers two common ways to automate drag-and-drop:

### 1. `dragTo()`

Use this when you want a simple built-in drag-and-drop solution.

```typescript
await source.dragTo(target);
```

### 2. Mouse actions

Use this when you want finer control over the drag-and-drop flow.

```typescript
await source.hover();
await page.mouse.down();
await target.hover();
await page.mouse.up();
```

## Assertions

To validate that items were dropped successfully, locate the elements inside the destination container and use `toHaveText()` to verify the expected values.
