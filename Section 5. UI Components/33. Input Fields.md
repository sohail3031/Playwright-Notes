## Overview

Input fields are among the most commonly used UI elements in web automation. Playwright provides multiple methods to:

- Enter text into input fields
- Clear existing values
- Simulate real keyboard typing
- Add delays between keystrokes
- Validate input values using assertions

---

# Test Setup

```typescript
import { test, expect } from "@playwright/test";

test.beforeEach(async ({ page }) => {
  await page.goto("http://localhost:4200/");
});

test.describe("Form Layout Page", () => {
  test.beforeEach(async ({ page }) => {
    await page.getByText("Forms").click();
    await page.getByText("Form Layouts").click();
  });
});
```

### What happens here?

Before every test:

1. Open the application
2. Navigate to:
   - Forms
   - Form Layouts

This ensures every test starts from the same location.

```typescript
const emailInput = page

```

### Explanation

#### Parent Locator

```typescript
page.locator("nb-card", { hasText: "Using the Grid" })
```

```text
Using the Grid
```

#### Child Locator

```typescript
.getByRole("textbox", { name: "Email" })
```

## How fill() Works



# Incorrect Chaining

## Code

```typescript
await emailInput.fill("test@test.com").clear();
```

---

## Why It Doesn't Work

`fill()` returns a Promise.

After:

```typescript
await emailInput.fill(...)
```

there is no locator available anymore.

You cannot chain:

```typescript
.fill().clear()
```

---

## Correct Approach

```typescript
await emailInput.fill("test@test.com");
await emailInput.clear();
```

# Typing with pressSequentially()

## Code

```typescript
test("Input Fields", async ({ page }) => {

  const emailInput = page
    .locator("nb-card", { hasText: "Using the Grid" })
    .getByRole("textbox", { name: "Email" });

  await emailInput.fill("test@test.com");
  await emailInput.clear();

  await emailInput.pressSequentially("test2@gmail.com");
});
```

---

## Result

Playwright simulates typing:

```text
t
te
tes
test
...
test2@gmail.com
```

---

## Difference Between fill() and pressSequentially()

### fill()

```typescript
await emailInput.fill("test@test.com");
```

Fast.

Instantly sets value.

---

### pressSequentially()

```typescript
await emailInput.pressSequentially("test@test.com");
```

Simulates real keyboard typing.

Triggers key events such as:

- keydown
- keypress
- keyup

Useful when applications rely on keyboard events.

## Explanation


# Generic Assertion Using inputValue()

## Code

```typescript
test("Input Fields", async ({ page }) => {

  const emailInput = page
    .locator("nb-card", { hasText: "Using the Grid" })
    .getByRole("textbox", { name: "Email" });

  await emailInput.fill("test@test.com");
  await emailInput.clear();

  await emailInput.pressSequentially(
    "test2@gmail.com",
    { delay: 500 }
  );

  const inputValue = await emailInput.inputValue();

  expect(inputValue).toEqual("test2@gmail.com");
});
```

---

## Why inputValue()?

Text inside an input field is not regular DOM text.

This will NOT work:

```typescript
await emailInput.textContent();
```

because input fields store data in:

```html
<input value="test2@gmail.com">
```

not as visible text nodes.

---

## Correct Method

```typescript
await emailInput.inputValue();
```

Returns:

```text
test2@gmail.com
```

---

## Assertion Flow

### Step 1

Get value

```typescript
const inputValue = await emailInput.inputValue();
```

---

### Step 2

Compare

```typescript
expect(inputValue).toEqual(
  "test2@gmail.com"
);
```

---

# Locator Assertion Using toHaveValue()

## Code

```typescript
test("Input Fields", async ({ page }) => {

  const emailInput = page
    .locator("nb-card", { hasText: "Using the Grid" })
    .getByRole("textbox", { name: "Email" });

  await emailInput.fill("test@test.com");
  await emailInput.clear();

  await emailInput.pressSequentially(
    "test2@gmail.com",
    { delay: 500 }
  );

  const inputValue = await emailInput.inputValue();

  expect(inputValue).toEqual("test2@gmail.com");

  await expect(emailInput)
    .toHaveValue("test2@gmail.com");
});
```

---

# Why Locator Assertions Are Better

### Generic Assertion

```typescript
const value = await emailInput.inputValue();

expect(value).toEqual(
  "test2@gmail.com"
);
```

You manually:

1. Extract value
2. Store value
3. Compare value

---

### Locator Assertion

```typescript
await expect(emailInput)
  .toHaveValue("test2@gmail.com");
```

Playwright automatically:

1. Waits for element
2. Reads value
3. Retries if needed
4. Performs assertion

---

# Comparison

| Feature | Generic Assertion | Locator Assertion |
|----------|----------|----------|
| Requires extracting value | ✅ Yes | ❌ No |
| Auto waiting | ❌ No | ✅ Yes |
| Retries automatically | ❌ No | ✅ Yes |
| Cleaner syntax | ❌ Less | ✅ More |
| Recommended | ⚠️ Sometimes | ✅ Yes |

# Common Input Field Methods

| Method | Purpose |
|----------|----------|
| `fill()` | Enter text instantly |
| `clear()` | Remove text |
| `pressSequentially()` | Simulate typing |
| `inputValue()` | Read current value |
| `toHaveValue()` | Assert value |

---

# Best Practices

### Prefer `fill()` for normal text entry

```typescript
await emailInput.fill("test@test.com");
```

Fast and reliable.

---

### Use `pressSequentially()` only when needed

```typescript
await emailInput.pressSequentially(
  "test@test.com"
);
```

Useful when testing:

- Autocomplete
- Search suggestions
- Keyboard event handlers

---

### Prefer Locator Assertions

Instead of:

```typescript
const value = await emailInput.inputValue();

expect(value).toEqual(
  "test@test.com"
);
```

Use:

```typescript
await expect(emailInput)
  .toHaveValue("test@test.com");
```

Cleaner and more reliable.

## Key Takeaway

For input fields:

- Use `fill()` to enter text
- Use `clear()` to remove text
- Use `pressSequentially()` to simulate real typing
- Use `inputValue()` to extract values
- Prefer `toHaveValue()` for assertions because it includes Playwright's built-in auto-waiting and retry mechanisms
