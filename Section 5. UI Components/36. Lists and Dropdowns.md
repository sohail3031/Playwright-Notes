In this lesson, we learn how to work with **lists and dropdowns in Playwright**.

At the top of the application, there is a dropdown menu. When a user selects an item from it, the **application theme color changes**.

---

## Important Concept: Dropdowns Are Not Always Inside the DOM Structure You Expect

Visually, the dropdown appears inside the header. However, in the DOM:

- The **trigger element (select button)** is inside the header    
- The **dropdown options list** is rendered separately in an overlay container

This means:

> The dropdown options may NOT be located inside the same parent element as the button.

They are often rendered in a **separate overlay container** by UI frameworks.

---

## Initial Setup

```typescript
import { test, expect } from "@playwright/test";

test.beforeEach(async ({ page }) => {
  await page.goto("http://localhost:4200/");
});
```

## Opening the Dropdown

We first locate and click the dropdown.

```typescript
test("Lists and Dropdowns", async ({ page }) => {
  const dropDownMenu = page.locator("ngx-header nb-select");

  await dropDownMenu.click();
});
```

---

## Understanding List Roles in Playwright

Playwright supports semantic roles:

```typescript
page.getByRole("list");     // when UL tag is used
page.getByRole("listitem"); // when LI tag is used
```

However, in this application:

- The dropdown does NOT use `<ul>` or `<li>`    
- Instead, it uses custom elements like `nb-option`

So we cannot rely on `listitem` role here.

---

## Getting All Dropdown Options

We locate all dropdown options using a CSS locator:

```typescript
test("Lists and Dropdowns", async ({ page }) => {
  const dropDownMenu = page.locator("ngx-header nb-select");
  await dropDownMenu.click();

  const optionList = page.locator("nb-option-list nb-option");

  await expect(optionList).toHaveText([
    "Light",
    "Dark",
    "Cosmic",
    "Corporate",
  ]);
});
```

### Assertion Failure Example

If expected values are incorrect:

```typescript
await expect(optionList).toHaveText([
  "Light",
  "Dark",
  "Cosmic",
  "Corporate2",
]);
```

The test fails because:

- Expected and actual list values do not match

## Selecting an Item from Dropdown

We use `.filter()` to select a specific option.

```typescript
test("Lists and Dropdowns", async ({ page }) => {
  const dropDownMenu = page.locator("ngx-header nb-select");
  await dropDownMenu.click();

  const optionList = page.locator("nb-option-list nb-option");

  await expect(optionList).toHaveText([
    "Light",
    "Dark",
    "Cosmic",
    "Corporate",
  ]);

  await optionList.filter({ hasText: "Cosmic" }).click();
});
```

---

## Validating Background Color Change

After selecting an option, the header background color changes.

We validate it using CSS property assertion.

```typescript
test("Lists and Dropdowns", async ({ page }) => {
  const dropDownMenu = page.locator("ngx-header nb-select");
  await dropDownMenu.click();

  const optionList = page.locator("nb-option-list nb-option");

  await optionList.filter({ hasText: "Cosmic" }).click();

  const header = page.locator("nb-layout-header");

  await expect(header).toHaveCSS(
    "background-color",
    "rgb(50, 50, 89)"
  );
});
```

## Handling Multiple Theme Selections Using Loop

We can automate testing of all dropdown options using a loop.

### Step 1: Define expected colors

```typescript
const colors = {
  Light: "rgb(255, 255, 255)",
  Dark: "rgb(34, 43, 69)",
  Cosmic: "rgb(50, 50, 89)",
  Corporate: "rgb(255, 255, 255)",
};
```

---

### Step 2: Loop through all options

```typescript
test("Lists and Dropdowns", async ({ page }) => {
  const dropDownMenu = page.locator("ngx-header nb-select");
  const optionList = page.locator("nb-option-list nb-option");
  const header = page.locator("nb-layout-header");

  await dropDownMenu.click();

  await expect(optionList).toHaveText([
    "Light",
    "Dark",
    "Cosmic",
    "Corporate",
  ]);

  for (const color in colors) {
    await optionList.filter({ hasText: color }).click();
    await expect(header).toHaveCSS("background-color", colors[color]);

    await dropDownMenu.click();
  }
});
```

---

## Handling the Last Iteration Properly

We avoid reopening the dropdown after the final selection:

```typescript
test("Lists and Dropdowns", async ({ page }) => {
  const dropDownMenu = page.locator("ngx-header nb-select");
  const optionList = page.locator("nb-option-list nb-option");
  const header = page.locator("nb-layout-header");

  const colors = {
    Light: "rgb(255, 255, 255)",
    Dark: "rgb(34, 43, 69)",
    Cosmic: "rgb(50, 50, 89)",
    Corporate: "rgb(255, 255, 255)",
  };

  await dropDownMenu.click();

  for (const color in colors) {
    await optionList.filter({ hasText: color }).click();
    await expect(header).toHaveCSS("background-color", colors[color]);

    if (color !== "Corporate") {
      await dropDownMenu.click();
    }
  }
});
```

## Summary

### 1. Dropdown Structure Insight

- Trigger button is in DOM    
- Options are rendered in a separate overlay container

---

### 2. Locating Options

- Use `locator("nb-option")` for custom dropdowns    
- Use `.filter({ hasText })` for selection

---

### 3. Assertions

- Use `toHaveText()` to validate dropdown options    
- Use `toHaveCSS()` to validate UI changes (like background color)

---

### 4. Looping Through Options

- Use `for...in` for objects    
- Store expected values in a map (object)
- Assert UI changes per iteration

---

## Key Takeaway

- Dropdown UI elements may not exist inside expected DOM hierarchy    
- Playwright locators are powerful enough to handle overlay-based components
- Always validate both:
    - UI content (`toHaveText`)
    - UI behavior (`toHaveCSS`)
