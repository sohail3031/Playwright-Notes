In this lesson, we learn how to work with **web tables in Playwright**.

Web tables can be tricky to automate because of their structure. A table is made up of rows and columns, and selecting the correct row or cell often requires careful locator strategies.

---

## Understanding Table Structure

A typical HTML table looks like this:

- `<table>` → the main table element    
- `<thead>` → table header
- `<tbody>` → table body
- `<tr>` → table row
- `<td>` → table cell / column value

So the hierarchy is usually:

```text
table
└── tbody
    ├── tr
    │   ├── td
    │   ├── td
    │   └── td
    ├── tr
    └── tr
```

### Why tables are tricky

Some common challenges are:

- locating a **specific row**
- locating a **specific cell**
- identifying a row based on a value in a **specific column**
- updating values when a row switches into **edit mode**
- validating data after the update

---

# Scenario 1: Update a User's Age by Finding the Row Using Email

We will:

1. open the **Smart Table** page    
2. locate a row using a unique email
3. click the **edit** icon in that row
4. update the **Age**
5. save the changes

---

## Test Setup

```typescript
import { test, expect } from "@playwright/test";

test.beforeEach(async ({ page }) => {
  await page.goto("http://localhost:4200/");
});
```

---

## Step 1: Navigate to the Smart Table page and locate a row by email

```typescript
test("Web Tables (Part 1)", async ({ page }) => {
  await page.getByText("Tables & Data").click();
  await page.getByText("Smart Table").click();

  const targetRow = page.getByRole("row", {
    name: "twitter@outlook.com",
  });

  await targetRow.locator(".nb-edit").click();
});
```

### Why this works

The email `twitter@outlook.com` is unique for that row, so we can use it to locate the entire row using:

```typescript
page.getByRole("row", { name: "twitter@outlook.com" })
```

Then, once we have the row, we find the edit icon inside it:

```typescript
targetRow.locator(".nb-edit").click();
```

---

## Step 2: Update the Age field

Once the row enters **edit mode**, the plain text values become **input fields**.

That means the original row text locator may no longer be enough for editing specific cells, because the cell value is no longer plain text in the DOM — it becomes an input value.

To update the age, we locate the input using its placeholder.

```typescript
test("Web Tables (Part 1)", async ({ page }) => {
  await page.getByText("Tables & Data").click();
  await page.getByText("Smart Table").click();

  const targetRow = page.getByRole("row", {
    name: "twitter@outlook.com",
  });

  await targetRow.locator(".nb-edit").click();
  await page.locator("input-editor").getByPlaceholder("Age").clear();
  await page.locator("input-editor").getByPlaceholder("Age").fill("35");
  await page.locator(".nb-checkmark").click();
});
```

### Important Note

After clicking **Edit**, the cell content changes from visible text to an input field.  
That is why we use:

```typescript
page.locator("input-editor").getByPlaceholder("Age")
```

instead of trying to directly manipulate the old row text.

# Scenario 2: Locate a Row by Value in a Specific Column

Now we make the scenario more complex.

Suppose we want to:

1. go to **page 2** of the table    
2. locate the row with **ID = 11**
3. narrow down the correct row using the **ID column**
4. edit the first row after locating the correct target row
5. update the **email**
6. validate the updated email

---

## Step 1: Navigate to page 2

```typescript
test("Web Tables (Part 1)", async ({ page }) => {
  await page.getByText("Tables & Data").click();
  await page.getByText("Smart Table").click();

  // 1. Get the row by any text in this row
  const targetRow = page.getByRole("row", { name: "twitter@outlook.com" });

  await targetRow.locator(".nb-edit").click();
  await page.locator("input-editor").getByPlaceholder("Age").clear();
  await page.locator("input-editor").getByPlaceholder("Age").fill("35");
  await page.locator(".nb-checkmark").click();

  // 2. Get the row based on the value in a specific column
  await page.locator(".ng2-smart-pagination-nav").getByText("2").click();
});
```

---

## Why `page.getByRole("row", { name: "11" })` is not enough

A naive approach would be:

```typescript
const targetRowById = page.getByRole("row", { name: "11" });
```

This is not reliable because the text `11` may appear in multiple places in the same row set, such as:

- the **ID column**    
- the **Age column**

So Playwright may match more than one row.

---

## Step 2: Filter the row by a specific column

To solve this, we:

1. first find all rows containing `11`    
2. then **filter** them by checking whether the value `11` exists specifically in the **ID column**

```typescript
const targetRowById = page
  .getByRole("row", { name: "11" })
  .filter({ has: page.locator("td").nth(1).getByText("11") });
```

This ensures that we target the row whose **ID column** contains `11`.

### Understanding the column index

In this table:

- column `0` → Actions
- column `1` → ID
- column `2` → First Name
- column `3` → Last Name
- column `4` → Username
- column `5` → E-mail
- column `6` → Age

So to target **ID = 11**, we filter using:

```typescript
page.locator("td").nth(1).getByText("11")
```

because the **ID column is index 1**.

---

## Step 3: Click the matched row and work with the first row in the table body

In the updated version of the code, after locating the correct row with `ID = 11`, we click it and then define the **first row in the table body**:

```typescript
const firstRow = page.locator("tbody").locator("tr").first();
```

Then we edit that row’s email field.

```typescript
test("Web Tables (Part 1)", async ({ page }) => {
  await page.getByText("Tables & Data").click();
  await page.getByText("Smart Table").click();

  // 1 get the row by any text in this row
  const targetRow = page.getByRole("row", { name: "twitter@outlook.com" });

  await targetRow.locator(".nb-edit").click();
  await page.locator("input-editor").getByPlaceholder("Age").clear();
  await page.locator("input-editor").getByPlaceholder("Age").fill("35");
  await page.locator(".nb-checkmark").click();

  // 2 get the row based on the value in the specific column
  await page.locator(".ng2-smart-pagination-nav").getByText("2").click();

  const targetRowById = page
    .getByRole("row", { name: "11" })
    .filter({ has: page.locator("td").nth(1).getByText("11") });

  await targetRowById.click();

  const firstRow = page.locator("tbody").locator("tr").first();

  await firstRow.locator(".nb-edit").click();
  await firstRow.getByPlaceholder("E-mail").clear();
  await firstRow.getByPlaceholder("E-mail").fill("test@test.com");
});
```

### Why use `firstRow` here?

After locating and clicking the correct row, the code switches to working with:

```typescript
page.locator("tbody").locator("tr").first()
```

This means the email update is now performed on the **first row of the table body**, rather than directly on `targetRowById`.

---

## Step 4: Save the changes and validate the updated email

After updating the email, click the checkmark to save and then validate that the row contains the updated email text.

```typescript
test("Web Tables (Part 1)", async ({ page }) => {
  await page.getByText("Tables & Data").click();
  await page.getByText("Smart Table").click();

  // 1 get the row by any text in this row
  const targetRow = page.getByRole("row", { name: "twitter@outlook.com" });

  await targetRow.locator(".nb-edit").click();
  await page.locator("input-editor").getByPlaceholder("Age").clear();
  await page.locator("input-editor").getByPlaceholder("Age").fill("35");
  await page.locator(".nb-checkmark").click();

  // 2 get the row based on the value in the specific column
  await page.locator(".ng2-smart-pagination-nav").getByText("2").click();

  const targetRowById = page
    .getByRole("row", { name: "11" })
    .filter({ has: page.locator("td").nth(1).getByText("11") });

  await targetRowById.click();

  const firstRow = page.locator("tbody").locator("tr").first();

  await firstRow.locator(".nb-edit").click();
  await firstRow.getByPlaceholder("E-mail").clear();
  await firstRow.getByPlaceholder("E-mail").fill("test@test.com");
  await firstRow.locator(".nb-checkmark").click();

  await expect(firstRow).toContainText("test@test.com");
});
```

### Why `toContainText()` instead of targeting a specific column?

In the updated code, the assertion is:

```typescript
await expect(firstRow).toContainText("test@test.com");
```

This checks whether the **entire row** contains the updated email value, instead of targeting a specific `td` cell by index.

# Summary

## 1. Get a row by unique text in the row

If a row contains a unique value such as an email, you can locate it using:

```typescript
page.getByRole("row", { name: "twitter@outlook.com" })
```

---

## 2. Once the row is in edit mode, text may become input fields

When a row is edited:

- the visible text is replaced by inputs    
- the old text locator may no longer be sufficient for editing

In such cases, use other locators such as:

- `getByPlaceholder()`
- input-specific locators

---

## 3. If the row text is not unique, filter by a specific column

When the same value appears in multiple rows, use:

```typescript
.filter({ has: page.locator("td").nth(columnIndex).getByText("value") })
```

This helps narrow the match to the correct row based on the value in a specific column.

---

## 4. You can work with a specific row using `first()`

In the updated code, after locating the desired row, we use:

```typescript
const firstRow = page.locator("tbody").locator("tr").first();
```

This gives us the first row in the table body, and we perform the edit operation on that row.

---

## 5. Validate the updated row using `toContainText()`

Instead of targeting a specific column, the updated code validates the row like this:

```typescript
await expect(firstRow).toContainText("test@test.com");
```

This confirms that the updated email appears somewhere inside the row.

---

# Key Takeaway

When automating web tables in Playwright:

- use **unique row text** when possible    
- if the same value appears in multiple rows, **filter by a specific column**
- remember that table cells may turn into **input fields** during editing
- you can switch to a row-level locator such as `firstRow` when needed
- use either **cell-level assertions** or **row-level assertions** depending on your test strategy
