## Overview

Radio buttons allow users to select **one option from a group of options**.

Example:

```text
(●) Option 1
( ) Option 2
( ) Option 3
```

Only one radio button can be selected at a time.

In this lesson, you'll learn:

- How to locate radio buttons
- How to select radio buttons
- Why `force: true` is sometimes required
- Generic assertions for radio buttons
- Locator assertions for radio buttons
- How to validate selected and unselected states

---

# Understanding the HTML Structure

The radio buttons look something like:

```html
<label>
  <input type="radio" />
  <span>Option 1</span>
</label>
```

Key observations:

```html
<label>...</label>
```

The radio button is wrapped inside a label.

---

The actual clickable element is:

```html
<input type="radio" />
```

---

The visible text is:

```html
<span>Option 1</span>
```

---

Because the input is wrapped inside a label, Playwright can locate it using:

```typescript
getByLabel()
```

or

```typescript
getByRole()
```

---

# Creating a Parent Locator

```typescript
const usingTheGridForm = page.locator(
  "nb-card",
  { hasText: "Using the Grid" }
);
```

---

## Why?

Instead of searching the entire page:

```typescript
page.getByRole(...)
```

we first locate:

```text
Using the Grid
```

card.

Then all child locators become scoped to that form.

This makes tests:

- More reliable
- Easier to maintain
- Faster

# Selecting a Radio Button Using Label

## Code

```typescript
test("Radio Buttons", async ({ page }) => {

  const usingTheGridForm = page.locator(
    "nb-card",
    { hasText: "Using the Grid" }
  );

  await usingTheGridForm
    .getByLabel("Option 1")
    .check({ force: true });
});
```

---

## Explanation

### Step 1

Find form:

```typescript
page.locator(
  "nb-card",
  { hasText: "Using the Grid" }
)
```

---

### Step 2

Find radio button:

```typescript
getByLabel("Option 1")
```

Because:

```html
<label>Option 1</label>
```

exists.

---

### Step 3

Select it:

```typescript
.check()
```

---

# What Does check() Do?

```typescript
await locator.check();
```

Used for:

- Radio buttons
- Checkboxes

It ensures the element becomes selected.

---

# Why is force: true Needed?

## Code

```typescript
.check({ force: true });
```

---

### Reason

The actual radio input is:

```html
<input type="radio" class="visually-hidden">
```

meaning:

```text
Visible to browser
❌ No

Visible to Playwright
❌ No
```

---

Normally Playwright checks:

- Visible
- Enabled
- Stable
- Clickable

before interacting.

Since the input is hidden:

```typescript
await locator.check();
```

would fail.

---

Using:

```typescript
{ force: true }
```

tells Playwright:

```text
Ignore visibility checks.
Perform action anyway.
```

# Selecting Radio Button Using Role

## Recommended Method

```typescript
test("Radio Buttons", async ({ page }) => {

  const usingTheGridForm = page.locator(
    "nb-card",
    { hasText: "Using the Grid" }
  );

  await usingTheGridForm
    .getByRole("radio", { name: "Option 2" })
    .check({ force: true });
});
```

---

# Why This Is Better

Playwright best practice:

```typescript
getByRole()
```

because it mimics user interaction.

---

Radio button role:

```typescript
getByRole("radio")
```

---

Accessible name:

```typescript
{ name: "Option 2" }
```

---

This creates a very reliable locator.

# Verifying Radio Button State (Generic Assertion)

## Code

```typescript
test("Radio Buttons", async ({ page }) => {

  const usingTheGridForm = page.locator(
    "nb-card",
    { hasText: "Using the Grid" }
  );

  await usingTheGridForm
    .getByRole("radio", { name: "Option 1" })
    .check({ force: true });

  const radioStatus =
    await usingTheGridForm
      .getByRole("radio", { name: "Option 1" })
      .isChecked();

  expect(radioStatus).toBeTruthy();
});
```

---

# Understanding isChecked()

```typescript
await locator.isChecked();
```

Returns:

```typescript
true
```

if selected.

---

Returns:

```typescript
false
```

if not selected.

---

Example:

```typescript
const selected =
  await locator.isChecked();
```

Possible result:

```typescript
selected = true;
```

---

# Assertion

```typescript
expect(radioStatus).toBeTruthy();
```

Equivalent to:

```typescript
expect(radioStatus).toBe(true);
```

---

Meaning:

```text
Radio button must be selected.
```

---

# Locator Assertion

## Better Approach

```typescript
await expect(
  usingTheGridForm.getByRole(
    "radio",
    { name: "Option 1" }
  )
).toBeChecked();
```

---

# Why This Is Better

Generic assertion:

```typescript
const status =
  await locator.isChecked();

expect(status).toBeTruthy();
```

Requires:

1. Read value
2. Store value
3. Assert value

---

Locator assertion:

```typescript
await expect(locator)
  .toBeChecked();
```

Playwright automatically:

- Waits
- Retries
- Checks state

---

Cleaner and more reliable.

---

# Complete Example

```typescript
test("Radio Buttons", async ({ page }) => {

  const usingTheGridForm = page.locator(
    "nb-card",
    { hasText: "Using the Grid" }
  );

  await usingTheGridForm
    .getByRole("radio", { name: "Option 1" })
    .check({ force: true });

  const radioStatus =
    await usingTheGridForm
      .getByRole("radio", { name: "Option 1" })
      .isChecked();

  expect(radioStatus).toBeTruthy();

  await expect(
    usingTheGridForm.getByRole(
      "radio",
      { name: "Option 1" }
    )
  ).toBeChecked();
});
```

# Selecting Another Radio Button

## Code

```typescript
await usingTheGridForm
  .getByRole("radio", { name: "Option 2" })
  .check({ force: true });
```

---

Result:

```text
Option 1 → Unselected
Option 2 → Selected
```

because radio buttons are mutually exclusive.

# Verifying Option 1 Is Unchecked

```typescript
expect(
  await usingTheGridForm

    .isChecked()
).toBeFalsy();
```

---

# Understanding toBeFalsy()

```typescript
expect(value).toBeFalsy();
```

passes when value is:

```typescript
false
```

---

Meaning:

```text
Option 1 should NOT be selected.
```

---

# Full Example

```typescript
test("Radio Buttons", async ({ page }) => {

  const usingTheGridForm = page.locator(
    "nb-card",
    { hasText: "Using the Grid" }
  );

  await usingTheGridForm
    .getByRole("radio", { name: "Option 1" })
    .check({ force: true });

  expect(
    await usingTheGridForm
      .getByRole("radio", { name: "Option 1" })
      .isChecked()
  ).toBeTruthy();

  await expect(
    usingTheGridForm.getByRole(
      "radio",
      { name: "Option 1" }
    )
  ).toBeChecked();

  await usingTheGridForm
    .getByRole("radio", { name: "Option 2" })
    .check({ force: true });

  expect(
    await usingTheGridForm
      .getByRole("radio", { name: "Option 1" })
      .isChecked()
  ).toBeFalsy();
});
```

---

# Generic vs Locator Assertions

| Generic Assertion | Locator Assertion |
|----------|----------|
| `isChecked()` | `toBeChecked()` |
| Returns boolean | Uses locator directly |
| No auto-waiting | Auto-waiting |
| More code | Less code |
| Useful sometimes | Recommended |

# Best Practices

## Prefer Role Locators

Recommended:

```typescript
getByRole("radio", {
  name: "Option 1"
});
```

Avoid:

```typescript
locator("input[type='radio']")
```

---

## Prefer Locator Assertions

Recommended:

```typescript
await expect(locator)
  .toBeChecked();
```

instead of:

```typescript
expect(
  await locator.isChecked()
).toBeTruthy();
```

---

## Use force Only When Necessary

Good:

```typescript
.check({ force: true });
```

when the element is intentionally hidden.

Avoid forcing interactions unless required.

## Key Takeaway

For radio buttons:

1. Prefer `getByRole("radio")`
2. Use `check()` to select
3. Use `force: true` only when elements are hidden
4. Use `toBeChecked()` for validation
5. Selecting one radio button automatically deselects the others in the same group
