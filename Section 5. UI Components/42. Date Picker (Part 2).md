In this lesson, we continue working with the **Date Picker** and make the test more dynamic and reusable.

In the previous lesson, we selected a fixed date directly from the calendar. That works for simple examples, but in real-world automation, hardcoding a date is not ideal. Usually, you want to select dates relative to the current day, such as:

- **tomorrow**
- **7 days from now**
- **2 weeks from now**
- **200 days from now**

To handle this properly, we use JavaScript’s **`Date` object** so the test can calculate the desired date dynamically every time it runs.

---

# What We Want to Achieve

We want to automate a date picker so that the script can:

1. open the **Datepicker** page
2. calculate a future date dynamically using JavaScript
3. extract the **day**, **month**, and **year**
4. handle **month/year changes** in the calendar if needed
5. select the correct date
6. verify that the input field contains the expected value

---

# Step 2: Use JavaScript `Date` to select tomorrow dynamically

Instead of hardcoding a value like `"1"`, we create a `Date` object and add **1 day** to the current date.

```typescript
test("Date Picker (Part 2)", async ({ page }) => {
  await page.getByText("Forms").click();
  await page.getByText("Datepicker").click();

  const calenderInputField = page.getByPlaceholder("Form Picker");

  await calenderInputField.click();

  let date = new Date();

  date.setDate(date.getDate() + 1);

  const expectedDate = date.getDate().toString();

  await page
    .locator('[class="day-cell ng-star-inserted"]')
    .getByText(expectedDate, { exact: true })
    .click();

  await expect(calenderInputField).toHaveValue("Jun 22, 2026");
});
```

---

## How this works

### `new Date()`

Creates a JavaScript `Date` object representing the **current date and time**.

```typescript
let date = new Date();
```

### `date.getDate()`

Returns the **day of the month**.

### `date.setDate(date.getDate() + 1)`

Takes the current day and adds **1**, which gives us **tomorrow**.

```typescript
date.setDate(date.getDate() + 1);
```

### `date.getDate().toString()`

Extracts the day number and converts it to a string so we can use it in `getByText()`.

```typescript
const expectedDate = date.getDate().toString();
```

# Step 3: Build the expected input value dynamically

Now we extract:

- the **day**
- the **short month name** (e.g. `Jun`)
- the **year**

Then we combine them into the exact format used by the date picker input.

```typescript
test("Date Picker (Part 2)", async ({ page }) => {
  await page.getByText("Forms").click();
  await page.getByText("Datepicker").click();

  const calenderInputField = page.getByPlaceholder("Form Picker");

  await calenderInputField.click();

  let date = new Date();

  date.setDate(date.getDate() + 1);

  const expectedDate = date.getDate().toString();
  const expectedMonthShot = date.toLocaleString("En-US", { month: "short" });
  const expectedYear = date.getFullYear();
  const dateToAssert = `${expectedMonthShot} ${expectedDate}, ${expectedYear}`;

  await page
    .locator('[class="day-cell ng-star-inserted"]')
    .getByText(expectedDate, { exact: true })
    .click();

  await expect(calenderInputField).toHaveValue(dateToAssert);
});
```

# The Next Problem: Month Change

Now consider this scenario:

```typescript
date.setDate(date.getDate() + 14);
```

If today is **21 June 2026**, then **14 days later** is **5 July 2026**.

But the currently visible calendar still shows **June**.

So if we simply try to click day `"5"`, Playwright may select **June 5** instead of **July 5**.

That means the test fails because the script does not yet know how to move to the next month.

---

# Step 5: Handle month and year changes dynamically

To solve this, we need to:

1. read the **currently displayed month and year** from the calendar    
2. compare it with the **expected month and year**
3. if they do not match, click the **next month arrow**
4. repeat until the calendar shows the month/year we need
5. only then click the target day

---

# Final Dynamic Solution

```typescript
test("Date Picker (Part 2)", async ({ page }) => {
  await page.getByText("Forms").click();
  await page.getByText("Datepicker").click();

  const calenderInputField = page.getByPlaceholder("Form Picker");

  await calenderInputField.click();

  let date = new Date();

  date.setDate(date.getDate() + 14);

  const expectedDate = date.getDate().toString();
  const expectedMonthShot = date.toLocaleString("En-US", { month: "short" });
  const expectedMonthLong = date.toLocaleString("En-US", { month: "long" });
  const expectedYear = date.getFullYear();
  const dateToAssert = `${expectedMonthShot} ${expectedDate}, ${expectedYear}`;
  let calenderMonthAndYear = await page.locator("nb-calendar-view-mode").textContent();
  const expectedMonthAndYear = ` ${expectedMonthLong} ${expectedYear} `;

  while (!calenderMonthAndYear?.includes(expectedMonthAndYear)) {
    await page.locator('nb-calendar-pageable-navigation [data-name="chevron-right"]').click();

    calenderMonthAndYear = await page.locator("nb-calendar-view-mode").textContent();
  }

  await page
    .locator('[class="day-cell ng-star-inserted"]')
    .getByText(expectedDate, { exact: true })
    .click();

  await expect(calenderInputField).toHaveValue(dateToAssert);
});
```

# Breaking Down the Final Solution

---

## 1. Create the future date

```typescript
let date = new Date();
date.setDate(date.getDate() + 14);
```

This gives us the target date **14 days in the future**.

---

## 2. Extract all date parts

```typescript
const expectedDate = date.getDate().toString();
const expectedMonthShot = date.toLocaleString("En-US", { month: "short" });
const expectedMonthLong = date.toLocaleString("En-US", { month: "long" });
const expectedYear = date.getFullYear();
```

### Why do we need both short and long month names?

We use:

- **short month** (`Jul`) for the input field assertion    
- **long month** (`July`) to compare with the calendar header

---

## 3. Build the final input value

```typescript
const dateToAssert = `${expectedMonthShot} ${expectedDate}, ${expectedYear}`;
```

Example:

```typescript
"Jul 5, 2026"
```

---

## 4. Read the month/year currently displayed in the calendar

```typescript
let calenderMonthAndYear = await page.locator("nb-calendar-view-mode").textContent();
```

This locator reads the calendar header, which contains values such as:

```text
 July 2026
```

---

## 5. Build the expected month/year string

```typescript
const expectedMonthAndYear = ` ${expectedMonthLong} ${expectedYear} `;
```

Example:

```typescript
" July 2026 "
```

We compare this with the calendar header text.

---

## 6. Use a `while` loop to move through months

```typescript
while (!calenderMonthAndYear?.includes(expectedMonthAndYear)) {
  await page
    .locator('nb-calendar-pageable-navigation [data-name="chevron-right"]')
    .click();

  calenderMonthAndYear = await page.locator("nb-calendar-view-mode").textContent();
}
```

### What this loop does

As long as the displayed calendar month/year is **not** the one we want:

- click the **right arrow**    
- read the header again
- repeat

Once the calendar reaches the correct month and year, the loop stops.

---

## 7. Click the day

```typescript
await page
  .locator('[class="day-cell ng-star-inserted"]')
  .getByText(expectedDate, { exact: true })
  .click();
```

Now that the correct month is visible, selecting the day is safe.

---

## 8. Assert the input value

```typescript
await expect(calenderInputField).toHaveValue(dateToAssert);
```

This confirms that the selected date was written correctly into the input field.

# Summary

## 1. Use JavaScript `Date` for dynamic dates

Instead of hardcoding values, create a `Date` object:

```typescript
let date = new Date();
date.setDate(date.getDate() + 7);
```

This makes your tests reusable and future-proof.

---

## 2. Extract the date pieces you need

Useful values include:

```typescript
date.getDate()
date.getFullYear()
date.toLocaleString("En-US", { month: "short" })
date.toLocaleString("En-US", { month: "long" })
```

---

## 3. Build the assertion string dynamically

```typescript
const dateToAssert = `${expectedMonthShot} ${expectedDate}, ${expectedYear}`;
```

This lets you validate the input value without hardcoding a specific date.

---

## 4. Handle month changes using a loop

If the desired date is in another month or year, compare the calendar header and keep clicking the next-month arrow until the expected month/year appears.

```typescript
while (!calenderMonthAndYear?.includes(expectedMonthAndYear)) {
  await page.locator('nb-calendar-pageable-navigation [data-name="chevron-right"]').click();
  calenderMonthAndYear = await page.locator("nb-calendar-view-mode").textContent();
}
```

---

## 5. Select the day only after the correct month is visible

Once the calendar shows the expected month and year, select the date using:

```typescript
await page
  .locator('[class="day-cell ng-star-inserted"]')
  .getByText(expectedDate, { exact: true })
  .click();
```

---

# Key Takeaway

To automate date pickers reliably in Playwright:

- avoid hardcoded dates    
- use JavaScript’s `Date` object to generate future dates dynamically
- build the expected input value dynamically
- compare the visible month/year in the calendar
- use a loop to navigate across months when needed
- only then click the target day and assert the final value
