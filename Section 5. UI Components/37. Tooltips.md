In this lesson, we learn how to work with **tooltips in Playwright**.

Tooltips are small popup messages that appear near an element when you **hover over it**. In this example, the tooltip appears when hovering over one of the buttons on the **Modal & Overlays → Tooltip** page.

---

## What Makes Tooltips Difficult to Automate?

Tooltips are often **dynamic UI elements**:

- They appear only when the mouse hovers over an element    
- They disappear as soon as the mouse moves away
- Because of that, it can be difficult to inspect them in DevTools before they vanish

---

## Trick for Inspecting Tooltip Elements in the DOM

If the tooltip disappears too quickly while inspecting it, use this trick:

### Freeze the browser while the tooltip is visible

1. Open DevTools    
2. Go to the **Sources** tab
3. Hover over the element so the tooltip appears
4. Press:
    - **`F8` on Windows**
    - **`Cmd + \` on Mac**

This pauses JavaScript execution and freezes the browser, allowing you to inspect the tooltip element in the DOM.

## Test Setup

```typescript
import { test, expect } from "@playwright/test";

test.beforeEach(async ({ page }) => {
  await page.goto("http://localhost:4200/");
});
```

---

## Navigating to the Tooltip Page

We first navigate to the **Tooltip** page under **Modal & Overlays**.

```typescript
test("Tooltips", async ({ page }) => {
  await page.getByText("Modal & Overlays").click();
  await page.getByText("Tooltip").click();

  const toolTipCard = page.locator("nb-card", {
    hasText: "Tooltip Placements",
  });

  await toolTipCard.getByRole("button", { name: "Top" }).hover();
});
```

---

## Triggering the Tooltip with `hover()`

To simulate the user hovering over a button, Playwright provides the `.hover()` method.

```typescript
await toolTipCard.getByRole("button", { name: "Top" }).hover();
```

This triggers the tooltip just like moving the mouse over the button in the browser.

---

## Locating the Tooltip

If the application assigns the correct ARIA role to the tooltip, you can use:

```typescript
page.getByRole("tooltip");
```

### Important Note

This only works **if the tooltip element actually has the role `tooltip`** in the DOM.

In this application, that role is **not assigned**, so `getByRole("tooltip")` does not work here.

---

## Validating Tooltip Text

Since the tooltip is rendered as an `nb-tooltip` element, we can locate it directly and validate its text content.

```typescript
test("Tooltips", async ({ page }) => {
  await page.getByText("Modal & Overlays").click();
  await page.getByText("Tooltip").click();

  const toolTipCard = page.locator("nb-card", {
    hasText: "Tooltip Placements",
  });

  await toolTipCard.getByRole("button", { name: "Top" }).hover();

  const tooltip = await page.locator("nb-tooltip").textContent();

  expect(tooltip).toEqual("This is a tooltip");
});
```

## Summary

### 1. Tooltips are dynamic

They usually:

- appear on hover    
- disappear when the mouse moves away
- can be tricky to inspect manually

---

### 2. Use `hover()` to trigger tooltips

Playwright provides:

```typescript
await locator.hover();
```

This simulates a real mouse hover and displays the tooltip.

---

### 3. Inspecting tooltips may require freezing the browser

If the tooltip disappears too fast in DevTools:

- press **F8** on Windows    
- press **Cmd + \** on Mac

This pauses the browser so you can inspect the tooltip element in the DOM.

---

### 4. `getByRole("tooltip")` only works when the role exists

Use:

```typescript
page.getByRole("tooltip");
```

only if the tooltip element has the proper ARIA role in the DOM.

---

### 5. Direct locator fallback

If no tooltip role exists, inspect the DOM and use a normal locator such as:

```typescript
page.locator("nb-tooltip")
```

---

## Key Takeaway

When automating tooltips in Playwright:

- use `.hover()` to display the tooltip    
- inspect the DOM carefully because tooltips are often dynamically rendered
- prefer `getByRole("tooltip")` if available
- otherwise use a regular locator and validate the tooltip text manually
