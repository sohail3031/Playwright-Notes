In this lesson, we learn about **classes and methods** in JavaScript.

Classes and methods provide another way to organize and structure reusable code, similar to functions, but in a more object-oriented way.

---

# What is a Class?

A class is a blueprint used to create objects.

Classes help organize related functionality together.

---

# What is a Method?

A method is a function defined inside a class.

Methods describe actions that objects created from the class can perform.

---

# Why Use Classes?

Classes help:

- Organize code better    
- Create reusable components
- Group related methods together
- Improve readability and maintainability

---

# Creating a Class

## Basic Syntax

```javascript
class ClassName {

	methodName() {
		// code here
	}

}
```

---

# Important Naming Conventions

| Item        | Convention |
| ----------- | ---------- |
| Class Name  | PascalCase |
| Method Name | camelCase  |

---

# Example: Creating a Class

## `helper.js`

```javascript
// helper.js

export class CustomerDetails {

	printFirstName(firstName) {
		console.log(firstName)
	}

}
```

---

# Understanding the Syntax

- `class` keyword creates a class    
- `CustomerDetails` is the class name
- `printFirstName()` is a method
- `firstName` is a parameter

---

# Using the Class

To use the class in another file:

1. Import the class    
2. Create an instance
3. Call the methods

---

# Example

## `main.js`

```javascript
// main.js

import { CustomerDetails } from "../helper/helper.js"

var customerDetails = new CustomerDetails()

customerDetails.printFirstName("Steve")
```

### Output

```javascript
Steve
```

---

# Understanding `new`

```javascript
var customerDetails = new CustomerDetails()
```

The `new` keyword creates an object (instance) from the class.

---

# Adding More Methods

A class can contain multiple methods.

---

# Example

## `helper.js`

```javascript
// helper.js

export class CustomerDetails {

	printFirstName(firstName) {
		console.log(firstName)
	}

	printLastName(lastName) {
		console.log(lastName)
	}

}
```

---

## `main.js`

```javascript
// main.js

import { CustomerDetails } from "../helper/helper.js"

var customerDetails = new CustomerDetails()

customerDetails.printFirstName("Steve")
customerDetails.printLastName("Smith")
```

### Output

```javascript
Steve
Smith
```

---

# Exporting an Instance Instead of a Class

Another common approach is exporting an already-created object instance.

This helps keep the code cleaner.

---

# Example

## `helper.js`

```javascript
// helper.js

class CustomerDetails {

	printFirstName(firstName) {
		console.log(firstName)
	}

	printLastName(lastName) {
		console.log(lastName)
	}

}

export const customerDetails = new CustomerDetails()
```

---

## `main.js`

```javascript
// main.js

import { customerDetails } from "../helper/helper.js"

customerDetails.printFirstName("Steve")
customerDetails.printLastName("Smith")
```

### Output

```javascript
Steve
Smith
```

---

# Adding Documentation to Methods

We can add documentation comments to methods using JSDoc syntax.

This helps explain:

- What the method does    
- Expected parameters
- Parameter types

---

# Example

## `helper.js`

```javascript
// helper.js

class CustomerDetails {

	printFirstName(firstName) {
		console.log(firstName)
	}

	/**
	 * This method will print the last name
	 * @param {string} lastName
	 */
	printLastName(lastName) {
		console.log(lastName)
	}

}

export const customerDetails = new CustomerDetails()
```

---

# Benefits of Documentation

When hovering over the method in VS Code, you can see:

- Method description    
- Parameter information
- Expected data types

This improves developer experience and code readability.

# Class vs Function

| Functions               | Classes                       |
| ----------------------- | ----------------------------- |
| Reusable blocks of code | Blueprints for objects        |
| Independent             | Organize related methods      |
| Simpler structure       | Better for large applications |
