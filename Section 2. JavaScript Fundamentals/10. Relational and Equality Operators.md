# Relational Operators

## What are Relational Operators?

Relational operators compare two values and return a boolean result:

- `true`    
- `false`

These operators are also called **comparison operators**.

---

# Types of Relational Operators

| Operator | Meaning                  |
| -------- | ------------------------ |
| `>`      | Greater than             |
| `>=`     | Greater than or equal to |
| `<`      | Less than                |
| `<=`     | Less than or equal to    |

---

# Greater Than Operator (`>`)

```javascript
var result = 10 > 5

console.log(result)
```

### Output

```javascript
true
```

Explanation:

- `10` is greater than `5`    
- Therefore, the result is `true`

---

# Another Example


### Output

Explanation:

# Equality Operators

Equality operators are used to compare whether two values are equal.

JavaScript provides two types of equality comparisons:

- Loose comparison (`==`)    
- Strict comparison (`===`)

---

# Loose Comparison (`==`)

```javascript
var x = 1

console.log(x == "1")
```

### Output

```javascript
true
```

---

## Why is it `true`?

Loose comparison checks only the value.

In this example:

- `1` (number)    
- `"1"` (string)

Both have the same value, so JavaScript converts the types automatically and returns `true`.

---

# Strict Comparison (`===`)

```javascript
var x = 1

console.log(x === "1")
```

### Output

```javascript
false
```

---

## Why is it `false`?

Strict comparison checks:

- Value    
- Data type

Here:

| Value | Type   |
| ----- | ------ |
| `1`   | Number |
| `"1"` | String |

The values look similar, but the data types are different.

Therefore, the result is `false`.

---

# Matching Both Value and Data Type

```javascript
var x = 1

console.log(x === 1)
```

### Output

```javascript
true
```

Explanation:

- Values are equal    
- Data types are also equal

---

# Loose vs Strict Comparison

| Operator | Type              | Checks            |
| -------- | ----------------- | ----------------- |
| `==`     | Loose comparison  | Only value        |
| `===`    | Strict comparison | Value + data type |

---

# Best Practice

> [!important]  
> In modern JavaScript, it is recommended to use `===` (strict comparison) to avoid unexpected behavior caused by automatic type conversion.
