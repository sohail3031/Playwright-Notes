# Variables (`var` & `let`)

## Introduction

In this lesson, we will learn about:

- Variables
- Constants
- Primitive data types in JavaScript

---

# Creating Variables

In JavaScript, variables are created using:

- `var`    
- `let`

Example:

```javascript
var firstName = "John";
var lastName = "Smith";
```

---

# Understanding Variables

A variable is used to store information during program execution.

Think of a variable like a shopping cart:

- You put items into it    
- Carry them around
- Remove them later
- Reuse the cart again

Similarly, variables temporarily hold data while the program is running.

Once the program execution ends, the stored values disappear.

---

# Variable Syntax

## Structure

```javascript
var variableName = value;
```

or

```javascript
let variableName = value;
```

---

# Printing Variable Values

We can print variable values using:

```javascript
console.log();
```

Example:

```javascript
console.log(firstName);
console.log(lastName);
```

---

# Output

```text
John
Smith
```

---

# Declaring Variables First

Variables can also be declared first and assigned values later.

Example:

```javascript
var age, dateOfBirth, sex;

age = 5;
sex = "Male";

console.log(age);
```

---

# Output

```text
5
```

---

# Reassigning Variables

Variables can be updated with new values.

Example:

```javascript
age = 6;

console.log(age);
```

---

# Output

```text
6
```

---

# Why Are They Called Variables?

They are called variables because their values can vary/change during runtime.

Example:

```javascript
var age = 5;

age = 6;
```

The same variable stores different values at different times.

---

# Constants (`const`)

## Creating Constants

Constants are created using:

```javascript
const occupation = "Engineer";

console.log(occupation);
```

---

# Output

```text
Engineer
```

---

# Difference Between Variables and Constants

A constant value cannot be reassigned after creation.

Example:

```javascript
const occupation = "Engineer";

occupation = "Driver";
```

---

# Result

This produces an error:

```text
Assignment to constant variable.
```

---

# Constants Must Be Initialized

Unlike variables, constants must receive a value immediately during declaration.

Invalid example:

```javascript
const occupation;
```

This produces an error because constants cannot exist without an assigned value.

---

# Summary: Variables vs Constants

## Variables

- Store data during runtime    
- Values can change
- Created using:
    - `var`
    - `let`

---

## Constants

- Store fixed values    
- Values cannot change after assignment
- Created using:
    - `const`

---

# JavaScript Primitive Data Types

JavaScript has 5 commonly used primitive data types:

1. String    
2. Number
3. Boolean
4. Null
5. Undefined

---

# 1. String

Strings represent text values.

Example:

```javascript
var middleName = "David";
var anotherName = 'David';
```

Both single quotes and double quotes work for strings.

---

# 2. Number

Numbers represent numeric values.

Example:

```javascript
var ageOfBrother = 25;
```

---

## Important Note

```javascript
25
```

is a number.

But:

```javascript
"25"
```

is a string because it is inside quotes.

Strings cannot be used directly for arithmetic operations like numbers.

---

# 3. Boolean

Booleans represent logical values.

They can have only two values:

- `true`    
- `false`

Example:

```javascript
var isHeMarried = false;
```

---

# Usage of Boolean

Booleans are heavily used in:

- Conditions    
- Decision making
- Logical operations

Example:

```javascript
if (isHeMarried) {
    console.log("Married");
}
```

---

# 4. Null

`null` represents an intentionally empty value.

Example:

```javascript
var yearsInMarriage = null;
```

---

## Understanding `null`

Suppose a person is not married.

Then logically:

- Years in marriage should not exist    

Instead of assigning a random value, we assign:

```javascript
null
```

which means:

> No value is expected here.

---

# 5. Undefined

`undefined` means a variable has no assigned value.

Example:

```javascript
var numberOfCars = undefined;
```

---

## Understanding `undefined`

Usually appears when:

- A variable does not exist    
- A value was never assigned
- An object property is missing

---

# Complete Example

```javascript
var middleName = "David";          // String
var ageOfBrother = 25;             // Number
var isHeMarried = false;           // Boolean
var yearsInMarriage = null;        // Null
var numberOfCars = undefined;      // Undefined
```
