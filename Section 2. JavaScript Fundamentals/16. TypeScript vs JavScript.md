In this lesson, we learn about the differences between **TypeScript** and **JavaScript**.

TypeScript is built on top of JavaScript and adds one major feature:

- **Static typing**    

This helps developers catch errors earlier during development.

---

# JavaScript vs TypeScript

| JavaScript                         | TypeScript                       |
| ---------------------------------- | -------------------------------- |
| Dynamically typed                  | Statically typed                 |
| Errors often appear during runtime | Errors appear during development |
| More flexible                      | More strict                      |
| Easier to start                    | Better for large applications    |

---

# Creating a TypeScript File

TypeScript files use the `.ts` extension.

Example:

```text
lesson10.ts
```

---

# Basic Variables in TypeScript

```typescript
// file.ts

var customerFirstName = "John"
var customerLastName = "Smith"
var customerAge = 25
```

At first glance, the syntax looks very similar to JavaScript.

---

# Automatic Type Inference

TypeScript automatically detects data types.

Example:

```typescript
var customerFirstName = "John"
```

TypeScript understands that:

```typescript
customerFirstName: string
```

---

# Type Safety

Once a variable gets a specific type, TypeScript prevents assigning incompatible values.

---

# Example

```typescript
// file.ts

var customerFirstName = "John"

customerFirstName = 100 // Error
```

### Why?

- `customerFirstName` is a string    
- `100` is a number
- TypeScript prevents invalid assignments

---

# JavaScript Behavior

JavaScript does not enforce strict typing.

---

# Example

```javascript
// file.js

var age = "6"
age = 25
```

This works in JavaScript because variables can change data types dynamically.

---

# Explicit Type Declaration

In TypeScript, we can explicitly define variable types.

---

# Syntax

```typescript
var variableName: type = value
```

---

# Example

```typescript
// file.ts

var customerFirstName: string = "John"
var customerLastName: string = "Smith"

var customerAge: number = 25 // Correct
```

---

# Incorrect Assignment

```typescript
var customerAge: number = "25" // Wrong
```

### Error

- `"25"` is a string    
- `customerAge` expects a number

---

# Primitive Types in TypeScript

| Type      | Example          |
| --------- | ---------------- |
| `string`  | `"John"`         |
| `number`  | `25`             |
| `boolean` | `true` / `false` |

---

# Custom Types

TypeScript also allows us to create custom data types.

---

# Creating a Custom Type

```typescript
type Customer = {
	firstName: string,
	lastName: string,
	active: boolean
}
```

---

# Understanding the Custom Type

The `Customer` type requires:

| Property    | Type    |
| ----------- | ------- |
| `firstName` | string  |
| `lastName`  | string  |
| `active`    | boolean |

---

# Correct Object Example

```typescript
var firstCustomer: Customer = {
	firstName: "Mary",
	lastName: "Jones",
	active: true
}
```

This works because all required properties match the correct types.

---

# Missing Property Example

```typescript
var firstCustomer: Customer = {
	firstName: "Mary",
	lastName: "Jones"
}
```

### Error

- `active` property is missing    

---

# Extra Property Example

```typescript
var firstCustomer: Customer = {
	firstName: "Mary",
	lastName: "Jones",
	active: true,
	occupation: ""
}
```

### Error

- `occupation` does not exist in the `Customer` type    

---

# Wrong Data Type Example

```typescript
var firstCustomer: Customer = {
	firstName: "Mary",
	lastName: "Jones",
	active: "true"
}
```

### Error

- `"true"` is a string    
- `active` expects a boolean
