# 🏪 Application Overview

The Tool Shop is a hardware e-commerce web application. It sells Hand Tools, Power Tools, Other Tools, and Rentals.

**Key pages relevant to this project:**

|Page|URL Pattern|Description|
|---|---|---|
|Homepage|/|Product listing grid with filters|
|Category Page|/category/hand-tools|Filtered product listing|
|Product Detail|/product/{id}|Individual product page|
|Search Results|/?q=searchterm|Keyword search results|

# ✅ Test Scenarios — Homepage Product Listing

## TC-PB-001 — Homepage loads and displays product grid

- Navigate to [https://practicesoftwaretesting.com](https://practicesoftwaretesting.com/)
- Expected: Page title contains "Toolshop"
- Expected: A product grid is visible on the page
- Expected: At least 1 product card is displayed

## TC-PB-002 — Each product card displays required elements

For each visible product card on the homepage:

- Expected: Product name is visible and not empty
- Expected: Product price is visible and follows format $XX.XX
- Expected: Product image is present (img element exists within the card)

## TC-PB-003 — Product card count on homepage

- Navigate to homepage
- Expected: Exactly 9 products are displayed per page (default pagination)

## TC-PB-004 — Clicking a product card navigates to product detail page

- Click on any product card
- Expected: URL changes to /product/{id}
- Expected: Product detail page loads with a product name heading
- Expected: An "Add to Cart" button is visible

# 🔧 Locator Strategy Notes

Based on your Section 4 skills, practice these locator types on this site:

- User-facing:
    - `getByRole('heading')`
    - `getByRole('img')`
    - `getByPlaceholder('Search')`
    - `getByText('Add to Cart')`
- Child elements:
    - Product card → name, price, image (nested locators)
- Parent elements:
    - Locate a price, then navigate to its parent card
- Reusable locators:
    - Define `productCards`, `searchInput`, `sortDropdown` as const locators reused across tests
- Extracting values:
    - Use `.textContent()` to extract price text, then parse it
