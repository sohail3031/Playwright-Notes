## Introduction

In this lesson, we make one more improvement to our Playwright framework by creating a **helper base class** that contains methods reusable across multiple page objects.

For example, in a real framework, you may have utility methods for:

- waiting    
- generating data
- filtering
- common helper actions

Instead of duplicating these methods across page objects, we can place them in a single base class and let other page objects inherit from it using **OOP inheritance**.

---

# Why use a Helper Base?

Suppose you have methods that multiple page objects need, such as:

- waiting for a number of seconds    
- generating random test data
- formatting values
- reusable UI helpers

Rather than copy-pasting these methods into every page object, it is better to:

1. Create a **Helper Base** class
2. Put shared methods inside it
3. Make page objects **extend** that base class

This keeps the framework cleaner and avoids duplication.

---

# Creating the `HelperBase` class

Create a new file under the `pages` folder:

```typescript
// helperBase.ts
import { Page } from "@playwright/test";

export class HelperBase {
  readonly page: Page;

  constructor(page: Page) {
    this.page = page;
  }

  async waitForNumberOfSeconds(timeInSeconds: number) {
    await this.page.waitForTimeout(timeInSeconds * 1000);
  }
}
```

---

# What does this helper do?

The `HelperBase` class contains:

- a shared `page` instance    
- reusable helper methods

In this lesson, the demo method is:

```typescript
async waitForNumberOfSeconds(timeInSeconds: number) {
  await this.page.waitForTimeout(timeInSeconds * 1000);
}
```

This allows us to write:

```typescript
await this.waitForNumberOfSeconds(2);
```

instead of directly using:

```typescript
await this.page.waitForTimeout(2000);
```

---

# Using Inheritance in Page Objects

To use helper methods inside a page object, we make the page object **extend** `HelperBase`.

## Example: `NavigationPage`

```typescript
// navigationPage.ts
import { Page } from "@playwright/test";
import { HelperBase } from "./helperBase";

export class NavigationPage extends HelperBase {
  constructor(page: Page) {
    super(page);
  }

  async formLayoutsPage() {
    await this.selectGroupMenuItem("Forms");
    await this.page.getByText("Form Layouts").click();
    await this.waitForNumberOfSeconds(2);
  }

  async datePickerPage() {
    await this.selectGroupMenuItem("Forms");
    await this.page.getByText("Datepicker").click();
  }

  async smartTablePage() {
    await this.selectGroupMenuItem("Tables & Data");
    await this.page.getByText("Smart Table").click();
  }

  async toastrPage() {
    await this.selectGroupMenuItem("Modal & Overlays");
    await this.page.getByText("Toastr").click();
  }

  async tooltipPage() {
    await this.selectGroupMenuItem("Modal & Overlays");
    await this.page.getByText("Tooltip").click();
  }

  private async selectGroupMenuItem(groupItemTitle: string) {
    const groupMenuItem = this.page.getByTitle(groupItemTitle);

    await this.page.waitForLoadState("domcontentloaded");

    const expandedState = await groupMenuItem.getAttribute("aria-expanded");

    if (expandedState == "false") {
      await groupMenuItem.click();
    }
  }
}
```

---

# Important change: `super(page)`

Once a page object extends `HelperBase`, it should use the page instance from the base class.

That’s why the constructor becomes:

```typescript
constructor(page: Page) {
  super(page);
}
```

## Why?

Because `HelperBase` already stores:

```typescript
readonly page: Page;
```

So we no longer need to redeclare `page` inside each page object.

// formLayoutsPage.ts
import { Page } from "@playwright/test";
import { HelperBase } from "./helperBase";

export class FormLayoutPage extends HelperBase {
  constructor(page: Page) {
    super(page);
  }

  async submitUsingTheGridFormWithCredentialsAndSelectOption(
    email: string,
    password: string,
    optionText: string
  ) {
    const usingTheGridFrom = this.page.locator("nb-card", { hasText: "Using the Grid" });

    await usingTheGridFrom.getByRole("textbox", { name: "Email" }).fill(email);
    await usingTheGridFrom.getByRole("textbox", { name: "Password" }).fill(password);
    await usingTheGridFrom.getByRole("radio", { name: optionText }).check({ force: true });
    await usingTheGridFrom.getByRole("button").click();
  }

  /**
   * This method fills out the inline form with user details.
   * @param name - first and last name
   * @param email - valid email for the test user
   * @param rememberMe - true or false depending on whether the user session should be saved
   */
  async submitInlineFormWithNameEmailAndCheckbox(
    name: string,
    email: string,
    rememberMe: boolean
  ) {
    const inlineForm = this.page.locator("nb-card", { hasText: "Inline form" });

    await inlineForm.getByRole("textbox", { name: "Jane Doe" }).fill(name);
    await inlineForm.getByRole("textbox", { name: "Email" }).fill(email);

    if (rememberMe) {
      await inlineForm.getByRole("checkbox").check({ force: true });
    }

    await inlineForm.getByRole("button").click();
  }
}

# Updated `DatePickerPage`

```typescript
// datePickerPage.ts
import { Page, expect } from "@playwright/test";
import { HelperBase } from "./helperBase";

export class DatePickerPage extends HelperBase {
  constructor(page: Page) {
    super(page);
  }

  async selectCommonDatePickerDateFormToday(numberOfDaysFromToday: number) {
    const calendarInputField = this.page.getByPlaceholder("Form Picker");

    await calendarInputField.click();

    const dateToAssert = await this.selectDateInTheCalendar(numberOfDaysFromToday);

    await expect(calendarInputField).toHaveValue(dateToAssert);
  }

  private async selectDateInTheCalendar(numberOfDaysFromToday: number) {
    let date = new Date();

    date.setDate(date.getDate() + numberOfDaysFromToday);

    const expectedDate = date.getDate().toString();
    const expectedMonthShot = date.toLocaleString("En-US", { month: "short" });
    const expectedMonthLong = date.toLocaleString("En-US", { month: "long" });
    const expectedYear = date.getFullYear();
    const dateToAssert = `${expectedMonthShot} ${expectedDate}, ${expectedYear}`;

    let calendarMonthAndYeat = await this.page.locator("nb-calendar-view-mode").textContent();
    const exectedMonthAndYear = ` ${expectedMonthLong} ${expectedYear} `;

    while (!calendarMonthAndYeat?.includes(exectedMonthAndYear)) {
      await this.page
        .locator('nb-calendar-pageable-navigation [data-name="chevron-right"]')
        .click();

      calendarMonthAndYeat = await this.page.locator("nb-calendar-view-mode").textContent();
    }

    await this.page
      .locator(".day-cell.ng-star-inserted")
      .getByText(expectedDate, { exact: true })
      .click();

    return dateToAssert;
  }

  async selectDatePickerWithRangeFromToday(startDayFromToday: number, endDayFromToday: number) {
    const calendarInputField = this.page.getByPlaceholder("Range Picker");

    await calendarInputField.click();

    const dateToAssertStart = await this.selectDateInTheCalendar(startDayFromToday);
    const dateToAssertEnd = await this.selectDateInTheCalendar(endDayFromToday);
    const dateToAssert = `${dateToAssertStart} - ${dateToAssertEnd}`;

    await expect(calendarInputField).toHaveValue(dateToAssert);
  }
}
```

# `PageManager` remains the same

The `PageManager` still creates and manages instances of all page objects.

```typescript
// pageManager.ts
import { Page } from "@playwright/test";
import { NavigationPage } from "./navigationPage";
import { FormLayoutPage } from "./formLayoutsPage";
import { DatePickerPage } from "./datePickerPage";

export class PageManager {
  private readonly page: Page;
  private readonly navigationPage: NavigationPage;
  private readonly formLayoutPage: FormLayoutPage;
  private readonly datePickerPage: DatePickerPage;

  constructor(page: Page) {
    this.page = page;
    this.navigationPage = new NavigationPage(page);
    this.formLayoutPage = new FormLayoutPage(page);
    this.datePickerPage = new DatePickerPage(page);
  }

  navigateTo() {
    return this.navigationPage;
  }

  onFormLayoutsPage() {
    return this.formLayoutPage;
  }

  onDatePickerPage() {
    return this.datePickerPage;
  }
}
```

# Framework Architecture After This Refactor

At this point, the framework structure looks like this:

## Test Layer

- contains only spec/test files    
- calls methods from page objects via `PageManager`

## Code Layer

- **Page Objects** → contain page-specific actions
- **HelperBase** → contains reusable shared helper methods
- **PageManager** → manages and returns page object instances

# Benefits of `HelperBase`

Using a helper base class gives several benefits:

## 1. Avoids duplication

Shared methods are written once and reused everywhere.

## 2. Cleaner page objects

Page objects only contain page-specific behavior, while common utility logic stays in the base class.

## 3. Easier maintenance

If a common helper changes, you update it in one place only.

## 4. Better scalability

As the framework grows, you can keep adding reusable methods to `HelperBase` without cluttering every page object.

---

# Important OOP Concept Used: Inheritance

This lesson introduces **inheritance**.

Example:

```typescript
export class NavigationPage extends HelperBase
```

This means `NavigationPage` inherits everything from `HelperBase`, including:

- the `page` property    
- reusable helper methods like `waitForNumberOfSeconds()`

So inside `NavigationPage`, `FormLayoutPage`, or `DatePickerPage`, you can directly call:

```typescript
await this.waitForNumberOfSeconds(2);
```

without creating a separate `HelperBase` object.
