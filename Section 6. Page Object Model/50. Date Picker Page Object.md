In this lesson, we’ll continue working with **Page Objects** and create a new page object for the **Datepicker page**.

I’ll show you how to **reuse the code** we already created for the common datepicker to select a date in the calendar, and then use that same logic for the **date range picker**, where we need to select **two dates** instead of one.

As you can see, both the **common date picker** and the **date range picker** use a very similar calendar UI. The only real difference is:

- In the **common date picker**, we click **once** to select a date.
- In the **date range picker**, we click **twice** to select a start and end date.

Everything else is almost the same, so we can **reuse that logic inside a page object** and create two methods:

1. One method to select a date in the **common date picker**
2. Another method to select a date range in the **date range picker**

---

## Create the `DatePickerPage` page object

We’ll create a new page object for the **Datepicker page**.

### `datePickerPage.ts`

```typescript
import { Page, expect } from "@playwright/test";

export class DatePickerPage {
  private readonly page: Page;

  constructor(page: Page) {
    this.page = page;
  }

  async selectCommonDatePickerDateFromToday(numberOfDaysFromToday: number) {
    const calendarInputField = this.page.getByPlaceholder("Form Picker");

    await calendarInputField.click();

    let date = new Date();
    date.setDate(date.getDate() + numberOfDaysFromToday);

    const expectedDate = date.getDate().toString();
    const expectedMonthShort = date.toLocaleString("en-US", { month: "short" });
    const expectedMonthLong = date.toLocaleString("en-US", { month: "long" });
    const expectedYear = date.getFullYear();

    const dateToAssert = `${expectedMonthShort} ${expectedDate}, ${expectedYear}`;

    let calendarMonthAndYear = await this.page.locator("nb-calendar-view-mode").textContent();
    const expectedMonthAndYear = ` ${expectedMonthLong} ${expectedYear} `;

    while (!calendarMonthAndYear?.includes(expectedMonthAndYear)) {
      await this.page
        .locator('nb-calendar-pageable-navigation [data-name="chevron-right"]')
        .click();

      calendarMonthAndYear = await this.page.locator("nb-calendar-view-mode").textContent();
    }

    await this.page
      .locator(".day-cell.ng-star-inserted")
      .getByText(expectedDate, { exact: true })
      .click();

    await expect(calendarInputField).toHaveValue(dateToAssert);
  }
}
```

# Refactor the calendar logic into a reusable private method

If you look at the method above, the **main reusable logic** is the part that actually selects the date in the calendar.

The steps that differ between the two datepickers are mainly:

- which input field we click (`Form Picker` vs `Range Picker`)    
- what we assert afterward (single date vs date range)

So instead of duplicating the calendar-selection logic, we’ll extract it into a **private helper method**.

---

## Refactored `DatePickerPage`


# Add support for the Date Range Picker

Now let’s create a second method for the **range picker**.

This method needs **two parameters**:

- `startDayFromToday`    
- `endDayFromToday`

We’ll click the **Range Picker** input, call the reusable helper method **twice**, and then assert the full range value.

---

## Final `DatePickerPage` with both methods

```typescript
import { Page, expect } from "@playwright/test";

export class DatePickerPage {
  private readonly page: Page;

  constructor(page: Page) {
    this.page = page;
  }

  async selectCommonDatePickerDateFromToday(numberOfDaysFromToday: number) {
    const calendarInputField = this.page.getByPlaceholder("Form Picker");

    await calendarInputField.click();

    const dateToAssert = await this.selectDateInTheCalendar(numberOfDaysFromToday);

    await expect(calendarInputField).toHaveValue(dateToAssert);
  }

  async selectDatePickerWithRangeFromToday(
    startDayFromToday: number,
    endDayFromToday: number
  ) {
    const calendarInputField = this.page.getByPlaceholder("Range Picker");

    await calendarInputField.click();

    const dateToAssertStart = await this.selectDateInTheCalendar(startDayFromToday);
    const dateToAssertEnd = await this.selectDateInTheCalendar(endDayFromToday);

    const dateToAssert = `${dateToAssertStart} - ${dateToAssertEnd}`;

    await expect(calendarInputField).toHaveValue(dateToAssert);
  }

  private async selectDateInTheCalendar(numberOfDaysFromToday: number) {
    let date = new Date();
    date.setDate(date.getDate() + numberOfDaysFromToday);

    const expectedDate = date.getDate().toString();
    const expectedMonthShort = date.toLocaleString("en-US", { month: "short" });
    const expectedMonthLong = date.toLocaleString("en-US", { month: "long" });
    const expectedYear = date.getFullYear();

    const dateToAssert = `${expectedMonthShort} ${expectedDate}, ${expectedYear}`;

    let calendarMonthAndYear = await this.page.locator("nb-calendar-view-mode").textContent();
    const expectedMonthAndYear = ` ${expectedMonthLong} ${expectedYear} `;

    while (!calendarMonthAndYear?.includes(expectedMonthAndYear)) {
      await this.page
        .locator('nb-calendar-pageable-navigation [data-name="chevron-right"]')
        .click();

      calendarMonthAndYear = await this.page.locator("nb-calendar-view-mode").textContent();
    }

    await this.page
      .locator(".day-cell.ng-star-inserted")
      .getByText(expectedDate, { exact: true })
      .click();

    return dateToAssert;
  }
}
```

# Important locator note for the range picker

When selecting a date in the range picker, the calendar cells may have slightly different class names compared to the single date picker.

For example:

- Single date picker might use:    
    - `.day-cell.ng-star-inserted`
- Range picker might use something like:
    - `.range-cell.day-cell.ng-star-inserted`

To make the locator work for **both calendars**, we use a selector that targets the shared classes:

```typescript
.day-cell.ng-star-inserted
```

That’s why the reusable method still works for both scenarios.

# Summary

In this lesson, we:

- created a new **`DatePickerPage` page object**    
- added a method to select a date in the **common date picker**
- extracted the calendar logic into a **private reusable helper method**
- reused that helper method to implement the **date range picker**
- avoided duplication by following the **DRY principle**

This is a very useful Page Object Model technique: if multiple flows share the same internal logic, move that logic into a **private reusable method** and call it from different public page-object methods.
