In this lesson, we create our **first Page Object** in Playwright.

We will build a Page Object for the **navigation menu** of the test application. Even though the navigation bar is not a standalone page, it appears across multiple pages in the application, so it makes sense to model it as a reusable Page Object.

The goal is to create a **Navigation Page Object** that helps us navigate to the **Form Layouts** page.

---

## Why use a Page Object here?

Instead of repeating navigation steps like:

- click **Forms**
- click **Form Layouts**

inside every test, we can move that logic into a reusable class method.

This gives us:

- **cleaner tests**
- **reusable navigation logic**
- **easier maintenance**
- **better readability**

# Step 4: Add the Playwright `Page` object

A Page Object needs access to Playwright’s `page` fixture so it can interact with the browser.

So we import `Page` and store it inside the class.

## `navigationPage.ts`

```typescript
import { Page } from "@playwright/test";

export class NavigationPage {
  readonly page: Page;

  constructor(page: Page) {
    this.page = page;
  }
}
```

---

# Understanding the constructor

```typescript
constructor(page: Page) {
  this.page = page;
}
```

### What this means:

- The constructor expects a **Playwright `page` object** as an argument.
- We store that page object inside the class using `this.page`.
- Now every method inside this class can use the same browser page instance.

---

# Step 5: Add a navigation method

Now we create a method that navigates to the **Form Layouts** page.

## `navigationPage.ts`

```typescript
import { Page } from "@playwright/test";

export class NavigationPage {
  readonly page: Page;

  constructor(page: Page) {
    this.page = page;
  }

  async formLayoutsPage() {
    await this.page.getByText("Forms").click();
    await this.page.getByText("Form Layouts").click();
  }
}
```

---

# What this method does

The `formLayoutsPage()` method performs the following steps:

1. Clicks **Forms**
2. Clicks **Form Layouts**

Now this logic is reusable from any test.

# Breakdown of the Page Object

## 1. Import `Page`

```typescript
import { Page } from "@playwright/test";
```

We import Playwright’s `Page` type so we can use it in our class.

---

## 2. Export the class

```typescript
export class NavigationPage
```

The `export` keyword makes this class available to other files, such as test files.

---

## 3. Create a page field

```typescript
readonly page: Page;
```

This creates a property named `page` inside the class.

- `readonly` means it should not be reassigned after initialization.
- `Page` is the type of the property.

---

## 4. Store the Playwright page in the constructor

```typescript
constructor(page: Page) {
  this.page = page;
}
```

This assigns the Playwright `page` object to the class property.

---

## 5. Create reusable methods

```typescript
async formLayoutsPage() {
  await this.page.getByText("Forms").click();
  await this.page.getByText("Form Layouts").click();
}
```

This method contains the reusable navigation steps.

---

# Why use `this.page`?

Inside a class, `this` refers to the current object instance.

So:

```typescript
this.page
```

means:

> “Use the `page` property that belongs to this instance of the `NavigationPage` class.”

---

# Benefits of this approach

## 1. Reusability

You can call:

```typescript
await navigateTo.formLayoutsPage();
```

from any test.

---

## 2. Cleaner tests

Instead of writing:

```typescript
await page.getByText("Forms").click();
await page.getByText("Form Layouts").click();
```

inside every test, you write:

```typescript
await navigateTo.formLayoutsPage();
```

---

## 3. Easier maintenance

If the UI changes later, you only need to update the locator in **one place** inside the Page Object.

---

## 4. Better readability

The test reads more like business steps:

```typescript
await navigateTo.formLayoutsPage();
```

which is much easier to understand than repeated low-level clicks.

# Key takeaway

A Page Object helps you move repeated UI interactions into reusable class methods.

Instead of writing low-level Playwright actions in every test, you can organize them into meaningful methods such as:

```typescript
await navigateTo.formLayoutsPage();
```

This makes your test framework cleaner, more scalable, and easier to maintain.
