In this lesson, we continue building our **Navigation Page Object** and add more reusable navigation methods for different pages in the demo application.

Previously, we created a `NavigationPage` class with a method to navigate to the **Form Layouts** page. Now we’ll extend it to support navigation to other pages we’ve already worked with, such as:

- **Datepicker**
- **Smart Table**
- **Toastr**
- **Tooltip**

We’ll also fix an important issue: menu groups like **Forms** or **Modal & Overlays** can be either **expanded** or **collapsed**, so our page object methods need to be smart enough to handle both states.

---

# Goal of This Lesson

We want our `NavigationPage` object to:

1. Navigate to multiple pages in the app.
2. Avoid repeating the same navigation logic.
3. Correctly handle expandable/collapsible menu groups.
4. Be reusable across tests.

---

# Problem with This Approach

At first glance, this looks fine, but there is a hidden problem.

Take these two methods:

```typescript
async formLayoutsPage() {
  await this.page.getByText("Forms").click();
  await this.page.getByText("Form Layouts").click();
}

async datePickerPage() {
  await this.page.getByText("Forms").click();
  await this.page.getByText("Datepicker").click();
}
```

If `formLayoutsPage()` is called first, the **Forms** menu expands.

Then if `datePickerPage()` is called immediately after, it clicks **Forms** again.

But clicking **Forms** again collapses the menu instead of expanding it.

That means the **Datepicker** item may no longer be visible, and the test can fail.

---

# Demonstrating the Failure

Here is an example where the test can fail more clearly by adding a delay:

```typescript
import { Page } from "@playwright/test";

export class NavigationPage {

    await this.page.getByText("Smart Table").click();
  }

  async toastrPage() {
    await this.page.getByText("Modal & Overlays").click();
    await this.page.getByText("Toastr").click();
  }

  async tooltipPage() {
    await this.page.getByText("Modal & Overlays").click();
    await this.page.getByText("Tooltip").click();
  }
}
```

---

# Why This Happens

The group menu items like **Forms**, **Tables & Data**, and **Modal & Overlays** are collapsible sections.

When you click them:

- if collapsed → they expand
- if expanded → they collapse

So our navigation logic must first check the current state of the group menu item before clicking it.

---

# Better Approach: Create a Helper Method

We’ll create a reusable helper method that:

1. receives the group menu title (e.g. `"Forms"`)    
2. checks whether that group is already expanded
3. clicks it only if it is currently collapsed

This helper method will be used by all navigation methods.

---

# Step 1: Add a Private Helper Method

We’ll add a private method called `selectGroupMenuItem()`.

## Updated `navigationPage.ts`

```typescript
import { Page } from "@playwright/test";

export class NavigationPage {
  readonly page: Page;

  constructor(page: Page) {
    this.page = page;
  }

  async formLayoutsPage() {
    await this.selectGroupMenuItem("Forms");
    await this.page.getByText("Form Layouts").click();
  }

  async datePickerPage() {
    await this.selectGroupMenuItem("Forms");
    await this.page.waitForTimeout(1000);
    await this.page.getByText("Datepicker").click();
  }

  async smartTablePage() {
    await this.page.getByText("Tables & Data").click();
    await this.page.getByText("Smart Table").click();
  }

  async toastrPage() {
    await this.page.getByText("Modal & Overlays").click();
    await this.page.getByText("Toastr").click();
  }

  async tooltipPage() {
    await this.page.getByText("Modal & Overlays").click();
    await this.page.getByText("Tooltip").click();
  }

  private async selectGroupMenuItem(groupItemTitle: string) {
    const groupMenuItem = this.page.getByTitle(groupItemTitle);
    const expandedState = await groupMenuItem.getAttribute("aria-expanded");

    if (expandedState == "false") {
      await groupMenuItem.click();
    }
  }
}
```

---

# Understanding the Helper Method

## The method

```typescript
private async selectGroupMenuItem(groupItemTitle: string) {
  const groupMenuItem = this.page.getByTitle(groupItemTitle);
  const expandedState = await groupMenuItem.getAttribute("aria-expanded");

  if (expandedState == "false") {
    await groupMenuItem.click();
  }
}
```

---

# What it does

### 1. Find the group menu item

```typescript
const groupMenuItem = this.page.getByTitle(groupItemTitle);
```

If `groupItemTitle` is `"Forms"`, it finds the **Forms** group menu item.

---

### 2. Read its expansion state

```typescript
const expandedState = await groupMenuItem.getAttribute("aria-expanded");
```

The `aria-expanded` attribute tells us whether the group is open or closed.

Possible values:

- `"true"` → expanded
- `"false"` → collapsed

---

### 3. Expand it only if needed

```typescript
if (expandedState == "false") {
  await groupMenuItem.click();
}
```

This means:

- if the menu is already expanded → do nothing    
- if the menu is collapsed → click it to expand

This prevents accidental collapsing of a menu that is already open.

---

# Why make it `private`?

We use:

```typescript
private async selectGroupMenuItem(...)
```

because this helper method is meant to be used **only inside the `NavigationPage` class**.

It is an internal implementation detail and should not be called directly from test files.

Instead of directly clicking the group item every time, we now call:


# Updated Test File

Now let’s use all navigation methods in a single test.

## `pageObjectModel.spec.ts`

```typescript
import { test, expect } from "@playwright/test";
import { NavigationPage } from "../pages/navigationPage";

test.beforeEach(async ({ page }) => {
  await page.goto("http://localhost:4200");
});

test("First Page Object", async ({ page }) => {
  const navigateTo = new NavigationPage(page);

  await navigateTo.formLayoutsPage();
  await navigateTo.datePickerPage();
  await navigateTo.smartTablePage();
  await navigateTo.toastrPage();
  await navigateTo.tooltipPage();
});
```

---

# Flow of the Test

When this test runs:

1. `formLayoutsPage()`
    - expands **Forms** if needed
    - clicks **Form Layouts**
2. `datePickerPage()`
    - checks whether **Forms** is already expanded
    - since it is already open, it does **not** click it again
    - clicks **Datepicker**
3. `smartTablePage()`
    - expands **Tables & Data** if needed
    - clicks **Smart Table**
4. `toastrPage()`
    - expands **Modal & Overlays** if needed
    - clicks **Toastr**
5. `tooltipPage()`
    - checks whether **Modal & Overlays** is already expanded
    - if yes, it doesn’t click it again
    - clicks **Tooltip**

---

# Why This Refactor Is Better

## 1. Removes duplicated logic

Without the helper method, every navigation method would need to manually handle expand/collapse behavior.

Now that logic lives in one place:

```typescript
private async selectGroupMenuItem(groupItemTitle: string)
```

---

## 2. Makes tests more stable

The menu state no longer breaks the test.

Even if the group is already expanded, the method won’t collapse it accidentally.

---

## 3. Makes the Page Object smarter

This is one of the main benefits of Page Object Model:

- the test stays simple    
- the page object handles UI details internally

---

## 4. Easier to maintain

If the application changes how menu groups work, you only need to update the helper method instead of changing every navigation method.

The navigation menu can be in different states:

---

# Important Note About `waitForTimeout()`

The lesson uses:

```typescript
await this.page.waitForTimeout(1000);
```

to demonstrate the collapsing issue.

This is fine for learning purposes, but in real automation frameworks you should avoid arbitrary waits whenever possible.

Prefer waiting for actual UI conditions such as:

- an element becoming visible
- a page finishing navigation
- a network response completing
