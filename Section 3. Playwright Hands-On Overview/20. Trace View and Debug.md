In this lesson, we will talk about how to analyze your test execution and debug your tests when you do not understand what is going on with the code.

Debugging helps you understand the logic of the application and identify why your tests are failing.

In the previous lesson, we already talked about the UI mode, which provides a very nice visual representation of all the test steps and actions performed by Playwright.

In this lesson, I will show you a few more ways to analyze code execution and debug your tests.

---

# Using Trace Viewer

One of the ways to analyze your tests from the command line is by using **Trace Viewer**.

To run tests with trace enabled, use the following command:

```bash
npx playwright test --project=chromium --trace on
```

After the tests execute, open the report.

You will notice a new **Traces** section in the report.

When you click on a trace, it opens a detailed execution viewer similar to Playwright UI mode.

You can see:

- All executed steps    
- Before and after states of actions
- Page screenshots
- Click locations
- Navigation flow

This is extremely useful when tests fail in CI/CD pipelines or remote environments.

---

# Trace Configuration

By default, Playwright generates traces only on the first retry of a failed test.

Inside `playwright.config.ts`, the default configuration looks like this:

```javascript
// playwright.config.ts
trace: 'on-first-retry'
```

If you want Playwright to generate traces for every test execution, change it to:

```javascript
// playwright.config.ts
trace: 'on'
```

This means traces will always be generated, even for successful tests.

---

# Using Playwright Debug Mode

Another way to debug tests is by using the built-in Playwright debugger.

Run the following command:

```bash
npx playwright test --project=chromium --debug
```

This opens two windows:

1. Playwright Inspector    
2. Browser Window

---

# Playwright Inspector

The Playwright Inspector allows you to execute your test step by step.

You can:

- Resume execution    
- Pause execution
- Step through each command
- Inspect locators
- Analyze actions in detail

---

# Step-by-Step Execution

For example:

- The first step navigates to the page    
- The next step clicks the **Get Started** button
- The next step verifies the heading

When stepping through the test, Playwright shows detailed logs such as:

- Waiting for locator
- Element visibility checks
- Scroll into view
- Click action execution
- Navigation completion

This helps you understand exactly what Playwright is doing internally.

---

# Using VS Code Debugger

The third debugging technique is using the built-in debugger in Visual Studio Code.

This is one of the most convenient approaches.

---

# Setting Breakpoints

Open your test file and place a breakpoint on the line where you want execution to pause.

For example:

```javascript
await page.getByRole("link", { name: "Get started" }).click();
```

Click next to the line number to add a breakpoint.

---

# Running Debug Mode from Test Explorer

In the VS Code Test Explorer:

- Click the **Debug Test** button    
- Playwright launches the browser in debug mode
- Execution pauses at the breakpoint

VS Code highlights the currently executing line.

You can then:

- Continue execution
- Step over commands
- Restart the test
- Stop debugging

---

# Variable Inspection

One major advantage of VS Code debugging is variable inspection.

When you hover over variables, VS Code displays their current values.

This becomes extremely useful in complex automation frameworks where:

- Data changes dynamically    
- Multiple variables are involved
- API responses are processed
- Conditional logic is used

You can inspect values live during execution.

---

# Summary

In this lesson, we learned three important debugging techniques in Playwright.

## 1. Trace Viewer

- Run tests with tracing enabled    
- Analyze test execution visually
- Useful for CI/CD debugging

## 2. Playwright Inspector

- Step-by-step execution
- Detailed action logs
- Real browser interaction

## 3. VS Code Debugger

- Breakpoints
- Variable inspection
- Full debugging experience inside VS Code

All three techniques are extremely useful during Playwright test development and troubleshooting.
