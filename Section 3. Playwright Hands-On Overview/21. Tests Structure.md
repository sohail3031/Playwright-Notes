In this lesson, we learn about the **Playwright test structure** in detail. We continue working in the **PW Practise App** project and start writing our first tests from scratch.

---

## Installing Playwright

To install Playwright in an existing project, use the following command:

```bash
npm init playwright@latest --force
```

After installation:

- Playwright config file is created    
- `package.json` is updated with required dependencies
- Default test folders are added

---

## Project Structure Cleanup

By default, Playwright creates sample folders and files such as:

- `tests-examples` folder    
- example spec files

We can safely delete them and start fresh.

Recommended structure:

```text
tests/
```

Create your own test files inside the `tests` folder.

---

## Creating Your First Test File

Create a new file inside the `tests` folder:

```text
first-test.spec.ts
```

## Basic Test Structure

To write a test, import `test` from Playwright:

```typescript
import { test } from "@playwright/test";

test("first test", () => {

});
```

---

## Adding Multiple Tests

You can define multiple tests inside the same file:

```typescript
import { test } from "@playwright/test";

test("test 1", () => {

});

test("test 2", () => {

});
```

---

## Grouping Tests Using `test.describe`

You can group related tests into a **test suite** using `test.describe`.

This helps organize tests logically and manage shared context.

```typescript
import { test } from "@playwright/test";

test.describe("Scenario 1", () => {

    test("test 1", () => {

    });

    test("test 2", () => {

    });

    test("test 3", () => {

    });

});

test.describe("Scenario 2", () => {

    test("test 1", () => {

    });

    test("test 2", () => {

    });

    test("test 3", () => {

    });

});
```

Each `describe` block creates an isolated context for its tests.

## Playwright Page Fixture

Playwright provides a built-in **page fixture**, which represents a blank browser tab.

It allows you to:

- Open pages    
- Interact with UI
- Perform actions like click, type, etc.

Example:

```typescript
test("test 1", ({ page }) => {

});
```

---

## Navigating to a URL

To open a page:

```typescript
import { test } from "@playwright/test";

test("test 1", async ({ page }) => {
    await page.goto("http://localhost:4200/");
});
```

---

## Important Concept: Async & Await

Playwright actions return **Promises**, so we must use `async/await`.

### Why?

A Promise can:

- Resolve successfully    
- Fail with an error
- Take time to complete

If we don’t wait for it, JavaScript continues execution and causes errors.

---

## Incorrect Example

```typescript
import { test } from "@playwright/test";

// This will not work
test("test 1", ({ page }) => {
    page.goto("http://localhost:4200/");
});
```

This fails because the promise is not awaited.

---

## Correct Example

```typescript
import { test } from "@playwright/test";

// Correct code
test("test 1", async ({ page }) => {
    await page.goto("http://localhost:4200/");
});
```

> If a Playwright method returns a Promise, always use `await`.

---

## Common Beginner Mistake

Many beginners forget to use `await`, which leads to:

- Test failures    
- Race conditions
- Unexpected behavior

Always check:

- Is this method asynchronous?
- Does it return a Promise?

If yes, use `await`.

## Navigating Through UI Elements

You can interact with UI elements using text selectors.

Example:

```typescript
import { test } from "@playwright/test";

// Navigating to Forms section
test("test 1", async ({ page }) => {
    await page.goto("http://localhost:4200/");

    await page.getByText("Forms").click();
    await page.getByText("Form Layouts").click();
});
```

This test:

1. Opens the application    
2. Clicks on **Forms**
3. Clicks on **Form Layouts**
4. Opens the Form Layouts page
