In this lesson, we learn about **Hooks** and **Control Flow** in Playwright.

Hooks help us:

- Reduce code duplication    
- Improve test organization
- Manage setup and teardown logic efficiently

---

# Problem: Repetitive Code

Consider the following example:

```typescript
import { test } from "@playwright/test";

test("navigate to form layout page", async ({ page }) => {
  await page.goto("http://localhost:4200/");
  await page.getByText("Forms").click();
  await page.getByText("Form Layouts").click();
});

test("navigate to datepicker page", async ({ page }) => {
  await page.goto("http://localhost:4200/");
  await page.getByText("Forms").click();
  await page.getByText("Datepicker").click();
});
```

Both tests repeat the same steps:

```typescript
await page.goto("http://localhost:4200/");
await page.getByText("Forms").click();
```

This creates unnecessary duplication.

---

# Using `beforeEach()` Hook

To avoid repeating the same setup code, we can use the `beforeEach()` hook.

The code inside `beforeEach()` runs before every test.

```typescript
import { test } from "@playwright/test";

test.beforeEach(async ({ page }) => {
  await page.goto("http://localhost:4200/");
  await page.getByText("Forms").click();
});

test("navigate to form layout page", async ({ page }) => {
  await page.getByText("Form Layouts").click();
});

test("navigate to datepicker page", async ({ page }) => {
  await page.getByText("Datepicker").click();
});
```

This makes the code:

- Cleaner    
- Easier to maintain
- Less repetitive

---

# Available Hooks in Playwright

Playwright provides four main hooks:

| Hook           | Description                |
| -------------- | -------------------------- |
| `beforeEach()` | Runs before every test     |
| `afterEach()`  | Runs after every test      |
| `beforeAll()`  | Runs once before all tests |
| `afterAll()`   | Runs once after all tests  |

---

# `beforeAll()` Hook

`beforeAll()` executes only once before the entire test file or suite starts.

Example use cases:

- Database setup
- Authentication setup
- Global initialization

```typescript
test.beforeAll(async () => {

});
```

---

# `afterEach()` and `afterAll()` Hooks

These hooks are used for cleanup operations.

Example use cases:

- Deleting test data
- Closing connections
- Resetting application state

```typescript
test.afterEach(async () => {

});

test.afterAll(async () => {

});
```

---

# Best Practice for Hooks

Avoid heavy usage of:

- `afterEach()`    
- `afterAll()`

Why?

- They can introduce flaky tests
- Cleanup failures may affect subsequent tests

A better approach is:

- Prepare a clean state inside `beforeEach()`
- Create/reset test data before test execution

This improves:

- Stability
- Reliability
- Performance

# Hooks Inside `test.describe()`

Each `describe` block can have its own hooks.

This allows different test suites to have different setups.

---

## Example: Multiple Test Suites

```typescript
import { test } from "@playwright/test";

test.describe("scenario 1", () => {

  test.beforeEach(async ({ page }) => {
    await page.goto("http://localhost:4200/");
    await page.getByText("Forms").click();
  });

  test("navigate to form layout page", async ({ page }) => {
    await page.getByText("Form Layouts").click();
  });

  test("navigate to datepicker page", async ({ page }) => {
    await page.getByText("Datepicker").click();
  });

});

test.describe("scenario 2", () => {

  test.beforeEach(async ({ page }) => {
    await page.goto("http://localhost:4200/");
    await page.getByText("Modal & Overlays").click();
  });

  test("navigate to form dialog page", async ({ page }) => {
    await page.getByText("Dialog").click();
  });

  test("navigate to window page", async ({ page }) => {
    await page.getByText("Window").click();
  });

});
```

---

# Hook Scope

In the previous example:

- The `beforeEach()` inside `"scenario 1"` runs only for tests inside `"scenario 1"`    
- The `beforeEach()` inside `"scenario 2"` runs only for tests inside `"scenario 2"`

Each suite has its own isolated setup.

---

# Global `beforeEach()` + Suite-Level Hooks

You can also combine:

- A global `beforeEach()`    
- Suite-specific `beforeEach()`

Example structure:

```typescript
test.beforeEach(async ({ page }) => {
  await page.goto("http://localhost:4200/");
});
```

Then inside each suite:

```typescript
test.beforeEach(async ({ page }) => {
  await page.getByText("Forms").click();
});
```

Execution order:

1. Global `beforeEach()`    
2. Suite-level `beforeEach()`
3. Test execution

# Running Specific Test Suites

You can run only one test suite using:

```typescript
test.describe.only()
```

Example:

```typescript
import { test } from "@playwright/test";

test.describe.only("scenario 1", () => {

  test.beforeEach(async ({ page }) => {
    await page.goto("http://localhost:4200/");
    await page.getByText("Forms").click();
  });

  test("navigate to form layout page", async ({ page }) => {
    await page.getByText("Form Layouts").click();
  });

  test("navigate to datepicker page", async ({ page }) => {
    await page.getByText("Datepicker").click();
  });

});

test.describe("scenario 2", () => {

  test.beforeEach(async ({ page }) => {
    await page.goto("http://localhost:4200/");
    await page.getByText("Modal & Overlays").click();
  });

  test("navigate to form dialog page", async ({ page }) => {
    await page.getByText("Dialog").click();
  });

  test("navigate to window page", async ({ page }) => {
    await page.getByText("Window").click();
  });

});
```

Only `"scenario 1"` will run.

# Skipping Test Suites

You can skip an entire suite using:

```typescript
test.describe.skip()
```

Example:

```typescript

  test.beforeEach(async ({ page }) => {

  test("navigate to form layout page", async ({ page }) => {
    await page.getByText("Form Layouts").click();

test.describe("scenario 2", () => {

  test.beforeEach(async ({ page }) => {
```
