In this lesson, we learn how to execute Playwright tests using the **Command Line Interface (CLI)**.

In the previous lesson, we installed Playwright and created a project with example test files.

Now, we will learn how to:

- Run tests
- Run tests in specific browsers
- Run tests in headed mode
- Run specific test files
- Run specific tests
- View reports
- Skip tests
- Run only selected tests

---

# Running All Tests

To execute all Playwright tests, run:

```bash
npx playwright test
```

---

# What Happens?

Playwright automatically:

- Finds all test files    
- Executes all tests
- Runs them headlessly by default
- Runs tests across multiple browsers

---

# Why Multiple Tests Run

You may notice more tests running than expected.

Example:

- 2 tests    
- 3 browsers

Result:

```text
6 tests executed
```

This happens because Playwright runs tests in:

- Chromium
- Firefox
- WebKit

---

# Browser Configuration

Browsers are configured inside:

```text
playwright.config.ts
```

Example:

```typescript
projects: [
	{
		name: 'chromium',
	},
	{
		name: 'firefox',
	},
	{
		name: 'webkit',
	},
]
```

---

# Viewing the HTML Report

After test execution, you can open the Playwright report.

```bash
npx playwright show-report
```

---

# What the Report Shows

The HTML report contains:

- Passed tests    
- Failed tests
- Skipped tests
- Test steps
- Error details
- Screenshots
- Execution time

---

# Running Tests in a Specific Browser

To run tests only in Chromium:

```bash
npx playwright test --project=chromium
```

---

# Running Tests in Headed Mode

By default, Playwright runs tests headlessly.

To see the actual browser execution:

```bash
npx playwright test --project=chromium --headed
```

---

# Headless vs Headed

| Mode     | Description                |
| -------- | -------------------------- |
| Headless | Browser runs in background |
| Headed   | Browser UI becomes visible |

---

# Running a Specific Test File

If you want to run only one spec file:

```bash
npx playwright test example.spec.ts --project=chromium
```

This executes only:

```text
example.spec.ts
```

---

# Running a Specific Test by Name

To run a single test using its title:

```bash
npx playwright test -g "has title" --project=chromium
```

---

# Understanding `-g`

```text
-g
```

Means:

```text
grep
```

It searches test titles and executes matching tests only.

---

# Example Test

```typescript
import { test, expect } from "@playwright/test";

test("has title", async ({ page }) => {

	await page.goto("https://playwright.dev/");

	// Expect a title "to contain" a substring.
	await expect(page).toHaveTitle(/Playwright1/);

});

test("get started link", async ({ page }) => {

	await page.goto("https://playwright.dev/");

	// Click the get started link.
	await page.getByRole("link", { name: "Get started" }).click();

	// Expects page to have a heading with the name of Installation.
	await expect(
		page.getByRole("heading", { name: "Installation" }),
	).toBeVisible();

});
```

---

# Example Failure

In this example:

```typescript
/Playwright1/
```

The expected title is incorrect on purpose.

Result:

- Test fails    
- Report shows expected vs actual value
- Exact failure location appears in the report

---

# Running the Failed Test

```bash
npx playwright test -g "has title" --project=chromium
```

# Skipping a Test

Sometimes we want to temporarily ignore a test.

Use:

```typescript
test.skip()
```

---

# Example

```typescript
import { test, expect } from "@playwright/test";

test.skip("has title", async ({ page }) => {

	await page.goto("https://playwright.dev/");

	// Expect a title "to contain" a substring.
	await expect(page).toHaveTitle(/Playwright1/);

});

test("get started link", async ({ page }) => {

	await page.goto("https://playwright.dev/");

	// Click the get started link.
	await page.getByRole("link", { name: "Get started" }).click();

	// Expects page to have a heading with the name of Installation.
	await expect(
		page.getByRole("heading", { name: "Installation" }),
	).toBeVisible();

});
```

# Running Only One Test

Sometimes during debugging, we want to run only one specific test.

Use:

```typescript
test.only()
```

---

# Example

```typescript
import { test, expect } from "@playwright/test";

test.only("has title", async ({ page }) => {

	await page.goto("https://playwright.dev/");

	// Expect a title "to contain" a substring.
	await expect(page).toHaveTitle(/Playwright1/);

});

test("get started link", async ({ page }) => {

	await page.goto("https://playwright.dev/");

	// Click the get started link.
	await page.getByRole("link", { name: "Get started" }).click();

	// Expects page to have a heading with the name of Installation.
	await expect(
		page.getByRole("heading", { name: "Installation" }),
	).toBeVisible();

});
```

---

# Running the File

```bash
npx playwright test example.spec.ts --project=chromium
```

---

# What Happens?

Only the test marked with:

```typescript
test.only()
```

will execute.

All other tests are ignored.

---

# Common CLI Commands

| Command                      | Purpose              |
| ---------------------------- | -------------------- |
| `npx playwright test`        | Run all tests        |
| `npx playwright show-report` | Open HTML report     |
| `--project=chromium`         | Run specific browser |
| `--headed`                   | Run with browser UI  |
| `-g "test name"`             | Run specific test    |
| `test.skip()`                | Skip test            |
| `test.only()`                | Run only one test    |
