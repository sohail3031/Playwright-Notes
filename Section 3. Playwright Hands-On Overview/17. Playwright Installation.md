In this lesson, we learn how to install the Playwright framework from scratch.

Playwright can be installed in two ways:

1. As a completely new standalone project
2. Inside an existing project

In this lesson, we focus on creating a brand-new Playwright project.

---

# Creating a New Project Folder

First, create a new folder for the project.

Example:

```text
DemoTest
```

Open this folder in **Visual Studio Code**.

---

# Installing Playwright

Open a terminal inside VS Code and run the following command:

```bash
npm init playwright@latest
```

---

# What This Command Does

This command:

- Initializes a new Node.js project    
- Installs Playwright
- Creates the Playwright folder structure
- Installs browsers
- Creates example test files

---

# Installation Wizard Questions

During installation, Playwright asks several setup questions.

---

# 1. Choose Language

Options:

- TypeScript    
- JavaScript

In this course, TypeScript is selected because it is the default language used in Playwright documentation.

---

# 2. Test Folder Location

Default:

```text
tests
```

Keep the default value.

---

# 3. GitHub Actions Workflow

Question:

```text
Add a GitHub Actions workflow?
```

Select:

```text
false
```

We can configure CI/CD later if needed.

---

# 4. Install Playwright Browsers

Question:

```text
Install Playwright browsers?
```

Select:

```text
Yes
```

Playwright will install browsers such as:

- Chromium    
- Firefox
- WebKit (Safari engine)

---

# Installation Complete

After installation finishes, Playwright automatically creates the project structure.

---

# Project Structure Overview

## `node_modules`

```text
node_modules/
```

This folder contains installed libraries and dependencies.

Inside this folder, Playwright itself is installed.

> [!note]  
> You can safely delete `node_modules` anytime and reinstall dependencies using:
> 
> ```bash
> npm install
> ```

---

# `tests` Folder

```text
tests/
```

This folder contains Playwright test files.

Example:

```text
example.spec.ts
```

---

# Naming Convention

Playwright test files usually follow this naming pattern:

```text
*.spec.ts
```

Example:

```text
login.spec.ts
```

The `.spec` naming convention helps identify test files.

---

# `tests-examples` Folder

```text
tests-examples/
```

This folder contains sample Playwright tests created automatically during installation.

These are reference examples and are not commonly used in real frameworks.

---

# `.gitignore`

```text
.gitignore
```

This file defines which files and folders should not be committed to Git.

Common ignored folders:

- `node_modules`
- `playwright-report`
- `test-results`

---

# `package.json`

```text
package.json
```

This is one of the most important files in a Node.js project.

It contains:

- Project information
- Dependencies
- Scripts
- Framework configuration

---

# Example Dependency

```json
"devDependencies": {
	"@playwright/test": "^version"
}
```

---

# `scripts` Section

The `scripts` section contains reusable terminal commands.

Example:

```json
"scripts": {
	"test": "playwright test"
}
```

---

# `package-lock.json`

```text
package-lock.json
```

This file locks dependency versions to avoid compatibility issues.

> [!note]  
> This file can also be safely regenerated using:
> 
> ```bash
> npm install
> ```

---

# `playwright.config.ts`

```text
playwright.config.ts
```

This is the main Playwright configuration file.

It controls:

- Browser settings
- Timeouts
- Retries
- Parallel execution
- Reporters
- Base URLs
- Test configuration

---

# Default Configuration

Initially, Playwright creates a default configuration with:

- Helpful comments    
- Example settings
- Recommended defaults

We usually keep the defaults at the beginning.

---

# Installed Browsers

Playwright downloads multiple browser engines:

| Browser  | Engine          |
| -------- | --------------- |
| Chromium | Chrome/Edge     |
| Firefox  | Mozilla Firefox |
| WebKit   | Safari          |

---

# Why Playwright Uses Multiple Browsers

This allows cross-browser testing using the same automation scripts.

---

# Running Tests

After installation, Playwright is ready to use immediately.

You can run tests using:

```bash
npx playwright test
```

---

# Important Notes

> [!important]  
> Playwright automatically:
> 
> - Sets up TypeScript support
>     
> - Creates test folders
>     
> - Installs browsers
>     
> - Creates configuration files
>     
> - Adds example tests
>     

---

# Best Practices

> [!note]  
> Recommended project structure:
> 
> ```text
> project/
> ├── tests/
> ├── pages/
> ├── utils/
> ├── node_modules/
> ├── playwright.config.ts
> ├── package.json
> ```
> 
> This structure becomes useful as the framework grows.

---

# Summary

- Playwright can be installed as a new or existing project.    
- Use:

```bash
npm init playwright@latest
```

- Playwright creates the project structure automatically.
- Important files include:
    - `package.json`
    - `playwright.config.ts`
    - `tests/`
- Multiple browsers are installed automatically.
- The framework is ready for automation immediately after installation.
