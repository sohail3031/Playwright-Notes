In this lesson, we will learn how to run tests using the Playwright extension and UI mode.

## Running Tests Using the Playwright Extension

Before this, we already installed the Playwright extension in Visual Studio Code. If you click on the **Testing** icon in VS Code, you will see the **Test Explorer** with the list of tests we currently have.

We have a `tests` folder, and under that folder we have `example.spec.ts` with two tests inside it. Those tests are displayed directly in the Test Explorer.

You can run tests directly from the Test Explorer by clicking the **Run** button next to the test.

---

## Playwright Extension Configuration

Before running the tests, let’s look at a few useful configurations.

You can enable or disable **headed mode**:

- If you want to see the browser during execution, enable headed mode.    
- If you do not care about seeing the browser, run the tests headlessly.

You can also select which browsers you want to execute tests on.

Click on the dropdown menu and select the default profiles:

- Chromium
- Firefox
- WebKit

For now, we will use only **Chromium**.

---

## Running the Test

Once Chromium is selected, click the run button.

The browser opens and the tests execute automatically.

One important difference between running tests through the command line and running them through the Playwright extension is:

- In the extension, the browser remains open after execution.    
- The browser stays on the page where the test finished.

For example:

- The test opens the Playwright homepage.
- Clicks the **Get Started** link.
- Navigates to another page.

After execution, the browser remains on that final page.

This is very convenient when you want to continue manual testing or inspect the application after automation finishes.

---

## Test Execution Details

When tests execute, you can also see:

- Execution time for each step    
- Pass/fail status
- Individual test results

You can run tests not only from the Test Explorer, but also directly from the spec file by clicking the run icon above the test.

---

## Running Tests in UI Mode

The second approach is using **Playwright UI Mode**, which is extremely useful for debugging.

To start UI mode, run the following command:

```bash
npx playwright test --ui
```

This opens the Playwright UI Runner.

---

## Features of UI Mode

The UI runner provides a powerful interface for debugging tests.

### Test Explorer

On the left side, you can see the list of tests.

You can:

- Run individual tests    
- Run entire spec files
- Re-run failed tests

---

## Viewing Test Steps

When you click on a test, you can see all recorded steps.

For example:

1. Navigate to the Playwright homepage    
2. Click the **Get Started** button
3. Verify the **Installation** heading

You can visually inspect every step of execution.

---

## Before and After Action States

For each action, Playwright allows you to inspect:

- The state **before** the action    
- The action itself
- The state **after** the action

This is extremely useful for debugging.

For example:

- Before click
- During click
- After navigation

You can even see where the click occurred on the page.

---

## Source Code Highlighting

Another very useful feature is source code highlighting.

As you move between test steps, Playwright automatically highlights the exact line of code responsible for that action.

This makes debugging very efficient.

---

## Difference Between UI Mode and Extension Mode

One important difference:

### UI Mode

- Displays screenshots of execution    
- Great for debugging
- Cannot continue interacting with the application manually

### Extension Mode

- Opens a real browser
- Browser stays open
- Allows manual exploration after test execution

In real projects, developers usually use both approaches together.

---

## Watch Mode

UI mode also supports automatic test re-execution when files change.

Enable the **Watch** icon in the UI runner.

Now whenever you modify your test file, Playwright automatically re-runs the tests.

For example, if you intentionally introduce a failure:

```javascript
await expect(page).toHaveTitle(/Playwright1/)
```

The test automatically fails and the UI runner shows:

- Error message    
- Failed line of code
- Execution logs
- Timeout details

This makes debugging very fast and convenient.

---

## Summary

In this lesson, we learned two different ways to run Playwright tests:

### Playwright Extension

- Run tests directly from VS Code    
- Execute tests in headed or headless mode
- Keep the browser open after execution

### UI Mode

- Powerful debugging interface
- Visual step-by-step execution
- Source code highlighting
- Watch mode support
- Detailed execution logs

Both approaches are very useful and are commonly used together during Playwright test development.
