## Overview

As Playwright projects grow, running tests directly from the terminal using long commands becomes repetitive and error-prone. While Playwright provides a powerful **Command Line Interface (CLI)** for executing tests, remembering and typing lengthy commands repeatedly isn't practical.

To simplify test execution, **NPM scripts** can be defined inside `package.json`. These scripts act as shortcuts for frequently used CLI commands, making it easier to run tests consistently with simple commands.

This lesson covers:

- What the Playwright CLI is
- How to execute tests from the terminal
- Creating reusable NPM scripts
- Running tests in different browsers
- Chaining multiple scripts sequentially
- Running multiple scripts in parallel
- Best practices for organizing project scripts

---

# What is CLI?

**CLI** stands for **Command Line Interface**.

Instead of interacting with Playwright through the VS Code Test Explorer or UI, you can execute Playwright commands directly from the terminal.

For example:

```bash
npx playwright test first.spec.ts --project=chromium
```

This command tells Playwright to:

- Execute a specific test file
- Use the Chromium browser
- Run everything from the terminal

---

# Basic Playwright CLI Command

General syntax:

```bash
npx playwright test <test-file> --project=<browser>
```

Example:

```bash
npx playwright test usePageObjects.spec.ts --project=chromium
```

### Breakdown

|Part|Description|
|---|---|
|`npx`|Executes a local Node.js package|
|`playwright`|Playwright CLI|
|`test`|Runs Playwright tests|
|`usePageObjects.spec.ts`|Test file to execute|
|`--project=chromium`|Browser project defined in `playwright.config.ts`|



# Why Long CLI Commands Become a Problem

Imagine typing this every time:

```bash
npx playwright test usePageObjects.spec.ts --project=chromium
```

Common issues include:

- Long commands
- Typographical mistakes
- Difficult to remember
- Repeated typing
- Harder for team members to standardize

A better solution is to create reusable scripts.

---

# NPM Scripts

Node.js allows commands to be stored inside the `scripts` section of `package.json`.

Example:

```json
{
  "scripts": {
    "pageObjects-chrome": "npx playwright test usePageObjects.spec.ts --project=chromium"
  }
}
```

Instead of typing the full Playwright command, simply run:

```bash
npm run pageObjects-chrome
```

NPM automatically executes the stored command.

---

# Creating Multiple Scripts

You can create separate scripts for different browsers.

```json
{
  "scripts": {
    "pageObjects-chrome":
      "npx playwright test usePageObjects.spec.ts --project=chromium",

    "pageObjects-firefox":
      "npx playwright test usePageObjects.spec.ts --project=firefox"
  }
}
```

Now execution becomes much simpler.

Run Chromium:

```bash
npm run pageObjects-chrome
```

Run Firefox:

```bash
npm run pageObjects-firefox
```

# Chaining Scripts Together

NPM scripts can execute other NPM scripts.

Example:

```json
{
  "scripts": {
    "pageObjects-chrome":
      "npx playwright test usePageObjects.spec.ts --project=chromium",

    "pageObjects-firefox":
      "npx playwright test usePageObjects.spec.ts --project=firefox",

    "pageObjects-all":
      "npm run pageObjects-chrome && npm run pageObjects-firefox"
  }
}
```

Notice:

```text
&&
```

means:

> Execute the second command **only after** the first command finishes successfully.

---

# Sequential Execution

Using:

```text
&&
```

creates sequential execution.

Flow:

```text
Run Chromium Tests
        │
        ▼
Completed Successfully
        │
        ▼
Run Firefox Tests
        │
        ▼
	Finished
```

Command:

```bash
npm run pageObjects-all
```

Execution order:

1. Chromium    
2. Firefox

---

# Parallel Execution

Instead of:

```text
&&
```

use:

```text
&
```

Example:

```json
{
  "scripts": {
    "pageObjects-all":
      "npm run pageObjects-chrome & npm run pageObjects-firefox"
  }
}
```

Flow:

```text
              Start
                │
        ┌───────┴────────┐
        ▼                ▼
 Chromium Tests     Firefox Tests
        │                │
        └───────┬────────┘
                ▼
            Both Finish
```

Both scripts start simultaneously.

---

# Sequential vs Parallel

| Sequential (`&&`)        | Parallel (`&`)             |
| ------------------------ | -------------------------- |
| One after another        | Simultaneously             |
| Easier debugging         | Faster execution           |
| Stops if first fails     | Both start immediately     |
| Good for dependent tasks | Good for independent tasks |

# Real-World Examples of NPM Scripts

Besides Playwright tests, scripts can be used for:

- Starting applications    
- Building projects
- Running linters
- Formatting code
- Generating reports
- Running CI/CD tasks
- Database migrations
- Docker commands
- Running multiple automation suites

Example:

```json
{
  "scripts": {
    "test": "playwright test",
    "test:chrome": "playwright test --project=chromium",
    "test:firefox": "playwright test --project=firefox",
    "lint": "eslint .",
    "format": "prettier --write .",
    "report": "playwright show-report"
  }
}
```

# Best Practices

- Use meaningful script names (e.g., `test:chrome`, `test:firefox`, `test:smoke`) to make their purpose obvious.    
- Keep browser-specific scripts separate for flexibility.
- Create aggregate scripts for running multiple suites.
- Prefer sequential execution (`&&`) when later commands depend on earlier ones.
- Use parallel execution (`&`) only when tasks are completely independent.
- Store commonly used commands in `package.json` instead of repeatedly typing long CLI commands.
- Standardize script names across the team to improve consistency.

---

# Key Takeaways

- The **Playwright CLI** enables running tests directly from the terminal using commands like `npx playwright test`.    
- **NPM scripts** in `package.json` provide convenient shortcuts for frequently used CLI commands.
- Scripts reduce typing, minimize mistakes, and improve consistency across a project.
- Multiple scripts can be chained using `&&` for sequential execution or `&` for parallel execution.
- NPM scripts are not limited to Playwright—they can automate any terminal command, making them valuable for builds, testing, reporting, linting, formatting, and CI/CD workflows.
- Organizing reusable scripts improves maintainability and simplifies day-to-day development.
