# Overview

In real-world automation testing, using **hardcoded test data** is rarely a good practice.

Example:

```ts
await page.fill("#name", "John Smith");
await page.fill("#email", "john@test.com");
```

If this test executes hundreds of times:

- Database becomes filled with identical users.    
- Duplicate email validation may fail.
- Searching created records becomes difficult.
- Tests become less reliable.
- Parallel execution can create conflicts.

Instead, automation engineers generate **dynamic test data** during runtime.

The most commonly used JavaScript library for this purpose is:

> **@faker-js/faker**

---

# Why Random Test Data?

Suppose your test creates users.

Without Faker:

```text
John Smith
john@test.com

John Smith
john@test.com

John Smith
john@test.com
```

Problems:

- Duplicate emails
- Duplicate usernames
- Database pollution
- Difficult debugging
- Parallel execution failures

---

Using Faker:

```text
Sarah Johnson
sarahjohnson528@test.com

James Williams
jameswilliams812@test.com

Emily Brown
emilybrown144@test.com
```

Every execution produces unique values.

# Using Faker in Tests

Instead of:

```ts
await pm
.onFormLayoutsPage()
.submitInlineFormWithNameEmailAndCheckbox(
    "John Smith",
    "john@test.com",
    true
);
```

Use:

```ts
const randomFullName = faker.person.fullName();

await pm
.onFormLayoutsPage()
.submitInlineFormWithNameEmailAndCheckbox(
    randomFullName,
    "john@test.com",
    true
);
```

Now the name changes every test execution.

# Generating Random Emails

Generate a random email using the previously created name.

```ts
const randomEmail =
`${randomFullName}${faker.number.int(1000)}@test.com`;
```

Example:

```text
John Smith542@test.com
```

Unfortunately:

```text
John Smith542@test.com
```

contains a space.

Spaces are invalid inside email addresses.

---

# Removing Spaces

Use JavaScript's `replace()` method.

```ts
const randomEmail =
`${randomFullName.replace(" ", "")}${faker.number.int(1000)}@test.com`;
```

Output:

```text
JohnSmith542@test.com
```

Valid email.

---

# Complete Example

```ts
const randomFullName = faker.person.fullName();

const randomEmail =
`${randomFullName.replace(" ", "")}${faker.number.int(1000)}@test.com`;

await pm
.onFormLayoutsPage()
.submitInlineFormWithNameEmailAndCheckbox(
    randomFullName,
    randomEmail,
    true
);
```

---

# Better Way to Remove Spaces

The instructor uses:

```ts
replace(" ", "")
```

This removes **only the first space**.

Example:

```text
John Michael Smith
```

becomes

```text
JohnMichael Smith
```

A more robust solution:

```ts
replace(/\s+/g, "")
```

Result:

```text
JohnMichaelSmith
```

This removes:

- spaces
- tabs
- multiple spaces

This is the preferred production approach.

# Faker with Playwright Fixtures

Instead of generating data inside every test:

```ts
const name = faker.person.fullName();
const email = faker.internet.email();
```

Create a helper.

```ts
// utils/testData.ts

import { faker } from "@faker-js/faker";

export function generateUser() {
    return {
        name: faker.person.fullName(),
        email: faker.internet.email(),
        password: faker.internet.password()
    };
}
```

Usage:

```ts
const user = generateUser();

await page.fill("#name", user.name);
await page.fill("#email", user.email);
```

This makes your tests cleaner and promotes reuse.

# Industry Best Practices

Instead of manually constructing emails:

```ts
const randomEmail =
`${randomFullName.replace(/\s+/g, "")}${faker.number.int(1000)}@test.com`;
```

Prefer Faker's built-in email generator:

```ts
const email = faker.internet.email();
```

Or use a consistent domain:

```ts
const email = faker.internet.email({
    provider: "test.com"
});
```

This produces valid email formats without manual string manipulation.

# Common Mistakes

### Using Static Data

```ts
"John Smith"
```

Leads to duplicate records and flaky tests.

---

### Manual Random Numbers Only

```ts
Math.random()
```

Less expressive and harder to control compared to Faker.

---

### Forgetting to Remove Spaces

```text
John Smith@test.com
```

Invalid email format.

---

### Generating Data Multiple Times

```ts
faker.person.fullName();

faker.person.fullName();
```

This may produce different values within the same test, making assertions difficult.

Instead:

```ts
const user = {
    name: faker.person.fullName(),
    email: faker.internet.email()
};
```

Reuse the same generated object throughout the test.

- Generates realistic data
- Eliminates duplicate records
- Improves test reliability
- Supports parallel execution
- Easy to use
- Highly configurable
- Large variety of data types
- Widely adopted in industry
- Integrates seamlessly with Playwright, Jest, Cypress, and other JavaScript testing frameworks

---

# Key Takeaways

- Avoid hardcoded test data in automation scripts.    
- Use **`@faker-js/faker`** to generate dynamic and realistic test data.
- Generate data once per test and reuse it consistently.
- Prefer built-in generators like `faker.internet.email()` over manual string concatenation.
- Extract data generation into helper functions or fixtures for better maintainability.
- Dynamic test data reduces duplication, improves parallel execution, and makes tests more reliable and production-ready.

# Code


```ts
import { test } from "@playwright/test";
import { PageManager } from "../pages/pageManager";
import { faker } from "@faker-js/faker";
  
test.beforeEach(async ({ page }) => {
  await page.goto("http://localhost:4200");
});
  
test("Parametrized Methods", async ({ page }) => {
  const pm = new PageManager(page);
  const randomFullName = faker.person.fullName();
  const randomEmail = `${randomFullName.replace(/\s+/g, "")}${faker.number.int(1000)}@test.com`;
  
  await pm.navigateTo().formLayoutsPage();
  await pm
    .onFormLayoutsPage()
    .submitUsingTheGridFormWithCredentialsAndSelectOption(
      "test@test.com",
      "Welcome1",
      "Option 2",
    )
  await pm
    .onFormLayoutsPage()
    .submitInlineFormWithNameEmailAndCheckbox(
      randomFullName
      randomEmail,
      true,
    );
});
```
