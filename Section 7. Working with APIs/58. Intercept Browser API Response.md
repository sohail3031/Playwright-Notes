## Overview

In the previous lesson, we learned how to make **direct API requests** from Playwright. In this lesson, we combine **browser automation** with **API requests**.

Instead of allowing the browser to communicate directly with the backend, Playwright will:

1. Intercept the browser's API request.    
2. Fetch the original API response.
3. Modify the response.
4. Send the modified response back to the browser.

This allows us to simulate different scenarios without changing the backend.

---

# Why Intercept API Responses?

Sometimes we want to test UI behavior without changing the backend database.

Examples:

- Display custom test data    
- Simulate edge cases
- Remove unwanted data
- Test error handling
- Verify UI with controlled responses

Instead of creating or deleting records in the real database, we simply modify the response before the browser receives it.

---

# Scenario

The application displays a list of articles.

When the Home page loads, the browser calls:

```text
GET /api/articles
```

The server returns something similar to:

```json
{
  "articles": [
    {
      "title": "Playwright Guide",
      "description": "Learning Playwright"
    },
    {
      "title": "Testing APIs",
      "description": "API Testing"
    }
  ]
}
```

Our goal is to remove one article before the browser receives the response.

---

# API Flow

```text
Browser
   │
   │ GET /api/articles
   ▼
Playwright intercepts request
   │
   ▼
Backend API
   │
   ▼
Original Response
   │
   ▼
Playwright modifies response
   │
   ▼
Modified Response
   │
   ▼
Browser UI
```

---

# Step 1: Intercept the Request

Before opening the application:

```ts
await page.route("*/**/api/articles*", async (route) => {

});
```

The wildcard matches every request ending with:

```http
/api/articles
```

or

```http
/api/articles?limit=10&offset=0
```

---

# Step 2: Fetch the Original Response

Allow the request to go to the server.

```ts
const response = await route.fetch();
```

Unlike mocking, this still contacts the real backend.

---

# Step 3: Convert Response to JSON

```ts
const responseBody = await response.json();
```

Now we can manipulate the returned object.

---

# Example Response

```json
{
  "articles": [
    {
      "title": "Article 1"
    },
    {
      "title": "Article 2"
    },
    {
      "title": "Article 3"
    }
  ]
}
```

---

# Step 4: Modify the Data

Suppose we want to remove the second article.

Use JavaScript's `splice()`.

```ts
responseBody.articles.splice(1, 1);
```

Meaning:

```text
Start at index 1
Remove 1 element
```

Result:

Before

```text
0 → Article 1
1 → Article 2
2 → Article 3
```

After

```text
0 → Article 1
1 → Article 3
```

---

# Step 5: Fulfill the Request

Send the modified response back to the browser.

```ts
await route.fulfill({
  body: JSON.stringify(responseBody)
});
```

The browser never knows the response was modified.

---

# Complete Example

```ts
await page.route("*/**/api/articles*", async (route) => {
    const response = await route.fetch();
    const responseBody = await response.json();

    responseBody.articles.splice(1, 1);

    await route.fulfill({
        body: JSON.stringify(responseBody),
    });
});
```

---

# Why Use `route.fetch()`?

Without it:

```text
Browser
   │
Playwright
   │
Mock Response
```

With it:

```text
Browser
   │
Playwright
   │
Backend
   │
Original Response
   │
Playwright modifies
   │
Browser
```

You're **editing** the real response rather than replacing it completely.

---

# Common JavaScript Array Methods

## Remove One Element

```ts
array.splice(index, 1);
```

Example

```ts
articles.splice(0, 1);
```

Removes the first article.

---

## Remove Multiple Elements

```ts
articles.splice(2, 3);
```

Removes three elements starting at index 2.

---

## Add an Element

```ts
articles.push(newArticle);
```

---

## Insert at Beginning

```ts
articles.unshift(newArticle);
```

---

## Replace an Element

```ts
articles[0] = {
    title: "New Title"
};
```

---

## Filter Elements

```ts
responseBody.articles =
    responseBody.articles.filter(
        article => article.author.username !== "admin"
    );
```

Removes all articles written by `admin`.

---

# Example: Modify First Article

```ts
responseBody.articles[0].title =
    "Playwright API Testing";

responseBody.articles[0].description =
    "Modified using Playwright";
```

---

# Example: Add a Fake Article

```ts
responseBody.articles.unshift({
    title: "Automation Testing",
    description: "Inserted by Playwright"
});
```

Now the fake article appears first.

---

# Example: Remove All Articles

```ts
responseBody.articles = [];
```

Useful for testing the empty-state UI.

---

# Difference Between Mocking and Modifying

| Mocking                      | Modifying Response                           |
| ---------------------------- | -------------------------------------------- |
| Doesn't contact backend      | Contacts backend                             |
| Returns completely fake data | Uses real backend data                       |
| Faster                       | Slightly slower                              |
| Useful for isolated testing  | Useful when only small changes are needed    |
| `route.fulfill()` only       | `route.fetch()` + modify + `route.fulfill()` |

# Real-World Uses

## Hide Test Data

Remove unwanted records.

```ts
responseBody.articles =
responseBody.articles.filter(
    article => article.title !== "Test"
);
```

---

## Change User Role

```ts
responseBody.user.role = "Admin";
```

Test admin features without modifying the database.

---

## Simulate Empty Results

```ts
responseBody.articles = [];
```

---

## Simulate One Result

```ts
responseBody.articles =
[
    responseBody.articles[0]
];
```

---

## Test Large Data

```ts
for (let i = 0; i < 100; i++) {
    responseBody.articles.push({
        title: `Article ${i}`,
        description: "Generated"
    });
}
```

---

# Important Notes

- Always define `page.route()` **before** `page.goto()`.    
- Use `await route.fetch()` to retrieve the original server response.
- Modify only the data you need.
- Convert the modified object back to JSON using `JSON.stringify()`.
- Send it back with `route.fulfill()`.

```text
Browser
     │
     ▼

     │
     ▼

     │
     ▼
Original JSON Response
     │
     ▼

     │
     ▼

     │
     ▼

     │
     ▼

```

---

# Key Takeaways

- **`page.route()`** intercepts network requests.    
- **`route.fetch()`** retrieves the original backend response.
- **`response.json()`** converts the response into a JavaScript object for easy modification.
- You can add, remove, replace, or filter data before the browser receives it.
- **`route.fulfill()`** sends the modified response back to the browser.
- This approach is ideal for testing UI behavior with controlled data while still using the real backend response.

```js
test("Create Article", async ({ page, request }) => {
  await page.getByText("New Article").click();
  await page
    .getByRole("textbox", { name: "Article Title" })
    .fill("Playwright is awesome");
  await page
    .getByRole("textbox", { name: "What's this article about?" })

  await page
    .getByRole("textbox", { name: "Write your article (in markdown)" })
    .fill("We like to use playwright for automation");

  const articleResponse = await page.waitForResponse(
    "https://conduit-api.bondaracademy.com/api/articles/",
  );
  const articleResponseBody = await articleResponse.json();
  const slugId = articleResponseBody.article.slug;

  await expect(page.locator(".article-page h1")).toContainText(
    "Playwright is awesome",
  );
  
  await page.getByText("Home").click();
  await page.getByText("Global Feed").click();
  
  await expect(page.locator("app-article-list h1").first()).toContainText(
    "Playwright is awesome",
  );
  
  const response = await request.post(
    "https://conduit-api.bondaracademy.com/api/users/login",
    {
      data: {
        user: { email: "email", password: "password" },
      },
    },
  );
  const responseBody = await response.json();
  const accessToken = responseBody.user.token;
  const deleteArticleResponse = await request.delete(
    `https://conduit-api.bondaracademy.com/api/articles/${slugId}`,
    {
      headers: {
        Authorization: `Token ${accessToken}`,
      },
    },
  );

  expect(deleteArticleResponse.status()).toEqual(204);
});
```
