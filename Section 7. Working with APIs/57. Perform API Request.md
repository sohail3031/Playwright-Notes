In this lesson, we move beyond intercepting browser requests and learn how to **send API requests directly from Playwright**.

Instead of waiting for the browser to perform an action (like clicking a button), Playwright can communicate directly with the backend API. This is extremely useful for:

- Setting up test data    
- Cleaning up test data
- Logging in without using the UI
- Speeding up test execution
- Testing APIs independently from the UI

---

# Why Perform API Requests Directly?

Normally the flow is:

```text
Playwright
     │
     ▼
Browser
     │
     ▼
Backend API
     │
     ▼
Database
```

When using Playwright's APIRequestContext:

```text
Playwright
     │
     ▼
Backend API
     │
     ▼
Database
```

The browser is skipped completely.

This makes tests:

- Much faster
- More reliable
- Easier to prepare test data

---

# Playwright APIRequestContext

Playwright provides a built-in fixture called:

```ts
request
```

This fixture allows sending HTTP requests directly.

Example:

```ts
test("API Request", async ({ request }) => {

});
```

Unlike previous tests:

```ts
async ({ page })
```

we now use

```ts
async ({ request })
```

or both

```ts
async ({ page, request })
```

if the test needs both browser and API.

---

# Common API Methods

Playwright supports all common HTTP methods.

| Method | Purpose        |
| ------ | -------------- |
| GET    | Retrieve data  |
| POST   | Create data    |
| PUT    | Update data    |
| PATCH  | Partial update |
| DELETE | Remove data    |

Examples:

```ts
await request.get(...)
```

```ts
await request.post(...)
```

```ts
await request.put(...)
```

```ts
await request.delete(...)
```

---

# Making a GET Request

Example:

```ts
const response = await request.get(
    "https://conduit-api.bondaracademy.com/api/tags"
);
```

This sends

```http
GET /api/tags
```

directly to the backend.

No browser opens.

---

# Response Object

The returned object contains lots of useful information.

```ts
const response = await request.get(...);
```

Available methods include:

```ts
response.status()
```

```ts
response.ok()
```

```ts
response.headers()
```

```ts
response.body()
```

```ts
response.json()
```

---

# Reading JSON Response

Most REST APIs return JSON.

Instead of reading raw text,

use

```ts
const responseBody = await response.json();
```

Now you can access properties normally.

Example:

```ts
console.log(responseBody.tags);
```

Output

```text
[
  "welcome",
  "implementations",
  "introduction",
  ...
]
```

---

# Verifying Status Code

Status code can be validated.

```ts
expect(response.status()).toBe(200);
```

or

```ts
expect(response.ok()).toBeTruthy();
```

Both verify the request succeeded.

---

# Example Test

```ts
import { test, expect } from "@playwright/test";

test("GET Tags API", async ({ request }) => {

    const response = await request.get(
        "https://conduit-api.bondaracademy.com/api/tags"
    );

    expect(response.status()).toBe(200);

    const body = await response.json();

    console.log(body);
});
```

# Sending Query Parameters

Instead of manually building URLs,

Playwright supports

```ts
params
```

Example:

```ts
await request.get("/api/articles", {
    params: {
        limit: 10,
        offset: 0
    }
});
```

Equivalent URL:

```http
/api/articles?limit=10&offset=0
```

---

# Sending Headers

Headers can also be added.

Example:

```ts
await request.get(url, {

    headers: {

        Authorization: "Token xyz",

        Accept: "application/json"

    }

});
```

Useful for authenticated requests.

---

# Base URL

Instead of writing the complete URL every time,

configure it inside

`playwright.config.ts`

```ts
use: {

    baseURL: "https://conduit-api.bondaracademy.com"

}
```

Now requests become

```ts
await request.get("/api/tags");
```

instead of

```ts
await request.get(
"https://conduit-api.bondaracademy.com/api/tags"
);
```

# Difference Between Browser and API Request

### Browser Request

```ts
await page.goto(...)

await page.click(...)

await page.fill(...)
```

Uses the UI.

Slower.

Tests UI and backend together.

---

### API Request

```ts
await request.get(...)

await request.post(...)
```

No browser.

Very fast.

Useful for:

- Creating users    
- Creating articles
- Cleaning test data
- Logging in
- Seeding database

---

# Browser vs API

| Browser Automation     | API Automation                        |
| ---------------------- | ------------------------------------- |
| Opens browser          | No browser                            |
| Slower                 | Faster                                |
| Tests UI               | Tests backend                         |
| Can fail because of UI | Independent of UI                     |
| Good for E2E           | Good for setup and backend validation |

---

# Best Practice

Use API requests for:

- Test setup    
- Test cleanup
- Login
- Creating test data
- Deleting test data
- Updating records
- Backend validation

Use browser automation for:

- User interactions
- Visual validation
- UI workflows
- End-to-End scenarios

The combination of both approaches results in fast, stable, and comprehensive automated tests.

---

# Key Takeaways

- **Playwright** provides the `request` fixture for direct API communication.    
- Use `request.get()`, `request.post()`, `request.put()`, `request.patch()`, and `request.delete()` to interact with REST APIs.
- API requests bypass the browser, making tests significantly faster.
- Parse JSON responses using `await response.json()`.
- Validate responses with `response.status()` or `response.ok()`.
- Configure a `baseURL` to simplify API calls.
- Direct API requests are ideal for test setup, cleanup, authentication, and backend verification, while browser automation remains essential for testing user-facing functionality.

# Code

```js
import { test, expect } from "@playwright/test";
import tags from "../test-data/tags.json";

test.beforeEach(async ({ page }) => {
  await page.goto("https://conduit.bondaracademy.com/");
  await page.route(
    "https://conduit-api.bondaracademy.com/api/tags",
    async (route) => {
      await route.fulfill({ body: JSON.stringify(tags) });
    },
  );
  await page.getByRole("link", { name: "Sign in" }).click();
  await page.getByRole("textbox", { name: "Email" }).fill("email");
  await page.getByRole("textbox", { name: "Password" }).fill("pasword");
  await page.getByRole("button", { name: "Sign in" }).click();
});

test("Create Article", async ({ page }) => {
  await page.waitForTimeout(1000);
  await page.route("*/**/api/articles*", async (route) => {
    const response = await route.fetch();
    const responseBody = await response.json();
    responseBody.articles[0].title = "This is a test mock title";
    responseBody.articles[0].description = "This is a mock description";

    await route.fulfill({
      body: JSON.stringify(responseBody),
    });
  });
  await page.getByText("Global Feed").click();
  await expect(page.locator(".navbar-brand")).toHaveText("conduit");
  await expect(page.locator("app-article-list h1").first()).toContainText(
    "This is a test mock title",
  );
  await expect(page.locator("app-article-list p").first()).toContainText(
    "This is a mock description",
  );
});

test("Delete Article", async ({ page, request }) => {
  const response = await request.post(
    "https://conduit-api.bondaracademy.com/api/users/login",
    {
      data: {
        user: { email: "email", password: "password" },
      },
    },
  );
  const responseBody = await response.json();
  const accessToken = responseBody.user.token;
  const articleResponse = await request.post(
    "https://conduit-api.bondaracademy.com/api/articles/",
    {
      data: {
        article: {
          title: "This is a test title",
          description: "This is a description",
          body: "This is a body",
          tagList: [],
        },
      },
      headers: {
        authorization: `Token ${accessToken}`,
      },
    },
  );
  
  expect(articleResponse.status()).toEqual(201);

  await page.getByText("Global Feed").click();
  await page.getByText("This is a test title").click();
  await page.getByRole("button", { name: "Delete Article" }).first().click();
  await page.getByText("Global Feed").click();
  await expect(page.locator("app-article-list h1").first()).not.toContainText(
    "This is a test title",
  );
});
```
