## Overview

In the previous lessons, every test logged into the application before executing.

Although this works, it has several drawbacks:

- Slower test execution    
- Repeated login steps in every test
- Code duplication
- More maintenance if the login flow changes

Playwright provides a better solution by allowing us to **save the authenticated browser state** (cookies, local storage, session storage) into a file and **reuse it across all tests**.

This technique is called **Sharing Authentication State**.

---

# The Problem

Without shared authentication, every test performs these steps:

```text
Open Website
      ↓
Click Sign In
      ↓
Enter Email
      ↓
Enter Password
      ↓
Click Login
      ↓
Wait for Login
      ↓
Execute Test
```

Now imagine you have:

- 100 tests    
- Login takes 3 seconds

Total login time:

```text
100 × 3 seconds = 300 seconds
```

Five minutes are spent only logging in.

---

# Better Approach

Login once.

Save authentication.

Reuse it everywhere.

```text
Login
    ↓
Save Browser State
    ↓
Run Test 1
Run Test 2
Run Test 3
Run Test 100
```

The login happens only once.

---

# What is Stored?

Playwright stores the browser authentication information inside a JSON file.

This includes:

- Cookies    
- Local Storage
- Session Storage (where applicable)

Example:

```text
.auth/
    user.json
```

This file represents a logged-in browser.

# Step 5: Wait Until Login Completes

Don't immediately save the browser state.

Wait until the application is fully authenticated.

Example:

```ts
await page.waitForResponse(
    "https://conduit-api.bondaracademy.com/api/tags"
);
```

Why?

Because login might still be processing.

Saving too early could result in an incomplete authentication state.

---

# Step 6: Save Browser State

```ts
await page.context().storageState({
    path: authFile
});
```

This creates:

```text
.auth/
    user.json
```

Now the browser state is saved.

---

# Complete `auth.setup.ts`

```ts
import { test as setup } from "@playwright/test";

const authFile = ".auth/user.json";

setup("Authentication", async ({ page }) => {

    await page.goto("https://conduit.bondaracademy.com/");

    await page.getByRole("link", {
        name: "Sign in"
    }).click();

    await page.getByRole("textbox", {
        name: "Email"
    }).fill("email");

    await page.getByRole("textbox", {
        name: "Password"
    }).fill("password");

    await page.getByRole("button", {
        name: "Sign in"
    }).click();

    await page.waitForResponse(
        "https://conduit-api.bondaracademy.com/api/tags"
    );

    await page.context().storageState({
        path: authFile
    });
});
```

Add a new project.

```ts
}
```

This tells Playwright:

> Run this file before all other tests.

# Use Authentication State

Inside every browser project:

```ts
storageState: ".auth/user.json"
```

Example:

```ts
{
    name: "chromium",

    use: {
        ...devices["Desktop Chrome"],
        storageState: ".auth/user.json"
    }
}
```

Now Chromium starts already logged in.

Do the same for Firefox.

```ts
{
    name: "firefox",

    use: {
        ...devices["Desktop Firefox"],
        storageState: ".auth/user.json"
    }
}
```

And WebKit.

```ts
{
    name: "webkit",

    use: {
        ...devices["Desktop Safari"],
        storageState: ".auth/user.json"
    }
}
```

---

# Add Dependencies

Each browser project should depend on the setup project.

```ts
dependencies: ["setup"]
```

Example:

```ts
{
    name: "chromium",

    use: {
        storageState: ".auth/user.json"
    },

    dependencies: ["setup"]
}
```

Meaning:

```text
Run setup first

↓

Then run Chromium tests
```

# Authentication State vs API Login

## Storage State

```text

	↓

Save Cookies

	↓

Reuse Cookies
```

Used for browser tests.

---

```text
POST /login

	↓

Receive Token

	↓

Use Token
```

Used for API testing.

# Common Mistakes

## Saving Too Early

❌

```ts
await page.click("Sign in");

await page.context().storageState(...);
```

The login may not have completed.

---

✅

```ts
await page.waitForResponse(...);

await page.context().storageState(...);
```

---

## Forgetting `dependencies`

Without:

```ts
dependencies: ["setup"]
```

Playwright may run tests before authentication is created.

---

## Committing `.auth`

Never commit:

```text
.auth/user.json
```

Always ignore it.

---

# Best Practices

- Create a dedicated `auth.setup.ts` file for authentication.    
- Save authentication using `page.context().storageState()`.
- Store the authentication file in a `.auth` folder.
- Add `.auth` to `.gitignore`.
- Configure `storageState` for all browser projects.
- Use `dependencies: ["setup"]` to ensure authentication runs first.
- Wait until login is fully complete before saving the browser state.

# Key Takeaways

- **Sharing authentication state** avoids repeated login operations across tests.    
- `page.context().storageState()` saves cookies and storage into a JSON file.
- A dedicated `auth.setup.ts` project performs the login once.
- Configure `storageState` in each browser project so tests start authenticated.
- Use `dependencies: ["setup"]` to ensure authentication is created before any test runs.
- This approach improves **test speed, maintainability, and readability**, especially in large test suites.

```ts
// auth.setup.ts
import { test as setup } from "@playwright/test";

const authFile = ".auth/user.json";

setup("Authentication", async ({ page }) => {
  await page.goto("https://conduit.bondaracademy.com/");
  await page.getByRole("link", { name: "Sign in" }).click();
  await page.getByRole("textbox", { name: "Email" }).fill("email");
  await page.getByRole("textbox", { name: "Password" }).fill("password");
  await page.getByRole("button", { name: "Sign in" }).click();
  await page.waitForResponse("https://conduit-api.bondaracademy.com/api/tags");
  await page.context().storageState({ path: authFile });
});
```

```ts
import { test, expect } from "@playwright/test";
import tags from "../test-data/tags.json";
  
test.beforeEach(async ({ page }) => {
  await page.goto("https://conduit.bondaracademy.com/");
  await page.route(
    "https://conduit-api.bondaracademy.com/api/tags",
    async (route) => {
      await route.fulfill({ body: JSON.stringify(tags) });
    },
  );
});
  
test("has title", async ({ page }) => {
  await page.waitForTimeout(1000);
  await page.route("*/**/api/articles*", async (route) => {
    const response = await route.fetch();
    const responseBody = await response.json();
    responseBody.articles[0].title = "This is a test mock title";
    responseBody.articles[0].description = "This is a mock description";

    await route.fulfill({
      body: JSON.stringify(responseBody),
    });
  });
  await page.getByText("Global Feed").click();
  await expect(page.locator(".navbar-brand")).toHaveText("conduit");
  await expect(page.locator("app-article-list h1").first()).toContainText(
    "This is a test mock title",
  );
  await expect(page.locator("app-article-list p").first()).toContainText(
    "This is a mock description",
  );
});
  
test("Delete Article", async ({ page, request }) => {
  const response = await request.post(
    "https://conduit-api.bondaracademy.com/api/users/login",
    {
      data: {
        user: { email: "email", password: "password" },
      },
    },
  );
  const responseBody = await response.json();
  const accessToken = responseBody.user.token;
  const articleResponse = await request.post(
    "https://conduit-api.bondaracademy.com/api/articles/",
    {
      data: {
        article: {
          title: "This is a test title",
          description: "This is a description",
          body: "This is a body",
          tagList: [],
        },
      },
      headers: {
        authorization: `Token ${accessToken}`,
      },
    },
  );
  
  expect(articleResponse.status()).toEqual(201);
  
  await page.getByText("Global Feed").click();
  await page.getByText("This is a test title").click();
  await page.getByRole("button", { name: "Delete Article" }).first().click();
  await page.getByText("Global Feed").click();
  await expect(page.locator("app-article-list h1").first()).not.toContainText(
    "This is a test title",
  );
});

test("Create Article", async ({ page, request }) => {
  await page.getByText("New Article").click();
  await page
    .getByRole("textbox", { name: "Article Title" })
    .fill("Playwright is awesome");
  await page
    .getByRole("textbox", { name: "What's this article about?" })
    .fill("About the Playwright");
  await page
    .getByRole("textbox", { name: "Write your article (in markdown)" })
    .fill("We like to use playwright for automation");
  await page.getByRole("button", { name: "Publish Article" }).click();

  const articleResponse = await page.waitForResponse(
    "https://conduit-api.bondaracademy.com/api/articles/",
  );
  const articleResponseBody = await articleResponse.json();
  const slugId = articleResponseBody.article.slug;

  await expect(page.locator(".article-page h1")).toContainText(
    "Playwright is awesome",
  );

  await page.getByText("Home").click();
  await page.getByText("Global Feed").click();

  await expect(page.locator("app-article-list h1").first()).toContainText(
    "Playwright is awesome",
  );

  const response = await request.post(
    "https://conduit-api.bondaracademy.com/api/users/login",
    {
      data: {
        user: { email: "email", password: "password" },
      },
    },
  );
  const responseBody = await response.json();
  const accessToken = responseBody.user.token;
  const deleteArticleResponse = await request.delete(
    `https://conduit-api.bondaracademy.com/api/articles/${slugId}`,
    {
      headers: {
        Authorization: `Token ${accessToken}`,
      },
    },
  );

  expect(deleteArticleResponse.status()).toEqual(204);
});
```
